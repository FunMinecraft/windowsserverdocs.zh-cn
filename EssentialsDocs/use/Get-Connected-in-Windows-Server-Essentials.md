###  <a name="BKMK_2"></a>一台计算机连接到服务器的先决条件  
 将计算机连接到该网络之前，必须满足以下要求：  
  
-   Windows Server Essentials 安装完成之后，并且正在运行的服务器。 如果无法与服务器，连接器软件将退出其安装。  
  

-   客户端计算机运行的受支持的操作系统。 有关详细信息，请参阅[受支持的客户端计算机的操作系统](Get-Connected-in-Windows-Server-Essentials.md#BKMK_4)。

  
-   客户端计算机必须具有有效的连接到 Internet。  
  
-   客户端计算机位于同一 IP 子网服务器的相同网络的客户端计算机时运行 Windows Server Essentials 服务器。  
  
-   客户端计算机具有.NET Framework 4.5 安装它。 接头软件会自动将其安装在计算机上。  
  
-   客户端计算机满足以下最低系统要求：  
  
    -   1.4 GHz 或更快的处理器  
  
    -   1 GB RAM 或更大  
  
    -   1 GB 的可用硬盘空间（此方面的一部分安装后释放）  
  
-   使用 Windows NT 文件系统进行格式化，启动分区（，即装有 Windows 操作系统的磁盘分区）。  
  
-   计算机名称不包括 15 多个字符。  
  
-   计算机名称不包括下划线 (_)。  
  
-   计算机的日期和时间设置对齐服务器上的设置。  
  
-   在任意给定时间，客户端计算机可以连接到只有一个 Windows Server Essentials 服务器。  
  
-   已加入了另一个 Active Directory 的域的客户端计算机不能加入 Windows Server Essentials 域。  
  
> [!NOTE]

>  在 Windows Server Essentials 或 Windows Server Essentials 本地客户端部署中，你可以将计算机连接到服务器而不是将它们添加到 Windows Server Essentials 域。 此方法也适用于所有受支持的客户端操作系统和的功能，例如组策略和虚拟专用网络 (Vpn)，这需要一台计算机连接到域，不可用。 对于要求和说明进行操作，请参阅[将计算机连接到 Windows Server Essentials 服务器，而不是加入域](Get-Connected-in-Windows-Server-Essentials.md#BKMK_10)。  
  
 若要将计算机连接到运行 Windows Server Essentials 服务器的分步说明，请参阅[将计算机连接到服务器](Get-Connected-in-Windows-Server-Essentials.md#BKMK_9)。  

>  在 Windows Server Essentials 或 Windows Server Essentials 本地客户端部署中，你可以将计算机连接到服务器而不是将它们添加到 Windows Server Essentials 域。 此方法也适用于所有受支持的客户端操作系统和的功能，例如组策略和虚拟专用网络 (Vpn)，这需要一台计算机连接到域，不可用。 对于要求和说明进行操作，请参阅[将计算机连接到 Windows Server Essentials 服务器，而不是加入域](Get-Connected-in-Windows-Server-Essentials.md#BKMK_10)。  
  
 若要将计算机连接到运行 Windows Server Essentials 服务器的分步说明，请参阅[将计算机连接到服务器](Get-Connected-in-Windows-Server-Essentials.md#BKMK_9)。  

  
###  <a name="BKMK_3"></a>将 Mac 计算机连接到该网络的先决条件  
 Mac 计算机连接到该网络之前，必须满足以下要求：  
  
-   服务器操作系统安装完成之后，并且正在运行的服务器。 如果它不能与服务器通信，将无法安装接头软件。  
  
-   计算机正在运行 Mac OS X 10.5（美洲虎）或更高版本。  
  
-   计算机是在同一 IP 子网服务器上。  
  
-   该计算机必须具有有效的连接到 Internet。  
  
-   确保计算机满足以下最低系统要求：  
  
    -   1.4 GHz 或更快的处理器  
  
    -   1 GB RAM 或更大  
  
    -   1 GB 的可用硬盘空间（此方面的一部分安装后释放）  
  
-   在任意给定时间，客户端计算机可以连接到只有一个服务器。

###  <a name="BKMK_5"></a>对做的更改服务器客户端计算机  
 时将计算机连接到服务器时，Windows Server Essentials 软件会对计算机进行了大量更改以便计算机和服务器可以在一起工作。  
  
 该软件将执行以下操作：  
  
-   安装在计算机上的接头软件  
  
-   在计算机上安装 Microsoft.NET Framework 4.5，如果您尚未安装  
  
-   在计算机的桌面上仪表板和启动栏创建快捷方式  
  
-   若要允许以下功能，用于工作的计算机上配置 Windows 防火墙端口：  
  
    -   Core 网络  
  
    -   远程桌面服务  
  
-   到计算机，便于实施备份，做出以下更改：  
  
    -   创建计划的任务中，运行自动备份  
  
    -   安装管理与服务器备份操作的服务  
  
    -   安装的文件和文件夹期间使用虚拟设备驱动器恢复过程  
  
-   安装健康代理检测问题和创建相应的警报通知  
  
-   定期健康评估和同步健康警报定义的计算机上创建的计划的任务  
  
-   添加到计算机，该计算机使用服务器和其他 Windows Server Essentials 功能进行通信的服务  
  
-   在运行 Windows 客户端，允许运行远程桌面服务客户端计算机中打开 TCP 端口 3389  
  
-   部署了 VPN 客户端计算机上，并提供一个单击体验，如果 VPN 功能已启用对 Windows Server Essentials，或者如果 VPN 功能启用 Windows Server Essentials 上提供 auto-connect 体验

###  <a name="BKMK_7"></a>服务器的管理员帐户  
 你必须能够提供的网络管理员帐户名称和密码才能安装接头软件。 网络管理员帐户使用户局域网针对你的组织进行管理，并且可以帮助管理和维护网络设备，如切换和路由器。  
  
 可以通过使用网络管理员帐户中执行的任务可能包括：  
  
-   安装网络应用程序和其他软件  
  
-   管理存储在服务器上  
  
-   分发软件更新  
  
-   执行例行备份  
  
-   监视网络上的日常活动  
  
 在 Windows Server Essentials、Windows Server Essentials 和 Windows Server 2012 R2 安装了 Windows Server Essentials 体验角色，可以将网络管理员访问到任一用户帐户级别。 这将授予执行网络管理员任务所需的权限。 当网络管理员联系分配用户访问级别，**用户访问控制**提示位置打开的任何需要管理员权限的任务。  
  
###  <a name="BKMK_8"></a>从 Windows 域删除一台计算机  
 若要删除一台计算机与其域，将会提示你输入的用户名和密码域帐户。  
  
##### <a name="to-remove-a-computer-from-a-windows-domain"></a>若要从 Windows 域删除一台计算机  
  
1.  单击**开始**，右键单击**计算机**，然后单击**属性**。  
  
2.  下**计算机名称、域和工作组设置**，单击**更改设置**。  
  
    > [!NOTE]
    >  如果系统提示您输入管理员密码或进行确认，请键入该域密码或进行确认。  
  
3.  在**系统属性**对话框中，单击**计算机名称**选项卡，然后单击**更改**。  
  
4.  在**计算机名月域更改**对话框下**的成员**，单击**工作组**，然后执行下列情况之一：  
  
    1.  若要加入现有工作组中，键入你想要加入，然后单击工作组名称**确定**。  
  
    2.  若要创建工作组中，键入你想要创建，然后单击工作组名称**确定**。  
  
        > [!NOTE]
        >  从域，将删除你的计算机并将禁用计算机上的域帐户。

###  <a name="BKMK_10"></a>将计算机连接到 Windows Server Essentials 服务器，而不是加入域  
 本主题介绍了如何将 Windows 7、Windows 8、Windows 8.1 或 Windows 10 的计算机添加到 Windows Server Essentials 网络，不到中本地客户端部署 Windows Server Essentials 域加入计算机。 Windows Server Essentials 和 Windows Server Essentials 支持这种连接方法。  
  
 这是通常的方法，需要将计算机添加到 Windows Server Essentials 域另一种方法。 使用此方法，如果在另一个域，计算机处于它必须从该域加入 Windows Server Essentials 域之前。  
  
#### <a name="feature-limits"></a>功能限制  
 客户端计算机未添加到 Windows Server Essentials 域时，某些功能将受到限制：  
  
-   需要计算机加入域的所有功能？包括域凭据、组策略和虚拟专用网络 (Vpn)？不可用。  
  
-   需要计算机加入域的任何第三方加载项无法正常工作。  
  
-   无法使用此方法连接到服务器的本地关闭计算机。  
  
#### <a name="prerequisites"></a>先决条件  
  
-   该计算机必须具有物理连接到本地网络。  
  
-   在计算机上必须安装下列操作系统之一：  
  
    -   Windows 10 专业版、Windows 10 企业版  
  
    -    Windows 8.1 专业版、Windows 8.1 企业版、Windows 8 专业版、Windows 8 企业版  
  
    -    Windows 7 专业（x86 和 x64），Windows 7 企业版（x86 和 x64）、Windows 7 旗舰版（x86 和 x64）  
  

-   计算机必须满足对于在 Windows Server Essentials 客户端计算机的所有其他要求。 有关详细信息，请参阅[计算机连接到服务器的先决条件](Get-Connected-in-Windows-Server-Essentials.md#BKMK_2)。  

  
-   启用不加入域的连接，你必须登录的组成员的本地管理员帐户的计算机。  
  
-   将计算机连接到 Windows Server Essentials 服务器，你将需要以下帐户信息：  
  
    -   使用管理员权限的 Windows Server Essentials（你的帐户）的帐户。  
  
    -   用户名和密码的人员姓名将使用该计算机的域帐户。 域帐户还必须管理员权限的 Windows Server Essentials 服务器上。  
  
#### <a name="connect-to-a-windows-server-essentials-network"></a>对 Windows Server Essentials 网络连接  
 在验证所有先决条件均已都满足后，将计算机连接到 Windows Server Essentials 网络。  
  
###### <a name="to-connect-a-computer-in-a-different-domain-to-a-windows-server-essentials-network"></a>将不同的域中的计算机连接到 Windows Server Essentials 网络  
  
1.  登录到的组成员的本地管理员帐户客户端计算机。  
  
2.  打开具有管理员权限的 command prompt。  
  
    -   Windows 10 中，单击**开始**按钮、依次选择**所有应用** -> **Windows 系统工具** -> **权限的命令提示符**，命令提示符下，右键单击，然后单击**以管理员身份运行**。  
  
    -   在 Windows 8 中，在**开始**页上，键入**命令**，然后按 Enter。 在结果中，右键单击**权限的命令提示符**，然后单击**以管理员身份运行**。  
  
    -   在 Windows 7 上**开始**菜单上，输入**命令**在搜索框中，右键单击**权限的命令提示符**，然后单击**以管理员身份运行**。  
  
3.  在命令提示符下，键入以下命令，，然后按 Enter:  
  
    ```  
    reg add "HKLM\SOFTWARE\Microsoft\Windows Server\ClientDeployment" /v SkipDomainJoin /t REG_DWORD /d 1  
    ```  
  

4.  完成中的步骤[将计算机连接到服务器](Get-Connected-in-Windows-Server-Essentials.md#BKMK_9)。

###  <a name="BKMK_13"></a>卸载接头软件  
 你可以通过使用控制面板从计算机上卸载接头软件。 你将通常执行此操作是否接头软件问题或者如果你需要安装接头软件较新版本。 您必须登录到计算机以 administrator 身份才能完成此过程。

##### <a name="to-uninstall-connector-software-from-a-computer"></a>若要从一台计算机上卸载接头软件  
  
1.  从计算机运行的 Windows 7、Windows 8、Windows 8.1 或 Windows 10 中，打开**控制面板**，然后在**程序**部分中，单击**查看已安装的更新**。  
  
2.  已安装程序的列表中选择**Windows Server Essentials Connector**，然后单击**卸载**。  
  
3.  在警告页面上，单击**是**。  
  
4.  如果**用户帐户控制**窗口时，请单击**允许**。  
  
5.  在 Windows Server Essentials，如果 Windows Server Essentials Connector 页面显示建议关闭启动栏，请单击**确定**。  
  
6.  等待要卸载的程序。 在删除该软件后，**Windows Server Essentials Connector**不会再显示在已安装的程序或更新的列表。 此外，启动栏和仪表板快捷方式不会再计算机 s 桌面上显示。  
  
> [!NOTE]
>  -   卸载接头软件不会从列表中显示的计算机中删除该计算机**设备**仪表板中的选项卡。 若要从仪表板中删除的计算机，请参阅[从服务器中删除一台计算机](../manage/Manage-Devices-in-Windows-Server-Essentials.md#BKMK_3)。  
> -   卸载接头软件的客户端计算机上已映射到服务器的共享的文件夹不会删除。 你必须手动删除映射到服务器的共享的文件夹。  

> -   卸载接头软件不会使计算机脱离原始域。 你必须手动脱离域中的计算机。 有关说明，请参阅[从 Windows 域删除一台计算机](Get-Connected-in-Windows-Server-Essentials.md#BKMK_8)。

3.  使用仪表板中服务器删除计算机。 有关的分步说明，请参阅[从服务器中删除一台计算机](../manage/Manage-Devices-in-Windows-Server-Essentials.md#BKMK_3)。  
  
 若要重新连接到服务器的之前已与你的 Windows Server Essentials 服务器网络断开连接的计算机，你必须完成以下步骤：  
  

1.  通过使用控制面板从计算机中卸载接头软件。 有关的分步说明，请参阅[卸载接头软件](Get-Connected-in-Windows-Server-Essentials.md#BKMK_13)。  
  
2.  脱离将计算机从 Windows Server Essentials 域和工作组到加入该。 加入 Windows 工作组中，对于的分步说明[加入或创建工作组](https://windows.microsoft.com/windows7/Join-or-create-a-workgroup)。  
  
3.  使用连接计算机向导将计算机连接到服务器。 有关的分步说明，请参阅[将计算机连接到服务器](Get-Connected-in-Windows-Server-Essentials.md#BKMK_9)。  
  
###  <a name="BKMK_Sleep"></a>如何备份适用于睡眠和休眠模式  
 如果你选择**唤醒此计算机是否存在备份**选项，当你将计算机连接到服务器，在计算机会自动从睡眠状态唤醒或休眠模式，以便它可以备份作为备份计划中指定每天。 备份完成后，在计算机将返回到进入睡眠状态或休眠模式，根据其电源管理的设置。 如果您不选择此选项，服务器不会备份计算机如果计算机处于睡眠或休眠状态。 有关详细信息，请参阅[管理客户端备份](../manage/Manage-Client-Computer-Backup-in-Windows-Server-Essentials.md)。
