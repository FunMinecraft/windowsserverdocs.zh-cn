##  <a name="BKMK_1"></a>修复备份数据库向导中的工作原理  
 Windows Server Essentials 检测到你的备份数据库中的错误，如果它向您发送健康通知，并警报的状态更改为红色，表示关键状态。  
  
 在 Windows Server Essentials 仪表板中，单击以查看备份数据库错误通知的通知状态图标。 通知包括**修复**按钮、开始修复备份数据库向导。 该向导，可能需要几个小时才能完成。  
  
 该向导检查所选计算机你备份的数据库，并尝试如果未检测到错误修复数据库。 在某些情况下，该向导将无法修复整个备份数据库。 你可以启动该向导之前，请验证所有外部硬盘驱动器，则您使用服务器上，连接到服务器，在打开，并且正在正常运行。 会重新连接缺少硬盘自动修复备份数据库错误。  
  
> [!CAUTION]
>  你应该之前，备份数据库尝试修复它。 找到备份数据库中的错误类型，根据该向导不可能能够恢复一些备份。 部分或全部备份将会永久消失。  
  
##  <a name="BKMK_2"></a>了解计算机备份设置  
 备份配置的客户端计算机后，你可以指定其他窗口的进行备份的时间。 同样，你可以指定而不是默认长或更短的备份保留时间。  
  
 **还原到默认值**选项允许你备份的窗口和保留策略重置为默认值提供初始备份配置过程。  
  
 是默认值：  
  
|备份设置|默认设置|描述|  
|--------------------|-------------|-----------------|  
|开始时间|6:00|指定日常备份启动的时间。 建议你将此设置为用户未使用他们的计算机时的时间。|  
|结束时间|上午 9:00 到|指定备份必须先完成的时间。 如果备份不需要指定的持续时间，它将使用仅成功备份计算机所需的时间。|  
|保留每天备份|5 天|指定的数天，将保留每日备份。 例如，使用默认设置，保留 5 每日备份。 6 日每一天之后，在古老的每日备份文件被删除。|  
|保留每周备份|4 周|指定的数周保留在一周的最后一个备份。 例如，使用默认设置，请保留 4 每周备份。 在第五个星期和此后每周，旧的每周备份也将被删除。|  
|保留每月的备份|6 个月|指定保留最后一次备份每个月的几个月数。 例如，使用默认设置，保留 6 个月的备份。 在 7 月和每月此后，将删除旧的每月备份。|  
  
##  <a name="BKMK_3"></a>设置的备份的客户端计算机  
 如果已禁用备份，你可以从仪表板设置计算机的备份。 当您设置了一台计算机的备份时，你可以选择备份到计算机的所有内容，或选择的卷和你想要备份的文件夹。  
  
> [!NOTE]
>  该计算机必须为你设置的备份在线。  
  
> [!IMPORTANT]
>   Windows Server 备份和还原客户端计算机上的动态磁盘 essentials 不支持。  
  
#### <a name="to-set-up-backup-for-a-client-computer"></a>若要设置备份的客户端计算机  
  
1.  打开**仪表板**，然后单击**设备**选项卡。  
  
2.  单击你想要设置备份，然后在客户端计算机名称**任务**窗格中，单击**备份设置此计算机**。  
  
    > [!NOTE]
    >  备份已设置为客户端计算机上，如果**备份自定义此计算机**中列出**任务**窗格中，而不是**备份设置此计算机**。  
  
3.  在设置备份向导，你可以选择备份的所有文件夹，或者选择你想要备份特定文件夹。 按照该向导中的说明进行操作。  
  
4.  单击**关闭**备份设置时计算机。  
  
### <a name="critical-system-files"></a>重要的系统文件  
 当你安装了 Windows 操作系统时，安装程序会系统驱动器放置系统启动和运行所需的文件上创建文件夹。 重要的系统文件包含与.dll、.exe、.ocx，并具有.sys 扩展名文件。 其中某些文件是如此类型字体。 此外，系统状态文件，如系统的注册表中，都需要能够正常运行的操作系统。  
  
### <a name="find-the-file-you-are-looking-for"></a>查找所需的文件  
 你可以从现有备份还原计算机、多个文件和文件夹，或者单个文件或文件夹的所有文件夹。  
  
 选择要用来还原从备份后，在阅读备份文件，并显示的所有文件和文件夹。 你可以深入特定文件或文件夹还原通过双击的顶层文件夹，然后向下钻取通过文件夹层次直到找到要查找的文件。  
  
### <a name="why-am-i-unable-to-select-some-items"></a>为什么我无法选择的某些项？  
 上的选项菜单的复选框**选择的项目备份页面**可以指示不同的每个文件夹的状态。 当未该复选框：  
  
-   **所选**，关联的文件夹的文件夹内容选择要备份。  
  
-   **取消选中**，关联的文件夹和文件夹的内容不包括在备份。  
  
-   **稳定**、为备份，选择关联的文件夹，但该文件夹中的一个或多项不包括在备份。  
  
 如果你无法选择特定的文件夹：  
  
-   音量可以配置为备份，但它可能处于离线状态。 这很常见可移动 USB 驱动器。 离线状态下的卷灰色文本所示。  
  
-   你仅可以为 NTFS 文件系统进行格式化本地驱动器从备份数据。 因为不显示在列表中的驱动器来备份（包括 FAT32）FAT 或 ReFS 文件系统格式化驱动器。  
  
> [!IMPORTANT]
>  卷影副本服务 (VSS) 不支持在相同的快照集中创建的卷影副本虚拟音量和主音量。 备份的虚拟音量是否需要 VSS 器 (VHD)，虚拟硬盘上支持创建的卷的快照。 有关详细信息，请参阅[维护服务和支持向上虚拟硬盘](https://go.microsoft.com/fwlink/p/?LinkId=256577)。  
  
##  <a name="BKMK_4"></a>更改时间备份计划的预定运行  
 应少人脉尽可能使用其联网的计算机时期间安排备份的过程。 这是通常期间深夜或早晨。 备份的默认时间是 6:00 上午 9:00 直到。 尝试仅在计划的时间窗口备份客户端计算机服务器。  
  
 但是，如果你的企业期间关闭小时传统这些处于活动状态，你可能想要更改这些默认设置。  
  
#### <a name="to-change-the-time-that-backup-is-scheduled-to-run"></a>若要更改的时间该备份计划运行  
  
1.  打开服务器**仪表板**，然后单击**设备**。  
  
2.  在**设备任务**，单击**自定义计算机备份和文件历史记录设置**。  
  
    > [!NOTE]
    >  已在 Windows Server Essentials 重命名此任务**客户端计算机备份任务**。  
  
3.  在**客户端计算机备份设置和工具**，在**计算机备份**选项卡上，你可以更改开始和结束时间，以满足你的需求。  
  
4.  单击**应用**，然后单击**确定**。  
  
##  <a name="BKMK_5"></a>更改计算机备份保持策略  
 每日备份的所有计算机随着时间的推移累积服务器上。 为了帮助你管理这些备份，Windows Server Essentials 可以帮助你管理计算机备份数据库。 您可以配置多少备份要保留的所有你的计算机。  
  
 备份保持策略确定备份清理过程中删除之前备份保留的多长时间。 备份清理会在每个周六晚上 11:59 运行。 它会删除所有超出备份保持策略的备份。 是备份保持策略默认值：  
  
-   **5 天保留每日备份。** 每日备份作为保留一天中的第一次备份。 5 天后，旧的每日备份清理进程中删除。  
  
-   **保留为 4 个星期的每周备份。** 一周中的第一次备份保留为每周一次备份。 后 4 周中，旧的每周备份将被删除清理进程中。  
  
-   **6 个月内保留每月的备份。** 每个月的第一次备份作为每月备份保留。 在 6 个月后古老的每月备份删除清理进程中  
  
#### <a name="to-change-the-computer-backup-retention-policy"></a>若要更改计算机备份保持策略  
  
1.  打开**仪表板**。  
  
2.  单击**设备**。  
  
3.  在**设备任务**窗格中，单击**自定义计算机备份和文件历史记录设置**。  
  
    > [!NOTE]
    >  已在 Windows Server Essentials 重命名此任务**客户端计算机备份任务**。  
  
4.  在**客户端计算机备份设置和工具**中**客户端计算机备份保持策略**部分中，对符合你的需求保留策略进行更改。  
  
5.  完成后，单击**确定**应用所做的更改并关闭对话框。 将应用的更新的保留策略备份清理运行下一次。  
  
    > [!NOTE]
    >  更新的保留策略适用于所有配置为进行备份你的网络上的客户端计算机。  
  
##  <a name="BKMK_6"></a>重置为默认设置的备份  
 备份配置的客户端计算机后，网络管理员可以指定其他窗口的时间。 同样，可能已经管理员指定而不是默认长或更短的备份保留时间。 **重置为默认值**按钮使你备份的窗口和保留策略重置为默认值提供初始备份配置过程。  
  
 是默认值：  
  
-   备份开始时间：6:00  
  
-   结束时间：上午 9:00 到  
  
-   将保留每日备份的天数：5 天  
  
-   数周保留在一周的最后一个备份：4 周  
  
-   保留最后一次备份每个月的几个月的数量：6 个月  
  
#### <a name="to-reset-computer-backup-to-the-default-settings"></a>若要将计算机备份重置为默认设置  
  
1.  打开**仪表板**，然后打开**设备**页面。  
  
2.  在**设备任务**，单击**自定义计算机备份和文件历史记录设置**。  
  
    > [!NOTE]
    >  在 Windows Server Essentials、单击**客户端计算机备份任务**。  
  
3.  在**计算机备份**的选项卡**客户端计算机和备份设置和工具**页上，单击**重置为默认值**。  
  
    > [!NOTE]
    >  你可能想要写下自定义的计划并保留策略设置，因为它们将在时将重置为默认设置消失。  
  
4.  单击**应用**，然后单击**确定**。  
  
##  <a name="BKMK_7"></a>使用修复和恢复工具  
 **修复备份：**计算机备份数据库变为已损坏或可用度出于某种原因，你可以尝试使用修复备份数据库向导修复数据库。 该向导分析备份的文件，以确定是否有任何可以修复的问题。 然后尝试修复发现任何问题向导。  
  
> [!WARNING]
>  如果不能解决问题，该向导可能会永久删除计算机备份文件。  
  
 **计算机恢复：**可以创建可启动 U 盘上用来从现有备份还原计算机。 你必须使用 U 盘上的 1 GB 或更大。 创建可启动 U 盘后，你将其插入你想要还原，然后启动到 U 盘启动全系统恢复过程的计算机。  
  
##  <a name="BKMK_8"></a>修复备份的数据库  
 如果你收到提醒告诉你的计算机备份数据库有问题，你可以尝试修复它。  
  
#### <a name="to-repair-the-backup-database"></a>若要修复备份的数据库  
  
1.  打开**仪表板**。  
  
2.  单击**设备**。  
  
3.  在**设备任务**窗格中，单击**自定义计算机备份和文件历史记录设置。**  
  
    > [!NOTE]
    >  在 Windows Server Essentials、单击**客户端计算机备份任务**。  
  
4.  在**客户端计算机备份设置和工具**，单击**工具**选项卡。  
  
5.  在**修复备份**部分中，单击**现在修复**。 打开备份数据库向导中的修复。  
  
6.  按照修复备份数据库向导中的说明进行操作。  
  
7.  根据如何大备份数据库多少，数据库修理可能需要几个小时。 单击**关闭**，然后单击**确定**关闭**自定义计算机备份和文件历史记录设置**页面。  
  
    > [!NOTE]
    >  在 Windows Server Essentials、单击**客户端计算机备份设置和工具**。  
  
#### <a name="to-review-the-results-of-the-backup-database-repair"></a>若要复查所备份数据库修复结果  
  
1.  打开**仪表板**。  
  
2.  单击**设备**。  
  
3.  在**设备任务**窗格中，单击**自定义计算机备份和文件历史记录设置。**  
  
    > [!NOTE]
    >  在 Windows Server Essentials、单击**客户端计算机备份任务**。  
  
4.  在**客户端计算机备份设置和工具**，单击**工具**选项卡。  
  
5.  在显示结果**修复备份**部分。  
  
##  <a name="BKMK_9"></a>禁用计算机的备份  
 使用仪表板快速禁用网络上计算机的备份。  
  
#### <a name="to-disable-backup-for-a-computer"></a>若要禁用的计算机的备份  
  
1.  打开**仪表板**。  
  
2.  单击**设备**选项卡。  
  
3.  单击你想要禁用备份的计算机名称。  
  
4.  在任务窗格中，单击**计算机自定义备份**。 显示自定义备份向导。  
  
5.  单击**禁用备份，这台计算机**，然后选择是否想要保留或删除现有备份的文件。  
  
6.  单击**保存更改**，然后单击**关闭**。  
  
 了解如何启用备份的计算机，已禁用备份后，请参阅[设置的备份的客户端计算机](Manage-Client-Computer-Backup-in-Windows-Server-Essentials.md#BKMK_3)。  
  
##  <a name="BKMK_10"></a>运行备份清理任务  
 客户端计算机备份清理任务计划运行晚上 11:59 星期六后备份的所有都已完成。 清理任务从备份保持策略根据的客户端计算机备份数据库中删除备份。 是默认设置的备份保持策略：  
  
-   将保留每日备份的天数：5 天  
  
-   数周保留在一周的最后一个备份：4 周  
  
-   保留最后一次备份每个月的几个月的数量：6 个月  
  
 有关更改备份保持策略的信息，请参阅[更改计算机备份保持策略](Manage-Client-Computer-Backup-in-Windows-Server-Essentials.md#BKMK_5)。  
  
#### <a name="to-run-the-client-backup-database-cleanup-task"></a>若要运行备份数据库清理任务的客户端  
  
1.  登录到你的管理员用户帐户和密码与服务器上。  
  
2.  在服务器上，打开**任务计划程序**。 你可以访问来自任务计划程序**管理工具**主机。  
  
3.  在任务计划程序中，展开**任务计划程序库**，展开**Microsoft**，展开**Windows**，然后单击**Windows Server Essentials**。  
  
4.  单击**备份清理**任务，，然后单击**运行**中**操作**窗格。 状态更改为**运行**直到完成任务。  
  
##  <a name="BKMK_11"></a>在任务栏中的客户端计算机上查看通知  
 你可以接收备份通知任务栏中计算机上出于以下原因：  
  
-   备份启动。  
  
-   已结束备份。  
  
-   不是最新的成功备份。 上次成功备份天数包含在通知。  
  
-    Windows Server 仅 essentials：新的音量添加到你的计算机。 管理你的网络的人员需要运行自定义备份向导包括或排除将来备份的音量。  
  
##  <a name="BKMK_12"></a>查看备份状态，从启动栏  
 使用启动栏以查看快速状态备份你的计算机。  
  
> [!TIP]
>  快速状态中，将光标移于桌面版的通知区域中的启动栏图标。 在工具提示显示备份状态。  
  
#### <a name="to-view-status-of-a-backup-for-your-computer"></a>若要查看备份你的计算机的状态  
  
1.  登录到**启动栏**与你的用户名和密码。  
  
2.  单击**备份**。  
  
3.  在**备份属性**对话框中，在**备份状态**会显示部分备份的状态。  
  
    -   **备份不以前**：既备份尚未运行，或者已备份的历史记录中删除。  
  
    -   **待处理的备份**：备份等待其他进程在服务器上才能完成。  
  
    -   **连接**：备份连接到服务器。  
  
    -   **无法连接**：备份无法连接到 Windows Server 客户端计算机备份提供程序服务。  
  
        > [!NOTE]
        >  在 Windows Server Essentials 已被此服务重命名 Windows Server Essentials 客户端计算机管理服务。  
  
    -   **在进行备份**：显示已完成备份的百分比。  
  
    -   **备份而被取消**：显示日期和时间，如果你或网络管理员已取消备份启动备份。  
  
    -   **备份成功**：显示日期和时间，如果备份成功完成启动备份。  
  
    -   **备份不完整**：显示日期和时间，如果未成功完成备份启动备份。  
  
    -   **备份不成功**：显示日期和时间，如果未成功完成备份启动备份。  
  
4.  单击**确定**关闭**备份属性**对话框。  
  
##  <a name="BKMK_13"></a>停止在启动栏中进行备份  
 你可以轻松地停止正在进行的备份。  
  
#### <a name="to-stop-a-backup-in-progress"></a>若要停止在进行备份  
  
1.  打开**启动栏**。  
  
    > [!NOTE]
    >  如果系统要求，登录到**启动栏**与你的用户名和密码。  
  
2.  单击**备份**。  
  
3.  在**备份属性**对话框中，在**备份状态**部分中，**开始备份**按钮将更改为**停止备份**时正在进行备份。 单击**停止备份**，然后单击**是**以确认。 备份将继续运行，直到你单击**是**。  
  
4.  状态时停止在进行备份后，会自动变为**备份而被取消**与日期和启动备份的时间。 如果相反，你看到的状态**成功**，**完整**，或**失败**，已经结束最后一次备份。  
  
5.  单击**确定**关闭**备份属性**对话框。  
  
##  <a name="BKMK_14"></a>启动栏中开始备份  
 这可能是当你想要重新安装你的文件和文件夹之前定期备份时间设置服务器上的时间。 启动栏使你可以手动启动备份你的计算机。  
  
#### <a name="to-start-a-backup"></a>若要开始备份  
  
1.  打开**启动栏**。  
  
    > [!NOTE]
    >  如果系统要求，登录到**启动栏**与你的用户名和密码。  
  
2.  单击**备份**。  
  
3.  在**备份属性**对话框中，在**备份状态**部分中，单击**开始备份**，然后单击**确定**。  
  
4.  键入的备份，描述，然后单击**确定**。 状态更改为**开始备份**，以及然后**进程中备份**完整的百分比。  
  
5.  后备份开始，该按钮将变为**停止备份**。 如果正在进行备份，并且你想要阻止它，请单击**停止备份**，然后单击**是**。 状态时停止在进行备份后，会自动变为**备份而被取消**与日期和启动备份时的时间。  
  
6.  备份成功完成后，当的状态更改为**备份成功**与的日期和时间开始完成的备份。  
  
7.  单击**确定**关闭**备份属性**对话框。  
  
##  <a name="BKMK_15"></a>计算机备份的工作原理  
 以下情况备份期间每一天：  
  
-   备份网络计算机向上逐个。  
  
-   正在进行备份完成后，即使您关闭备份的窗口。  
  
-   安装 Windows 更新以及 Windows Server Essentials 重启（如果需要）。  
  
-   星期日，备份清理运行。  
  
> [!IMPORTANT]
>  卷影副本服务 (VSS) 不支持在相同的快照集中创建的卷影副本虚拟音量和主音量。 备份的虚拟音量是否需要 VSS 器 (VHD)，虚拟硬盘上支持创建的卷的快照。 有关详细信息，请参阅[维护服务和支持向上虚拟硬盘](https://go.microsoft.com/fwlink/p/?LinkId=256577)。  
  
##  <a name="BKMK_16"></a>若要防止数据丢失的客户端备份数据库损坏由于的使用技巧  
 如果客户端备份数据库损坏，你可能失去紧急数据。  
  
 若要防止数据丢失损坏的客户端备份数据库由于，请考虑以下：  
  
-   启用备份客户端备份数据库其他付款方式。 例如，具体取决于你运行的 Windows Server Essentials 的版本，使用服务器备份、Online Backup 或 Microsoft Azure 备份来备份数据库。  
  
    > [!IMPORTANT]
    >  请确保你选择要备份中的所有文件**客户端计算机备份**文件夹。  
  
-   在你必须还原数据库中，请务必还原中的所有文件**客户端计算机备份**文件夹。 如果你无法还原的所有文件，的数据库可能变得不一致。  
  
-   在清理或还原客户端备份数据库之前，请确保停止当前正在进行的所有客户端备份。  
  
     如果你运行的 Windows Server Essentials，则应停止 Windows Server 客户端计算机备份服务和 Windows Server 客户端计算机备份提供程序服务。  
  
     如果你运行的 Windows Server Essentials，你还应停止 Windows Server 计算机备份服务。  
  
     在完成恢复操作后，请后重启该服务。

##  <a name="BKMK_FileHistory"></a>了解文件历史记录  
 Windows Server Essentials 的文件历史记录功能自动备份文件历史记录功能的网络计算机的库、联系人、桌面和收藏夹文件夹中的文件。 如果原始文件丢失、损坏或被删除，你可以还原它们。 你还可以找到特定点文件的不同版本的时间。 随着时间的推移，你将有你的文件的完整历史记录。  
  
 Windows Server Essentials 中，你可以自定义中的文件历史记录设置**文件历史记录**页**客户端计算机备份设置和工具**。  
  
 在 Windows Server Essentials，你可以自定义文件历史记录设置方法是转到**用户**选项卡，然后依次选择**更改文件历史记录设置**任务。  
  
 文件历史记录页面将提供以下选项：  
  
|备份设置|默认设置|描述|  
|--------------------|-------------|-----------------|  
|打开/关闭|在|文件历史记录处于打开状态默认情况下，当你安装了 Windows Server Essentials。|  
|备份数据|文档和桌面|有三个预配置的设置，这允许你指定的各种备份的解决方案。 你可以选择以下选项之一：<br /><br /> -文档和桌面<br /><br /> -所有库、桌面、联系人和收藏夹<br /><br /> 的库、桌面、联系人和除外数据音乐、视频和图片库中的收藏夹中所有数据|  
|备份频率|每小时|指定文件历史记录将所选的数据的备份创建的频率。 你可以从多个选项选择此范围是从每 10 分钟为每日。|  
|保留为副本|1 年|指定的文件历史记录保存的备份副本的时间。|  
  
 文件历史记录的问题有关的信息，请参阅[疑难解答文件历史记录](../support/Troubleshoot-File-History-in-Windows-Server-Essentials.md)。
