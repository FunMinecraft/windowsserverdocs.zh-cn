## <a name="guidance-on-disabling-system-services-on-windows-server-2016-with-desktop-experience"></a>禁用系统桌面体验与 Windows Server 2016 服务指南

适用于： Windows Server 2016

Windows 操作系统包括许多系统服务提供重要功能。 不同服务具有不同的默认启动策略： 一些启动默认情况下 （自动） 一些在需要时 （手动） 和一些默认处于禁用状态，并且之前，必须明确启用它们可以运行。 这些默认值为每个服务平衡性能、 功能和安全性的典型客户可以选择仔细。

但是，一些企业客户可能为它们的 Windows 电脑和一个，减少了他们攻击 surface 连接到绝对最小，因此可能想要完全禁用不需要在其特定的环境中所有服务的服务器喜欢更安全关注的余额。 对于那些客户，Microsoft® 将提供关于该服务可以安全地禁用实现此目的随附指南。

指南 （除非为最终用户使用桌面替换为） 是 Windows Server 2016 的桌面体验。 每个服务系统上的进行分类，如下所示：

-   **应禁用：**专注安全于企业最有可能会想禁用该服务并放弃其功能 （请参阅下方的其他详细信息）。
- **禁用确定：**该服务提供的一些但并非所有企业非常有用的功能和不要使用它的安全重点企业安全禁用它。
- **不要禁用：**禁用该服务将影响必要的功能或阻止特定的角色或功能正常工作。 因此它不应禁用。
-  **（无指南）：**不完全计算禁用这些服务的影响。 因此，不应更改默认配置这些服务。


客户可以配置他们的 Windows 电脑和服务器来禁用安全模板在其组策略使用或 PowerShell 自动化所选的服务。 在某些情况下，指南中包含禁用该服务本身替代直接，禁用服务的功能的特定组策略设置。

Microsoft 建议客户禁用以下服务和桌面体验与 Windows Server 2016 上的各自计划的任务：

服务： 
1. Xbox Live Auth 管理器
2. Xbox Live 的游戏的保存

计划的任务： 
1. \Microsoft\XblGameSave\XblGameSaveTask
2. \Microsoft\XblGameSave\XblGameSaveTaskLogon

(你还可以访问在通过查看附加的 Microsoft Excel 电子表格详细本文中所有服务的信息：[的禁用系统桌面体验与 Windows Server 2016 服务指南](https://msdnshared.blob.core.windows.net/media/2017/05/Service-management-WS2016.xlsx))

<br />

### <a name="disabling-services-not-installed-by-default"></a>禁用显示器的服务不默认情况下安装

Microsoft 建议不要应用策略禁用不默认情况下安装的服务。
-  如果已安装了功能，通常需要服务。 安装该服务或功能，需要管理权限。 不允许功能安装，而不服务启动。
-  Microsoft Windows 的服务的阻止不会停止管理员 （或在某些情况下非管理员） 从安装类似的第三方等效，可能是一个更高版本的安全风险。
-  基准或禁用非默认 Windows 服务 (例如，W3SVC) 的基准将提供一些审核错误的印象技术 (例如，IIS) 本身不安全，并应永远不会使用。
-  如果从未已安装的功能和服务），这只需添加不必要的批量基准到和验证工作。
