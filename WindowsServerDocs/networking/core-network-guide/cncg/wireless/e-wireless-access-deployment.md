### <a name="bkmk_wirelessaps"></a>配置无线接入点

使用以下信息以及无线接入点的制造商提供的产品文档来配置你的无线接入点。

此过程枚举通常配置无线接入点上的项目。 项目名称因品牌和型号，并且可能不同于下面的列表。 有关特定的详细信息，请参阅你无线接入点的文档。

#### <a name="to-configure-your-wireless-aps"></a>若要配置你的无线接入点  

- **SSID**。 指定的名称无线 network\(s\) \ (例如，ExampleWLAN\)。 这是公布到无线客户端的名称。

- **加密**。 指定 WPA2\ 企业 \(preferred\) 或 WPA\-企业版和 AES \(preferred\) 或 TKIP 加密密钥，具体取决于这版本受你无线客户端计算机的网络适配器。

- **无线接入点的 IP 地址 \(static\)**。 在每次接入点，配置在子网 DHCP 范围的排除项的范围内一个唯一的静态 IP 地址。 使用排除从分配了 DHCP 地址阻止 DHCP 服务器到计算机或其他设备中指定的同一 IP 地址。

- **子网掩码**。 配置此匹配的子网掩码 lan 之外，你已连接的无线接入点设置。  

- **DNS 名称**。 可以与 DNS 名称配置某些无线接入点。 DNS 服务网络上的可以解决 DNS 名称 IP 地址。 在每个无线接入点支持此功能，输入一个唯一的名称 DNS 分辨率。  

- **DHCP 服务**。 如果你的无线接入点都有 built\ 中 DHCP 服务，请将其禁用。  

- **RADIUS 共享的密钥**。 使用唯一 RADIUS 幽静分享的每个无线接入点，除非你正计划配置为在按组 NPS RADIUS 客户端的接入点。 如果你计划将通过组接入点配置 NPS 中，共享的幽静必须为每个成员组的相同。 此外，你使用的每个共享的机密应该随机一连串至少 22 混合大写形式的字符和小写字母、数字和标点符号。 若要确保随机，你可以使用随机字符 generator，如在 NPS 找到随机字符 generator**配置 802.1 X**向导中，以创建共享机密信息。

>[!TIP]
>录制共享的机密的每个无线接入点，并将其存储在安全位置，如 office 安全。 配置中 NPS RADIUS 客户端时，你必须为每个无线接入点知道共享的机密。  

- **RADIUS 服务器 IP 地址**。 键入运行 NPS 服务器的 IP 地址。

- **UDP port\(s\)**。 默认情况下，NPS 身份验证邮件和 UDP 端口 1813 年和记帐消息的 1646 年使用 1812 年和 1645 UDP 端口。 建议你在你的接入点上使用这些相同的 UDP 端口，但如果你有有效的原因，若要使用不同的端口，确保你为接入点配置新端口号不仅，然后再重新所有 NPS 服务器使用相同的端口号作为接入点。 如果未使用的相同 UDP 端口配置接入点和 NPS 服务器，NPS 无法接收或处理连接请求从接入点，并将无法在网络上的所有无线连接尝试。

- **Vsa**。 一些无线接入点需要 vendor\ 特定于属性 \(VSAs\) 提供完全的无线接入点的功能。 Vsa NPS 网络策略中添加。

- **DHCP 筛选**。 将阻止无线客户端将 IP 数据包从 UDP 端口 68 发送到该网络，述无线接入点制造商的无线接入点的配置。

- **DNS 筛选**。 将阻止无线客户端将 IP 数据包从 TCP 或 UDP 端口延长 53 发送到该网络，述无线接入点制造商的无线接入点的配置。

6. 如果默认文本**NEWSSID**是出示、选择它，然后单击**删除**。

7. 如果你部署配置取消广播的信号，请选择的无线接入点**即使网络未进行广播也连接**。

    > [!NOTE]
    > 启用该选项可能产生安全隐患因为无线客户端将探测和尝试连接到任何无线网络。 默认情况下，未启用此设置。  

8. 单击**安全**选项卡上，单击**高级**，然后配置以下：

    1. 若要配置高级 802.1 X 的设置，在**IEEE 802.1 X**、选择**强制高级 802.1 X 设置**。

        当高级的 802.1 X 设置实施的默认值为**最大 Eapol\ 开始消息数**，**保持时间**，**启动期间**，和**验证时间**有足够的典型无线部署。 出于此原因，你不需要更改默认值，除非你已执行此操作的特定的原因。

    2. 若要启用单一登录，请选择**该网络启用单一登录**。

    3. 其余的默认值**单一登录**有足够的典型无线部署。

    4. 在**快速漫游**，如果你的无线接入点配置为 pre\ 认证，选择**此网络使用 pre\ 身份验证**。

9. 若要指定无线通信满足 FIPS 140\ 2 标准，请选择**型 FIPS 140\ 2 认证执行加密**。

10. 单击**确定**返回到**安全**选项卡。在**选择该网络的安全措施**中**身份验证**、选择**WPA2\ 企业**如果你的无线接入点和客户端的无线网络适配器不支持。 否则，请选择**WPA\ 企业**。

11. 在**加密**，如果你的无线接入点和客户端的无线网络适配器，选择支持**AES-CCMP**。 如果你使用的接入点和支持 802.11ac 的无线网络适配器，请选择**AES GCMP**。 否则，请选择**TKIP**。

    > [!NOTE]  
    > 两个设置**身份验证**和**加密**必须匹配你的无线接入点配置的设置。 默认设置为**身份验证模式**，**最大的身份验证失败**，并**缓存后续连接到该网络的用户信息**有足够的典型无线部署。  

12. 在**选择网络身份验证方法**、选择**保护 EAP \(PEAP\)**，然后单击**属性**。 **保护 EAP 属性**对话框中打开。

13. 在**保护 EAP 属性**，确认**验证服务器的身份验证的证书**选择。
