### <a name="bkmk_channel"></a>指定无线 AP 通道频率

当部署多个无线 Ap 位于单个地理站点时，必须配置有重叠的信号，使用唯一的通道频率来减少干扰无线访问点之间的无线访问点。

可以使用以下指导原则可帮助您选择通道不会与您的无线网络的地理位置的其他无线网络冲突的频率。

- 如果没有其他的办事处中的组织非常接近的或在同一栋作为您的组织，识别是否存在这些组织拥有的任何无线网络。 找出的位置和分配的通道频率其无线 AP 的因为需要将不同的通道频率分配给您的 AP，需要确定的最佳位置安装您的 AP

- 确定重叠上您自己的组织内的相邻楼层无线信号。 标识重叠的覆盖区域外部的组织中分配的无线 Ap 的通道频率后，确保任何两个具有重叠范围的无线 Ap 分配不同的通道的频率。

### <a name="bkmk_wirelessaps"></a>配置无线 Ap

使用以下信息以及无线 AP 制造商提供的产品文档来配置无线 Ap。

此过程通常在无线 AP 上配置的项进行枚举。 项名称按品牌和型号而异，因此可能不同于与以下列表中。 特定详细信息，请参阅无线 AP 文档。

#### <a name="to-configure-your-wireless-aps"></a>若要配置无线 Ap  

- **SSID**。 指定无线网络的名称\(s\) \(等 ExampleWLAN\)。 这是播发到无线客户端的名称。

- **加密**。 指定 WPA2\-企业\(首选\)或 WPA\-企业版和任一 AES\(首选\)或 TKIP 加密密码，具体取决于该版本支持的应用无线客户端计算机网络适配器。

- **无线接入点 IP 地址\(静态\)**。 每个 AP 上配置唯一的静态 IP 地址子网的 DHCP 作用域排除范围内。 使用 DHCP 分配中排除的地址可阻止 DHCP 服务器将相同的 IP 地址分配给计算机或其他设备。

- **子网掩码**。 配置无线 AP 具有连接到 LAN 的子网掩码设置相匹配。  

- **DNS 名称**。 可以将一些无线 Ap 配置具有 DNS 名称。 在网络上的 DNS 服务可以解析为 IP 地址的 DNS 名称。 上支持此功能，每个无线 AP，输入 DNS 解析的唯一名称。  

- **DHCP 服务**。 如果在无线 AP 具有内置\-DHCP 服务中禁用它。  

- **RADIUS 共享的机密**。 唯一的 RADIUS 共享的机密用于每个无线 AP，除非你打算将 Ap 配置为 RADIUS 客户端在 NPS 中的组。 如果你打算在 NPS 中配置按组 Ap，共享的机密必须相同的组的每个成员。 此外，您使用的每个共享的机密应至少 22 混合使用大写字符和小写字母、 数字和标点符号的随机序列。 若要确保随机性，可以使用随机字符生成器，如在 NPS 中找到的随机字符生成器**配置 802.1x**向导中，若要创建的共享的机密。

>[!TIP]
>记录每个无线 AP 的共享的机密并将其存储在一个安全位置，例如 office 安全。 当在 NPS 中配置 RADIUS 客户端时，必须知道每个无线 AP 的共享的机密。  

- **RADIUS 服务器 IP 地址**。 键入运行 NPS 的服务器的 IP 地址。

- **UDP 端口\(s\)**。 默认情况下，NPS 使用 UDP 端口 1812年和 1645年用于身份验证消息和 UDP 端口 1813年和 1646 用于记帐消息。 建议您在您的 Ap 上, 使用这些相同的 UDP 端口，但如果您有有效的理由使用不同的端口，确保你不只使用新的端口号配置 Ap，但还重新配置所有使用相同的端口号作为 APs 你 NPSs。 如果未使用的相同 UDP 端口配置 APs 和 NPSs，NPS 无法接收或处理连接请求从 Ap，和在网络上的所有无线连接尝试将失败。

- **Vsa**。 一些无线 Ap 需要供应商\-特定属性\(Vsa\)以提供完整无线 AP 的功能。 在 NPS 网络策略中添加 Vsa。

- **DHCP 筛选**。 配置无线 Ap 以阻止将 IP 数据包从 UDP 端口 68 发送到网络，无线客户端无线 AP 制造商所述。

- **DNS 筛选**。 配置无线 Ap 以阻止将 IP 数据包从 TCP 或 UDP 端口 53 发送到网络，无线客户端无线 AP 制造商所述。

## <a name="create-security-groups-for-wireless-users"></a>为无线用户创建安全组

### <a name="bkmk_groups"></a>创建无线用户安全组

可以使用此过程在 Active Directory 用户和计算机 Microsoft 管理控制台中创建的无线安全组\(MMC\)对齐\-中。  

“Domain Admins”成员身份或同等身份是执行此过程的最低要求。

#### <a name="to-create-a-wireless-users-security-group"></a>若要创建无线用户安全组

1. 依次单击 **“开始”**、**“管理工具”** 和 **“Active Directory 用户和计算机”**。 在 Active Directory 用户和计算机管理单元\-中打开。 单击域的节点（如果尚未选定）。 例如，如果你的域 example.com，请单击**example.com**。

2. 在详细信息窗格中，右键\-单击想要添加新组的文件夹\(例如，右键\-单击**用户**\)，指向**新建**，然后单击**组**。

3. 在 **“新建对象 - 组”** 的 **“组名”** 中，键入新组的名称。 例如，键入**无线组**。

4. 在中**组作用域**，选择以下选项之一：

    - **本地域**

    - **全局**

    - **世界**

5. 在 **“组类型”** 中，选择 **“安全组”**。

6. 单击 **“确定”**。

如果为无线用户需要多个安全组，重复这些步骤来创建其他无线用户和组。 更高版本可以在 NPS 将不同的条件和约束应用到每个组，并为他们提供不同的访问权限和连接规则中创建单独的网络策略。

### <a name="bkmk_addusers"></a> 将用户添加到无线用户安全组

可以使用此过程将用户、 计算机或组添加到 Active Directory 用户和计算机 Microsoft 管理控制台中的无线安全组\(MMC\)对齐\-中。

“Domain Admins”中的成员身份或同等身份是执行此过程的最低要求。

#### <a name="to-add-users-to-the-wireless-security-group"></a>若要将用户添加到的无线安全组

1. 依次单击 **“开始”**、**“管理工具”** 和 **“Active Directory 用户和计算机”**。 将打开“Active Directory 用户和计算机”MMC。 单击域的节点（如果尚未选定）。 例如，如果你的域 example.com，请单击**example.com**。

2. 在详细信息窗格中，双击\-单击包含您的无线安全组的文件夹。

3. 在详细信息窗格中，右键\-无线安全组中，单击，然后单击**属性**。 **属性**安全组对话框随即打开。

4. 上**成员**选项卡上，单击**添加**，然后完成以下过程来添加计算机或添加用户或组之一。

##### <a name="to-add-a-user-or-group"></a>若要添加用户或组

1. 在中**输入要选择的对象名称**，键入用户的名称或组你想要添加，然后单击**确定**。

2. 若要将组成员身份分配给其他用户或组，请重复此过程的步骤 1。  

##### <a name="to-add-a-computer"></a>添加计算机

1. 单击**对象类型**。 **对象类型**对话框随即打开。

2. 在中**对象类型**，选择**计算机**，然后单击**确定**。

3. 在中**输入要选择的对象名称**，键入你想要添加，并单击的计算机的名称**确定**。

4. 若要将组成员身份分配到其他计算机，请重复步骤 1\-3 在此过程。

### <a name="bkmk_opengpme"></a>打开或添加并打开组策略对象

“Domain Admins”成员身份或同等身份是执行此过程的最低要求。

#### <a name="to-open-or-add-and-open-a-group-policy-object"></a>若要打开或添加并打开组策略对象

1. 在域控制器上，单击**启动**，单击**Windows 管理工具**，然后单击**组策略管理**。 此时将打开组策略管理控制台。  

2. 在左窗格中，双击\-单击你的林。 例如，双击\-单击**林： example.com**。  

3. 在左窗格中，双击\-单击**域**，然后双击\-单击你想要管理的组策略对象的域。 例如，双击\-单击**example.com**。  

4. 执行下列操作之一：

    -   **若要打开现有的域\-级别进行编辑的 GPO**，双击包含您想要管理权限的组策略对象的域\-单击你想要管理，默认域策略，如域策略然后单击**编辑**。 **组策略管理编辑器**随即打开。

    -   **若要创建新的组策略对象并打开以进行编辑**，右键\-单击你想要创建新的组策略对象，并单击的域**在此域中创建 GPO 并在此处链接**。

        在中**新的 GPO**，在**名称**，为新的组策略对象中，键入一个名称，然后单击**确定**。

        右\-单击新建的组策略对象，然后依次**编辑**。 **组策略管理编辑器**随即打开。

#### <a name="bkmk_configureprofile"></a>为 PEAP 配置无线连接配置文件\-MS\-CHAP v2

此过程提供了将 PEAP 配置所需的步骤\-MS\-CHAP v2 无线配置文件。  

必须至少具有 **Domain Admins** 中的成员身份或同等身份才能完成此过程。

##### <a name="to-configure-a-wireless-connection-profile-for-peap-ms-chap-v2"></a>为 PEAP 配置无线连接配置文件\-MS\-CHAP v2

1. 在 GPME，刚刚创建，在该策略的无线网络属性对话框中**常规**选项卡并在**说明**，键入策略的简要说明。

2. 若要指定 WLAN 自动配置用于配置无线网络适配器设置，请确保**使用 Windows WLAN 自动配置服务用于客户端**处于选中状态。  

3. 在中**连接到可用网络按照下面列出的配置文件顺序**，单击**添加**，然后选择**基础结构**。 **新的配置文件属性**对话框随即打开。

4. 在中**新的配置文件属性**对话框中，在**连接**选项卡上，在**配置文件名称**字段中，键入配置文件的新名称。 例如，键入**Example.com WLAN 配置文件适用于 Windows 10**。

5. 在中**网络名称\(s\) \(SSID\)**，键入对应于无线 Ap 上 SSID 配置的 SSID，然后单击**添加**。

    如果你的部署使用多个 SSID，并且每个无线 AP 都使用相同的无线安全设置，则重复此步骤来为你希望应用此配置文件的每个无线 AP 添加 SSID。

    如果你的部署使用多个 SSID，并且每个 SSID 的安全设置不匹配，请为使用相同安全设置的每组 SSID 配置单独的配置文件。 例如，如果你有一组无线 Ap 配置为使用 WPA2\-企业和 AES，而另一组无线 Ap 使用 WPA\-企业和 TKIP，为每个组无线 Ap 配置配置文件。

6. 如果默认文本**NEWSSID**是显示中，选择它，然后单击**删除**。

7. 如果你已部署了为取消广播信号而配置的无线访问点，则选择“即使网络未进行广播也连接”。 

    > [!NOTE]
    > 启用此选项可以创建安全风险，因为无线客户端将探测并尝试连接到任何无线网络。 默认情况下未启用此设置。  

8. 依次单击“安全”  选项卡、“高级” ，然后配置以下内容：

    1. 若要配置高级 802.1X 设置，请在“IEEE 802.1X”中选择“强制高级 802.1X 设置”。

        当高级的 802.1x 设置强制的的默认值为**最大 Eapol\-启动消息数**，**保持时间**，**启动时间**，和**验证时间**能够满足典型无线部署。 因此，不需要更改默认设置，除非有特定原因需要执行此操作。

    2. 若要启用单一登录，请选择“为此网络启用单一登录”。 

    3. “单一登录”中的其余默认值能够满足典型的无线部署。 

    4. 在中**快速漫游**，如果前期配置无线 AP\-身份验证中，选择**此网络使用预\-身份验证**。

9. 若要指定无线通信符合 FIPS 140\-2 个标准，选择**FIPS 140 中执行加密\-2 验证的模式下**。

10. 单击“确定”返回到“安全”选项卡。在中**选择此网络的安全方法**，在**身份验证**，选择**WPA2\-企业**如果受无线 AP 和无线客户端网络适配器。 否则，请选择**WPA\-企业**。

11. 在中**加密**，如果受无线 AP 和无线客户端网络适配器，选择**AES CCMP**。 如果在使用访问点和支持 802.11 a c 的无线网络适配器，选择**AES GCMP**。 否则，请选择“TKIP”。 

    > [!NOTE]  
    > 两个设置**身份验证**并**加密**必须与无线 Ap 上配置的设置相匹配。 默认设置**身份验证模式**，**最大身份验证失败次数**，并**缓存用户信息以便随后连接到此网络**是能够满足典型无线部署。  

12. 在中**选择网络身份验证方法**，选择**受保护的 EAP \(PEAP\)**，然后单击**属性**。 **受保护的 EAP 属性**对话框随即打开。

13. 在中**受保护的 EAP 属性**，确认**通过验证证书来验证服务器的标识**处于选中状态。

### <a name="bkmk_npsreg"></a>在 Active Directory 域服务中注册 NPS
您可以使用此过程注册运行网络策略服务器的服务器\(NPS\) Active Directory 域服务中\(AD DS\)其中 NPS 是成员的域中。 有关 NPSs 要对其授予权限来读取拨\-必须在属性中的用户帐户在授权过程中，在 AD DS 中注册每个 NPS。 注册 NPS 将添加到服务器**RAS 和 IAS 服务器**AD DS 中的安全组。

>[!NOTE]
>您可以在域控制器或专用服务器上安装 NPS。 运行以下 Windows PowerShell 命令来安装 NPS，如果尚未这样做：
    
    Install-WindowsFeature NPAS -IncludeManagementTools
    
必须至少具有 **Domain Admins** 中的成员身份或同等身份才能完成此过程。

### <a name="bkmk_radiusclient"></a>将无线 AP 配置为 NPS RADIUS 客户端
可以使用此过程也称为配置 AP*网络访问服务器\(NAS\)*，为远程身份验证拨\-入用户服务\(RADIUS\)使用 NPS 管理单元的客户端\-中。 

>[!IMPORTANT]
>客户端计算机（例如，无线便携式计算机和其他运行客户端操作系统的计算机）并非 RADIUS 客户端。 RADIUS 客户端是网络访问服务器，如无线访问点、 802.1 X\-支持交换机、 虚拟专用网络\(VPN\)服务器和拨号\-向上服务器 — 因为它们使用 RADIUS 协议与 RADIUS 服务器，例如 NPSs 进行通信。

必须至少具有 **Domain Admins** 中的成员身份或同等身份才能完成此过程。

    例如，如果你想要添加的无线访问点\(亚太\)名为 AP\-01，类型**AP\-01**。

5. 在中**地址\(IP 或 DNS\)**，键入 IP 地址或完全限定的域名\(FQDN\) nas。

    如果输入的 FQDN，以便验证名称是否正确以及映射到有效的 IP 地址，则单击**验证**，然后在**验证地址**，在**地址**字段中，单击**解决**。 如果 FQDN 名称映射到有效的 IP 地址，该 NAS 的 IP 地址将自动显示在**IP 地址**。 如果 FQDN 不会解析为 IP 地址将收到一条指示此主机不已知消息。 如果发生这种情况，验证具有正确的接入点名称和 AP 都已启动并连接到网络。  

    单击**确定**以关闭**验证地址**。  

6. 在中**新建 RADIUS 客户端**，在**共享机密**，执行下列操作之一：  

    -   若要手动配置 RADIUS 共享的机密，请选择**手动**，然后在**共享机密**，键入在 NAS 也输入强密码。 重新键入中的共享的机密**确认共享的机密**。

        >[!IMPORTANT]

## <a name="bkmk_domain"></a>将新的无线计算机加入到域
若要将新的无线计算机加入到域的最简单方法是以物理方式将该计算机连接到有线 LAN 的段\(802.1 X 交换机不受控制的段\)之前将计算机加入到域。 这是最简单的因为无线组策略设置自动并立即应用，并且，如果部署了自己的 PKI，计算机接收的 CA 证书，并将其放在受信任的根证书颁发机构证书存储中允许使用由 CA 颁发的服务器证书信任 NPSs 无线客户端。

同样，新的无线计算机加入到域后，用户登录到域的首选的方法是使用有线的连接到网络上执行日志。

### <a name="other-domain-join-methods"></a>其他域加入方法
在情况下不是实际将计算机加入到域中，通过使用有线以太网连接，或在情况下用户无法登录到通过使用有线的连接，在第一次域必须使用另一种方法。

- **IT 员工计算机配置**。 IT 团队的成员加入到域中的无线计算机，并配置单一登录启动无线配置文件。 使用此方法，IT 管理员可将无线计算机连接到有线以太网网络，并将计算机加入到域。 然后管理员将分发给用户的计算机。 当同时使用而无需使用有线的连接，而手动指定用户登录的域凭据的计算机在用户开始建立连接到无线网络和登录到域。

有关详细信息，请参阅部分[使用 IT 员工计算机配置方法加入的域和登录](#bkmk_itstaff)

-   **启动无线配置文件配置用户**。 用户手动通过启动无线配置文件配置无线计算机，并加入域，从 IT 管理员获取说明基础。 启动无线配置文件允许用户建立无线连接，然后加入域。 将计算机加入到域并重新启动计算机之后, 用户可以登录到域使用无线连接和其域帐户凭据。

有关详细信息，请参阅明[使用启动无线配置文件配置用户加入的域和登录](#bkmk_userbootstrap)。

### <a name="bkmk_itstaff"></a>使用 IT 员工计算机配置方法加入的域和登录
与域的域成员用户\-加入的无线客户端计算机可以使用临时无线配置文件以连接到 802.1x\-通过无线网络身份验证而无需首先连接到有线 LAN。 此临时无线配置文件称为*启动无线配置文件*。

启动无线配置文件要求用户手动指定其域用户帐户凭据，而不会验证的证书的远程身份验证拨\-入用户服务\(RADIUS\)服务器运行网络策略服务器\(NPS\)。

建立无线连接后，无线客户端计算机上应用了组策略，并自动颁发新的无线配置文件。 新策略使用的计算机和用户帐户凭据进行客户端身份验证。 

此外，PEAP 的一部分\-MS\-CHAP v2 进行相互身份验证而不启动配置文件，客户端使用新的配置文件验证 RADIUS 服务器的凭据。

将计算机加入到域后，使用此过程之前分发无线计算机加入域中配置单一登录启动无线配置文件，\-成员用户。

#### <a name="to-configure-a-single-sign-on-bootstrap-wireless-profile"></a>若要配置单一登录启动无线配置文件

1. 通过使用名为本指南中的过程创建启动配置文件[peap 配置无线连接配置文件\-MS\-CHAP v2](#bkmk_configureprofile)，并使用以下设置：

    - PEAP\-MS\-CHAP v2 身份验证

    - 验证 RADIUS 服务器证书已禁用

    - 启用单一登录

2. 中的无线网络策略在其中创建新的启动配置文件属性**常规**选项卡上，选择启动配置文件，然后单击**导出**导出到配置文件网络共享、 USB 闪存驱动器或其他方便访问的位置。 该配置文件将保存为指定的位置的 *.xml 文件。

3. 将新的无线计算机加入到域\(例如，通过以太网连接，不需要 IEEE 802.1x 身份验证\)并将启动无线配置文件添加到计算机，通过使用**netsh wlan添加配置文件**命令。

    >[!NOTE]
    >有关详细信息，请参阅 Netsh 命令用于无线局域网\(WLAN\)处[http:\/\/technet.microsoft.com\/库\/dd744890.aspx](https://technet.microsoft.com/library/dd744890).

4. 分发新的无线计算机向用户与"登录到域使用运行 Windows 10 的计算机。"的过程

当用户启动计算机时，Windows 将提示用户输入其域用户帐户名和密码。 由于启用了单一登录，计算机将使用域用户帐户凭据首先建立与无线网络，然后登录到域的连接。

#### <a name="bkmk_w10"></a>登录到使用运行 Windows 10 的计算机在域上

1. 注销计算机，或重新启动计算机。

2. 在键盘上按任意键或在桌面上单击。 显示本地用户帐户名称和密码项字段名称的下方出现登录屏幕。 不要登录使用本地用户帐户。

3. 在屏幕的左下角，单击**其他用户**。 其他用户登录屏幕显示使用两个字段、 用户名和密码。 以下密码字段是文本**登录到：** ，然后将计算机加入其中的域的名称。 例如，如果你的域名为 example.com，文本将读取**登录到：示例**。

4. 在中**用户名**，键入您的域用户名称。

5. 在 **“密码”** 中，键入域密码，然后单击箭头，或按 ENTER 键。

>[!NOTE]
>如果**其他用户**屏幕不包含文本**登录到：** 和你的域名，应采用格式输入你的用户名*域\\用户*。 例如，以登录到使用名为帐户域 example.com**用户\-01**，键入**示例\\用户\-01**。

#### <a name="to-configure-a-bootstrap-wireless-profile"></a>若要配置启动无线配置文件

1. 使用提供的网络管理员或 IT 支持的凭据登录到本地计算机的管理员帐户的计算机上的专业人员。

2. 右\-单击桌面上的网络图标，然后单击**开放的网络和共享中心**。 这将打开“网络和共享中心”。 在中**更改网络设置**，单击**设置新的连接或网络**。 **设置连接或网络**对话框随即打开。

4. 在中**手动连接到无线网络**，在**网络名称**，键入 AP 的 SSID 名称。  

5. 在中**安全类型**，选择管理员提供的设置。

7. 选择**自动启动这个连接**，然后单击**下一步**。

8. 在 **已成功添加 * * * 你网络 SSID*，单击**更改连接设置**。

10. 单击**安全**选项卡上，然后在**选择网络身份验证方法**，选择**受保护的 EAP \(PEAP\)**。

11. 单击“设置”。 **受保护的 EAP \(PEAP\)属性**页随即打开。

12. 在中**受保护的 EAP \(PEAP\)属性**页上，确保**验证服务器证书**是未选中状态，单击**确定**两次，并然后单击**关闭**。

13. 然后，Windows 将尝试连接到无线网络。 启动无线配置文件的设置指定，你必须提供域凭据。 当 Windows 提示您输入的帐户名和密码时，键入您的域帐户凭据，如下所示：*域名称\\用户名*，*域密码*。
