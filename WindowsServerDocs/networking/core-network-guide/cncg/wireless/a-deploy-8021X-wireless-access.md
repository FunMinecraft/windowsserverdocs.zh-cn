## <a name="technology-overviews"></a>技术概述
以下是部署无线访问的一些技术概述：

### <a name="ieee-8021x"></a>IEEE 802.1X
IEEE 802.1x 标准定义的端口\-基于用于提供对以太网网络的身份验证的网络访问的网络访问控制。 此端口\-基于的网络访问控制使用交换 LAN 基础结构的物理特征进行身份验证连接到 LAN 端口的设备。 如果身份验证过程失败，对端口的访问可能会被拒绝。 虽然此标准专为有线以太网网络设计的但它已针对 802.11 无线 Lan 上使用。

### <a name="8021x-capable-wireless-access-points-aps"></a>802.1x\-支持的无线访问点\(APs\)
此方案需要一个或多个 802.1 X 的部署\-能够与远程身份验证拨兼容的无线 Ap\-入用户服务\(RADIUS\)协议。

802.1x 和 RADIUS\-称为符合 Ap，在具有如 NPS RADIUS 服务器的 RADIUS 基础结构中部署*RADIUS 客户端*。

### <a name="wireless-clients"></a>无线客户端
本指南提供了全面的配置详细信息，以提供 802.1x 身份验证访问域\-与运行 Windows 10、 Windows 8.1 和 Windows 8 的无线客户端计算机连接到网络的成员用户。 计算机必须加入到域中，以便成功建立经过身份验证的访问权限。

>[!NOTE]
>此外可以使用为无线客户端运行 Windows Server 2016、 Windows Server 2012 R2 和 Windows Server 2012 的计算机。

### <a name="support-for-ieee-80211-standards"></a>IEEE 802.11 标准的支持
支持的 Windows 和 Windows Server 操作系统提供构建\-中对 802.11 无线网络。 在这些操作系统，已安装的 802.11 无线网络适配器显示为网络和共享中心中的无线网络连接。 

尽管存在生成\-中对 802.11 无线网络的支持，Windows 的无线组件是依赖于以下：

- 无线网络适配器的功能。 已安装的无线网络适配器必须支持无线 LAN 或所需的无线安全标准。 例如，如果无线网络适配器不支持 Wi\-Fi 受保护的访问权限\(WPA\)，不能启用或配置 WPA 安全选项。

- 无线网络适配器驱动程序的功能。 若要允许您配置无线网络选项，无线网络适配器的驱动程序必须支持报告其功能的所有到 Windows。 验证您的无线网络适配器的驱动程序将被写入你的操作系统的功能。 此外请确保该驱动程序通过检查 Microsoft 更新或无线网络适配器供应商的网站是最新版本。

### <a name="wireless-network-security-methods"></a>无线网络安全方法
**无线网络安全方法**是无线身份验证的非正式分组\(有时称为无线安全\)和无线安全加密。 无线身份验证和加密中使用对以阻止未经授权的用户访问无线网络，以及保护无线传输。 

在无线网络策略的组策略中配置的无线安全设置，当有多个组合，可供选择。 但是，仅 WPA2\-企业版、 WPA\-企业版和打开与 802.1x 身份验证标准支持 802.1 X 身份验证的无线部署。

>[!NOTE]
>在配置的无线网络策略时，必须选择**WPA2\-企业**， **WPA\-企业**，或者**用 802.1x 打开**中顺序访问的 EAP 设置的所需的 802.1x 身份验证的无线部署。  

#### <a name="wireless-authentication"></a>无线身份验证
本指南建议使用以下无线身份验证标准对 802.1 X 身份验证的无线部署。  

**Wi\-Fi 受保护的访问 – 企业\(WPA\-企业\)** WPA 是临时标准开发的 WiFi 联盟以符合 802.11 的无线安全协议。 WPA 协议开发响应数前面有线等效保密中发现的严重缺陷\(WEP\)协议。

WPA\-企业通过 WEP 通过提供改进的安全性：  

1. 要求，以确保集中式的相互身份验证和动态密钥管理的基础结构中使用 802.1 X EAP 框架的身份验证  

2. 增强完整性检查值\(ICV\)与消息完整性检查\(MIC\)，若要保护的标头和有效负载  

3. 实现以防止重播攻击的一个帧计数器  

**Wi\-Fi 保护访问 2 – 企业\(WPA2\-企业\)** 如 WPA\-企业标准，WPA2\-企业使用 802.1x 协议和 EAP 框架。 WPA2\-Enterprise 提供了更强的数据保护多个用户和大受管理的网络。 WPA2\-Enterprise 是一种可靠的协议，旨在通过验证网络用户通过身份验证服务器来防止未经授权的网络访问。  

#### <a name="wireless-security-encryption"></a>无线安全加密
无线安全加密用于保护无线传输的无线客户端和无线接入点之间发送。 无线安全加密结合使用与所选的网络安全身份验证方法。 默认情况下，运行 Windows 10、 Windows 8.1 和 Windows 8 的计算机支持这两个加密标准：

1. **临时密钥完整性协议** \(TKIP\)是一个较旧的加密协议，最初设计用于提供更安全的无线加密比无法提供的本质上是弱有线等效保密\(WEP\)协议。 TKIP 由 IEEE 802.11i 设计任务组和 Wi\-联盟 WEP 替换而无需替换为旧的硬件。 TKIP 是封装 WEP 有效负载，并允许用户的旧 WiFi 设备而无需替换硬件升级到 TKIP 的算法套件。 与 WEP、 TKIP 用作它的基础使用 RC4 流加密算法。 新协议，但是，会使用唯一的加密密钥，加密的每个数据包，较强的 wep 密钥。 尽管 TKIP 可用于升级较旧的设备设计为使用仅 WEP 的安全性，它不能解决所有面向无线 Lan 的安全问题，而且在大多数情况下是不足够强大，保护敏感的政府或公司数据传输。  

2. **高级加密标准** \(AES\)是首选的加密协议，用于商业和政府数据的加密。 AES 提供了比 TKIP 或 wep 做更高级别的无线传输安全性。 与 TKIP 和 WEP 中，不同 AES 需要支持 AES 标准的无线硬件。 AES 是对称\-公钥加密标准使用三个块加密法，AES\-128，AES\-192 和 AES\-256。

在 Windows Server 2016 中，以下 AES\-基于无线加密方法是可用于配置无线配置文件属性中选择一种身份验证的 WPA2 方法时\-企业，建议这样做。

1. **AES\-CCMP**。 计数器模式密码块链接消息身份验证代码协议\(CCMP\)实现 802.11i 标准和专为更高版本所提供的 WEP，相比安全加密并使用 128 位 AES 加密密钥。
2. **AES\-GCMP**。 Galois 计数器模式协议\(GCMP\)受 802.11 a c，AES 比效率更高\-CCMP，并提供更好的无线客户端的性能。 GCMP 使用 256 位 AES 加密密钥。

> [!IMPORTANT]
> 有线等效保密\(WEP\)是原始的无线安全标准，用于加密网络流量。 不应部署 WEP 在网络上有很好地因为\-知道在这种过时形式的安全漏洞。

### <a name="active-directorydoman-services-adds"></a>Active Directory 域服务\(AD DS\)
AD DS 提供了一个分布式的数据库，存储和管理有关网络资源和应用程序的信息\-目录中的特定数据\-启用应用程序。 管理员可以使用 AD DS 将网络元素（如用户、计算机和其他设备）整理到层次内嵌结构。 内嵌层次结构包括 Active Directory 林、 域林，以及组织单位\(Ou\)每个域中。 运行 AD DS 的服务器称为*域控制器*。  

AD DS 包含用户帐户、 计算机帐户和帐户属性所需的 IEEE 802.1x 和 PEAP\-MS\-CHAP v2 进行身份验证用户凭据，并评估为无线连接的授权。

### <a name="active-directory-users-and-computers"></a>Active Directory 用户和计算机
Active Directory 用户和计算机是包含代表物理实体，如计算机、 用户或安全组的帐户的 AD ds 的组件。 一个*安全组*是管理员可以作为一个单元管理的用户或计算机帐户的集合。 属于特定组的用户和计算机帐户嘿 *组成员*。  

### <a name="group-policy-management"></a>组策略管理
组策略管理可让目录\-基于用户和计算机设置，包括安全和用户信息的更改和配置管理。 使用组策略来定义组的用户和计算机配置。 使用组策略，可以指定注册表项、 安全性、 软件安装、 脚本、 文件夹重定向、 远程安装服务和 Internet Explorer 维护的设置。 您创建的组策略设置包含在组策略对象\(GPO\)。 通过将 GPO 与所选的 Active Directory 系统容器相关联 — 站点、 域和 Ou，您可以将 GPO 的设置应用于的用户和计算机在这些 Active Directory 容器中的。 若要管理整个企业内的组策略对象，可以使用组策略管理编辑器 Microsoft 管理控制台\(MMC\)。  

本指南提供有关如何在无线网络中指定设置的详细的说明\(IEEE 802.11\)策略组策略管理扩展。 无线网络\(IEEE 802.11\)策略配置域\-具有必要的连接和对 802.1 X 无线设置的成员无线客户端计算机身份验证无线访问。

### <a name="server-certificates"></a>服务器证书
此部署方案需要为每个执行 802.1x 身份验证的 NPS 服务器证书。  

服务器证书是一种数字文件，通常用于进行身份验证并保护开放网络的信息。 证书将公钥与持有相应私钥的实体牢固地绑定在一起。 证书由颁发 CA 进行数字签名，并在可以发出有关用户、 计算机或服务。  

证书颁发机构\(CA\)是一个实体负责建立和复核属于使用者公钥的真实性\(通常是用户或计算机\)或其他 Ca。 证书颁发机构的活动可以包括将公钥绑定到通过签署的证书，管理证书序列号和吊销证书的可分辨名称。  

Active Directory 证书服务\(AD CS\)是作为网络 CA 颁发证书的服务器角色。 AD CS 证书基础结构，也称为*公钥基础结构\(PKI\)*，提供可自定义服务，用于颁发和管理适用于企业的证书。

### <a name="eap-peap-and-peap-ms-chap-v2"></a>EAP，PEAP 和 PEAP\-MS\-CHAP v2
可扩展身份验证协议\(EAP\)扩展点\-到\-点协议\(PPP\)通过允许使用的凭据和信息的附加身份验证方法exchanges 任意长度。 利用 EAP 身份验证，这两个网络访问客户端和身份验证器\(如 NPS\)必须支持相同的 EAP 类型为成功进行身份验证发生。 Windows Server 2016 包括 EAP 基础结构，支持两种 EAP 类型，以及将 EAP 消息传递到 NPSs 的能力。 通过使用 EAP，可以支持其他身份验证方案，称为*EAP 类型*。 Windows Server 2016 支持的 EAP 类型为：  

- 传输层安全性\(TLS\)

- Microsoft 质询握手身份验证协议版本 2 \(MS\-CHAP v2\)

>[!IMPORTANT]
>强 EAP 类型\(如基于证书的那些\)提供更好的安全性暴力\-强制攻击、 词典式攻击以及密码猜测攻击比密码\-基于身份验证协议\(如 CHAP 或 MS\-CHAP 版本 1\)。

受保护的 EAP \(PEAP\)使用 TLS 加密的通道进行身份验证 PEAP 客户端，如无线计算机和 PEAP 身份验证器等的 NPS 或其他 RADIUS 服务器之间创建。 PEAP 不指定身份验证方法，但它对于其他 EAP 身份验证协议提供额外的安全性\(如 EAP\-MS\-CHAP v2\) ，可以通过 TLS 加密通道进行操作通过 PEAP 提供。 PEAP 用作连接到你组织的网络通过以下类型的网络访问服务器的访问客户端身份验证方法\(Nas\):

- 802.1x\-支持无线访问点

- 802.1x\-支持身份验证交换机

- 运行 Windows Server 2016 和远程访问服务的计算机\(RAS\)的配置为虚拟专用网络\(VPN\) 、 DirectAccess 服务器和 / 或服务器

- 运行 Windows Server 2016 和远程桌面服务的计算机

PEAP\-MS\-CHAP v2 是更轻松地部署比 EAP\-TLS 使用密码进行用户身份验证因为\-基于凭据\(用户名和密码\)，而不是证书或智能卡。 只有 NPS 或其他 RADIUS 服务器都需要有一个证书。 NPS 证书用于 nps 身份验证过程向 PEAP 客户端证明其身份。

本指南说明了如何配置无线客户端和在 NPS\(s\)若要使用 PEAP\-MS\-CHAP v2 对 802.1 X 身份验证的访问。

### <a name="network-policy-server"></a>网络策略服务器
网络策略服务器\(NPS\)允许您集中配置和管理网络策略通过使用远程身份验证拨\-入用户服务\(RADIUS\)服务器和 RADIUS 代理。 在部署 802.1x 无线访问时，NPS 是必需的。

在你 802.1x 无线访问点配置为 NPS 中的 RADIUS 客户端时，NPS 会处理发送的 APs 连接请求。 在连接请求处理期间 NPS 执行身份验证和授权。 身份验证确定客户端是否已提供有效的凭据。 如果 NPS 成功进行身份验证请求的客户端，然后 NPS 确定是否被授权客户端发出请求的连接，并允许或拒绝连接。 对此进行更多详细信息，如下所示：

#### <a name="authentication"></a>身份验证

成功的相互 PEAP\-MS\-CHAP v2 身份验证有两个主要部分：

1.  客户端进行身份验证的 NPS。  在此阶段的相互身份验证，在 NPS 服务器将其证书发送到客户端计算机，以便客户端可以验证该证书与 NPS 的标识。 若要成功进行身份验证 NPS，客户端计算机必须信任颁发 NPS 证书的 CA。 当客户端计算机上的受信任的根证书颁发机构证书存储中存在 CA 的证书时，客户端将信任此 CA。

    如果部署您自己专用的 CA，CA 证书自动安装当前的用户和本地计算机的受信任的根证书颁发机构证书存储中域成员客户端计算机上刷新组策略。 如果您决定将从公共 CA 的服务器证书部署，请确保公共 CA 证书已在受信任的根证书颁发机构证书存储区中。  

2.  NPS 对用户进行身份验证。 客户端成功进行身份验证 NPS 后，客户端发送用户的密码\-基于凭据向 NPS，Active Directory 域服务中的用户帐户数据库对用户的凭据进行电子验证\(ADDS\)。

如果凭据有效并且身份验证成功，NPS 将开始处理连接请求的授权阶段。 如果凭据不是有效并且身份验证失败，将 NPS 发送访问拒绝消息，拒绝连接请求。  

#### <a name="authorization"></a>授权

运行 NPS 的服务器执行授权，如下所示：  

1. NPS 检查中用户或计算机帐户拨号限制\-在 AD DS 中的属性。 在 Active Directory 用户和计算机中的每个用户和计算机帐户包括多个属性，包括那些上找到**拨\-中**选项卡。在此选项卡上，在**网络访问权限**，如果值为**允许访问**，用户或计算机有权连接到网络。 如果值为**拒绝访问**，用户或计算机无权连接到网络。 如果值为**通过 NPS 网络策略控制访问**，NPS 将评估配置的网络策略，以确定是否用户或计算机有权连接到网络。 

2. 然后，NPS 会处理网络策略，以便查找匹配连接请求策略。 如果找到匹配的策略，则 NPS 授予或拒绝基于该策略配置的连接。  

身份验证和授权已成功，如果匹配的网络策略授予访问权限，NPS 授予网络访问权限和用户和计算机可以连接到他们具有权限的网络资源。  

>[!NOTE]
>若要部署无线访问，必须配置 NPS 策略。 本指南介绍如何使用**配置 802.1 X 向导**在 NPS 中创建用于 802.1x 身份验证无线访问的 NPS 策略。  

### <a name="bootstrap-profiles"></a>启动配置文件
在 802.1x\-身份验证的无线网络，无线客户端必须提供进行 RADIUS 服务器身份验证，以便连接到网络的安全凭据。 有关受保护的 EAP \[PEAP\]\-Microsoft 质询握手身份验证协议版本 2 \[MS\-CHAP v2\]，安全凭据是用户名和密码。 有关 EAP\-传输层安全性\[TLS\]或 PEAP\-TLS，安全凭据是证书，如客户端用户和计算机证书或智能卡。

当连接到网络配置为执行 PEAP\-MS\-CHAP v2，PEAP\-TLS 或 EAP\-TLS 身份验证，默认情况下，Windows 无线客户端还必须验证的计算机证书由 RADIUS 服务器发送。 由 RADIUS 服务器发送的每个身份验证会话的计算机证书通常称为服务器证书。

正如前面提到的可以发出 RADIUS 服务器中通过两种方式之一其服务器证书： 从商业 CA \(VeriSign，Inc.，如\)，或在网络部署的专用 CA。 如果 RADIUS 服务器发送由商业 CA 已安装客户端的受信任的根证书颁发机构证书存储中的根证书颁发计算机证书，通过然后无线客户端可以验证 RADIUS 服务器计算机证书，不管该无线客户端是否已加入 Active Directory 域。 在这种情况下，无线客户端可以连接到无线网络，然后将计算机加入到域。

>[!NOTE]
>可以禁用要求客户端验证服务器证书的行为，但在生产环境中不建议禁用服务器证书验证。

Bootstrap 的无线配置文件是临时的配置文件配置中的方式使无线客户端用户能够连接到 802.1x\-先在计算机加入到域，身份验证，无线网络和\/或之前用户已成功登录到域的第一次使用给定的无线计算机。  本部分总结了当尝试将无线计算机加入到域，或对使用的域的用户遇到哪些问题\-加入第一次登录到域的无线计算机。

为部署中的用户或 IT 管理员不能以物理方式将计算机连接到有线以太网网络，将计算机加入到域，且计算机时，没有必要发出根 CA 证书安装在其**受信任的根证书颁发机构**证书存储中，您可以使用临时无线连接配置文件，称为配置无线客户端*启动配置文件*，用于连接到无线网络。

一个*启动配置文件*不要求验证 RADIUS 服务器的计算机证书。 此临时配置使无线用户可以将计算机加入到域，无线网络中哪些时间\(IEEE 802.11\)应用策略和上自动安装相应的根 CA 证书计算机。

一个或多个无线连接配置文件强制执行相互身份验证的要求的应用组策略时，将应用于计算机;启动配置文件时无需再并删除。 将计算机加入到域并重新启动计算机之后, 用户可以使用无线连接登录到域。
