# <a name="wireless-access-deployment-planning"></a>无线访问部署规划

>适用于：Windows 服务器 （半年频道），Windows Server 2016

在部署无线访问之前，必须规划以下各项：

- 无线访问点的安装\(APs\)网络上

- 无线客户端配置和访问

以下各节将详细介绍这些规划步骤。

## <a name="planning-wireless-ap-installations"></a>规划无线接入点安装
在设计您的无线网络访问解决方案时，必须执行以下操作：

1. 确定您的无线 Ap 必须支持哪些标准
2. 确定想要提供无线服务的覆盖区域
3. 确定您要在其中放置无线 Ap

此外，必须规划 IP 地址方案在无线 AP 和无线客户端。 请参阅的部分**计划的无线 AP 的在 NPS 中配置**下面有关相关信息。

### <a name="verify-wireless-ap-support-for-standards"></a>验证对标准的无线 AP 支持
有关保持一致性和易于部署和亚太管理，建议你部署的相同品牌和型号的无线访问点。

在部署无线 Ap 必须支持以下功能：

- **IEEE 802.1X**

- **RADIUS 身份验证**

- **无线身份验证和密码。** 列出按顺序的最具体到最不优先：

    1.  WPA2\-企业使用 AES

    2.  WPA2\-TKIP 与企业

    3.  WPA\-企业使用 AES

    4.  WPA\-TKIP 与企业

>[!NOTE]
>若要部署 WPA2，必须使用无线网络适配器和还支持 WPA2 的无线访问点。 否则，请使用 WPA\-企业。

此外，若要为网络提供增强的安全性，无线 Ap 必须支持以下安全选项：

- **DHCP 筛选。** 无线 AP 必须筛选 IP 端口，以防止传输 DHCP 广播消息，在这些情况下，在其中无线客户端配置为 DHCP 服务器。 无线 AP 必须阻止客户端将 IP 数据包从 UDP 端口 68 发送到网络。

- **DNS 筛选。** 无线 AP 必须筛选 IP 端口，以阻止客户端作为 DNS 服务器执行。 无线 AP 必须阻止客户端将发送 IP 数据包从 TCP 或 UDP 端口 53 到网络。

- **客户端隔离**如果将无线访问点提供了客户端隔离功能，则应启用功能以防止可能的地址解析协议\(ARP\)电子欺骗攻击。

### <a name="identify-areas-of-coverage-for-wireless-users"></a>确定无线用户可以覆盖区域
使用体系结构的每个构建每个楼层绘图来标识你想要提供无线覆盖率的区域。 例如，标识相应机构、 会议房间、 大堂、 餐厅或 courtyards。

在绘图，指示干扰无线信号，例如医疗设备、 无线摄像机、 无绳电话操作通过 2.5ghz 工业、 科学记数和医疗 2.4 中的任何设备\(ISM\)范围和蓝牙\-启用设备。

在绘图中，将标记可能会干扰无线信号; 构建的方面金属物品的建筑物的构造中使用可能会影响无线信号。 例如，以下常见对象可能会干扰信号传播：升降机、 加热和空气\-调节管道管道和具体支持 girders。

请参阅联系 AP 制造商以获得有关可能会导致无线 AP 广播频率衰减的源的信息。 大多数 APs 提供可用于检查信号强度、 错误速率和数据吞吐量的测试软件。

### <a name="determine-where-to-install-wireless-aps"></a>确定安装无线接入点位置
在体系结构绘图中，找到无线 Ap 关闭足够一起提供足够无线覆盖率但足够远，它们不会干扰其他。

APs 之间的必要距离取决于的亚太的类型和亚太天线，阻止构建的方面无线信号和干扰其他源。 可以将标记无线接入点位置，以便每个无线 AP 不超过 300 英尺从任何相邻的无线 AP。 请参阅亚太规范的无线 AP 制造商的文档和放置指导方针。

在体系结构绘图中指定的位置中暂时安装无线访问点。 然后，使用便携式计算机配备有无线适配器与 802.11 无线适配器和通常提供站点调查软件，确定每个主管的领域内的信号强度。

在信号强度较低覆盖率区域，AP 提高覆盖率区域的信号强度，安装其他无线 Ap 提供必要的覆盖范围、 重新定位或删除信号干扰源的位置。  

更新体系结构绘图以指示所有无线 Ap 的最后一个位置。 更高版本将在故障排除操作或你想要升级或替换 Ap 时，具有准确的 AP 放置映射将提供帮助。

### <a name="plan-wireless-ap-and-nps-radius-client-configuration"></a>计划无线 AP 和 NPS 的 RADIUS 客户端配置
NPS 可用于配置无线 Ap 单独或成组。

如果部署包括许多 Ap 的大型无线网络，很容易地在组中配置 Ap。 若要将 Ap 添加为 RADIUS 客户端组在 NPS 中，必须具有这些属性配置 Ap。

- 无线访问点使用相同的 IP 地址范围中的 IP 地址配置。

- 无线访问点都被配置的相同共享机密。

### <a name="plan-the-use-of-peap-fast-reconnect"></a>计划使用 PEAP 快速重新连接
802.1 X 基础结构中的无线访问点配置为 RADIUS 客户端到 RADIUS 服务器。 部署 PEAP 快速重新连接时，两个或多个访问点之间漫游的无线客户端不需要使用每个新的关联进行身份验证。

PEAP 快速重新连接减少了客户端和身份验证器之间的身份验证的响应时间，因为身份验证请求从新的访问点转发到原来的客户端执行身份验证和授权的 NPS连接请求。

因为 PEAP 客户端和 NPS 同时使用以前缓存的传输层安全性\(TLS\)连接属性\(的集合名为 TLS 句柄\)，NPS 可以快速确定的客户端已重新连接获得授权。

>[!IMPORTANT]
>实现快速重新连接才能正常工作，必须将 Ap 配置为相同 NPS 的 RADIUS 客户端。

如果原始 NPS 变得不可用，或者客户端移到配置为不同的 RADIUS 服务器的 RADIUS 客户端访问点，必须在客户端和新的身份验证器之间进行完全身份验证。

### <a name="wireless-ap-configuration"></a>无线 AP 配置
以下列表汇总了通常对 802.1 X 配置项目\-支持无线访问点：

> [!NOTE]
> 项名称按品牌和型号而异，因此可能不同于与以下列表中。 请参阅用于配置无线 AP 文档\-的特定详细信息。

- **服务设置标识符\(SSID\)**。 这是无线网络的名称\(例如，ExampleWlan\)，以及播发到无线客户端的名称。 若要减少混淆，选择要播发的 SSID 不应与广播接收您的无线网络范围内的任何无线网络的 SSID。

    在同一无线网络的一部分部署多个无线 Ap 的情况下，使用同一个 SSID 配置每个无线接入点。 在同一无线网络的一部分部署多个无线 Ap 的情况下，使用同一个 SSID 配置每个无线接入点。  

    无线 AP 的一个网络上应在其中有需要部署不同的无线网络，以满足特定业务需求的情况下，广播 SSID 其他网络比不同 SSID\(s\)。 例如，如果需要为员工和来宾的单独的无线网络，你可以配置您的企业网络的无线 Ap 具有设置为广播 SSID **ExampleWLAN**。 为来宾网络，然后，可以设置每个无线 AP 的 SSID 将广播**GuestWLAN**。 在这种方式中的员工和访可以连接到预期的网络而不会不必要的混淆。  

    > [!TIP]  
    > 一些无线 AP 能够广播多个 SSID 的以适应多\-网络部署。 无线 AP 的可以广播多个 SSID，可以减少部署和操作维护成本。  

- **无线身份验证和加密**。

    无线身份验证是安全的身份验证无线客户端与无线访问点关联时使用。  

    无线加密是与无线身份验证一起使用来保护无线 AP 和无线客户端之间发送的通信安全加密密码。  

- **无线接入点 IP 地址\(静态\)**。 每个无线 AP 上配置唯一的静态 IP 地址。 如果子网由 DHCP 服务器提供服务，请确保所有接入点 IP 地址位于 DHCP 排除范围，以便不会尝试向另一台计算机或设备颁发相同的 IP 地址的 DHCP 服务器。 排除范围"来创建并激活新的 DHCP 作用域"的过程中所述中[核心网络指南](https://technet.microsoft.com/windows-server-docs/networking/core-network-guide/core-network-guide)。 如果你打算将 Ap 配置为 RADIUS 客户端通过在 NPS 中的组，组中的每个 AP 必须具有相同的 IP 地址范围中的 IP 地址。

- **DNS 名称**。 可以将一些无线 Ap 配置具有 DNS 名称。 配置每个无线 AP 的唯一名称。 例如，如果在多具有已部署的无线 Ap\-情景构建，可以命名部署在三楼 AP3 的前三个无线 Ap\-01，AP3\-02 和 AP3\-03。

- **无线 AP 子网掩码**。 配置要指定部分的 IP 地址是网络 ID 和 IP 地址的哪个部分是主机的掩码。

- **亚太 DHCP 服务**。 如果在无线 AP 具有内置\-DHCP 服务中禁用它。

- **RADIUS 共享的机密**。 使用唯一 RADIUS 共享机密为每个无线 AP，除非想要在组-在哪种情况下必须配置所有的组中的 Ap 的相同共享机密配置 NPS RADIUS 客户端。 共享的机密应为一系列随机至少 22 个字符之间，与大写和小写字母、 数字和标点。 若要确保随机性，可以使用随机字符生成程序来创建共享的机密。 建议您记录每个无线 AP 的共享的机密，并将其存储在安全位置，例如 office 安全。 在 NPS 控制台中配置 RADIUS 客户端时将创建的虚拟版本，每个 AP。 在 NPS 中每个虚拟 AP 配置的共享的机密必须与匹配实际的物理接入点上的共享的机密。

- **RADIUS 服务器 IP 地址**。 键入想要用于身份验证和授权连接请求此接入点到 NPS 的 IP 地址。

- **UDP 端口\(s\)**。 默认情况下，NPS 使用 UDP 端口 1812年和 1645年用于 RADIUS 身份验证消息和 UDP 端口 1813年和 1646 用于 RADIUS 记帐消息。 建议您不要更改默认 RADIUS UDP 端口设置。

- **Vsa**。 一些无线 Ap 需要供应商\-特定属性\(Vsa\)以提供完整无线 AP 的功能。

- **DHCP 筛选**。 配置无线 Ap，阻止将 IP 数据包从 UDP 端口 68 发送到网络的无线客户端。 请参阅在无线 AP，若要配置 DHCP 筛选的文档。

- **DNS 筛选**。 配置无线 Ap，阻止将 IP 数据包从 TCP 或 UDP 端口 53 发送到网络的无线客户端。 请参阅在无线 AP，若要配置 DNS 筛选的文档。

## <a name="planning-wireless-client-configuration-and-access"></a>规划无线客户端配置和访问

在规划部署 802.1x\-身份验证无线访问，必须考虑几个客户端\-特定因素：

- **规划对多个标准的支持**。

    确定是否将无线计算机都使用相同版本的 Windows，或者是否是运行不同操作系统的计算机的混合。 如果它们不同，请确保您了解在标准操作系统支持的任何差异。

    确定是否所有的无线网络适配器上的所有无线客户端计算机都支持相同的无线标准，或者您是否需要支持不同的标准。 例如，确定某些网络适配器硬件驱动程序是否支持 WPA2\-企业和 AES，有些则支持仅 WPA\-企业和 TKIP。

- **规划客户端身份验证模式**。 身份验证模式定义 Windows 客户端如何处理域凭据。 您可以从以下三种网络身份验证模式的无线网络策略中进行选择。  

    1. **用户重新\-身份验证**。 此模式下指定始终通过使用基于计算机的当前状态的安全凭据执行身份验证。 如果没有用户登录到计算机，通过使用计算机凭据执行身份验证。 当用户登录到计算机时，始终使用用户凭据执行身份验证。  

    2. **仅计算机** 计算机仅模式指定身份验证始终执行通过仅使用计算机凭据。  

    3.  **用户身份验证** 用户身份验证模式指定用户登录到计算机时，才执行身份验证。 当没有用户登录到计算机时，不会执行身份验证尝试。  

- **规划无线限制**。 确定是否想要为所有无线用户提供相同级别的访问无线网络，或是否想要限制某些无线用户访问权限。 您可以应用在 NPS 中针对特定的无线用户组的限制。  例如，你可以定义特定日和小时数的特定组允许对无线网络的访问。  

- **规划用于添加新的无线计算机方法**。 为无线\-支持计算机已加入到域之前部署您的无线网络，如果计算机连接到不受 802.1x 的有线网络段，无线配置设置是自动应用配置无线网络后\(IEEE 802.11\)策略在域控制器上和无线客户端上刷新组策略。  

    对于未已加入到你的域的计算机，但是，你必须计划要应用对 802.1 X 所需的设置方法\-身份验证的访问。 例如，确定您是否要将计算机加入到域，使用以下方法之一。

    1.  将计算机连接到不受 802.1 X 有线网络段，然后将计算机加入到域。

    2.  无线用户提供的步骤和所需添加其自己无线启动配置文件，这使他们可以将计算机加入到域的设置。

    3.  将分配 IT 人员将无线客户端加入到域。

### <a name="planning-support-for-multiple-standards"></a>规划对多个标准的支持

无线网络\(IEEE 802.11\)策略扩展在组策略中的提供了各种配置选项来支持多种部署选项。

可以部署与你想要支持，标准配置的无线 Ap，然后在无线网络中配置多个无线配置文件\(IEEE 802.11\)策略，指定一组标准的每个配置文件您需要。

例如，如果您的网络具有支持 WPA2 的无线计算机\-企业和 AES，支持 WPA 的其他计算机\-企业和 AES 和其他计算机的支持仅 WPA\-企业和 TKIP，您必须确定是要：

- 配置单个配置文件来支持所有的无线计算机使用最弱的加密方法，你的所有计算机支持-在这种情况下，WPA\-企业和 TKIP。  
- 配置两个配置文件来提供支持的每个无线计算机的最佳可能的安全。 在此实例中，你将配置一个配置文件指定的最强加密\(WPA2\-企业和 AES\)，另一个配置文件使用较弱的 WPA\-企业和 TKIP 加密。 在此示例中，是基本配置文件使用 WPA2 放\-企业和 AES 按优先顺序最高。 计算机不能使用 WPA2\-企业和 AES 将自动跳到下一个配置文件中的首选项顺序并处理的配置文件的指定 WPA\-企业和 TKIP。

> [!IMPORTANT]
> 因为连接的计算机使用的第一个配置文件所支持的使用，必须将具有更高版本的最安全标准的配置文件放在排序列表的配置文件中。

### <a name="planning-restricted-access-to-the-wireless-network"></a>规划对无线网络访问受到限制

在许多情况下，你可能想要无线用户提供不同的无线网络访问权限级别。 例如，你可能想要允许某些用户不受限制访问权限，在一天，每个日期是星期几的任何小时。 对于其他用户，您可能只希望在核心小时数，星期一到星期五，允许访问，并在星期六和星期日上拒绝访问。

本指南介绍如何创建所有无线用户置于无线资源的公共访问权限的组的访问环境。 在 Active Directory 用户中创建一个无线用户安全组和计算机管理单元\-在中，然后使您要为其授予此组的成员的无线访问的每个用户。

在配置 NPS 网络策略时，NPS 会处理时确定授权的对象作为指定无线用户安全组。

但是，如果您部署的不同级别的访问需要支持您需要只进行以下：  

1. 创建多个无线用户安全组在 Active Directory 用户和计算机创建其他无线安全组。 例如，可以创建一个包含具有完全访问权限，组的用户只能访问正常工作时间期间以及其他符合其他条件符合你需求的组的用户组。

2. 将用户添加到您创建适当的安全组。

3. 配置每个其他的无线安全组、 其他 NPS 网络策略和配置要应用的条件和约束，你需要为每个组的策略。

### <a name="planning-methods-for-adding-new-wireless-computers"></a>规划将添加新的无线计算机的方法

若要将新的无线计算机加入到域，然后登录到域的首选的方法是通过使用有线的连接到有权访问域控制器，且不受 802.1x 身份验证的以太网交换机的 LAN 的段。

在某些情况下，但是，可能不会实际使用有线的连接来将计算机加入到域，或者，对于其第一个登录尝试使用已加入域的计算机来使用有线的连接的用户。

若要将计算机加入到域，通过使用无线连接或供用户使用域登录到域的第一个时间\-加入的计算机和无线连接，无线客户端首先必须在建立与无线网络的连接使用以下方法之一有权访问网络域控制器的段。

1. **IT 团队的成员将无线计算机加入到域，然后配置单一登录启动无线配置文件。** 使用此方法，IT 管理员可将无线计算机连接到有线以太网网络，并再将计算机加入到域。 然后管理员将分发给用户的计算机。 当用户启动计算机时，而手动指定用户登录过程的域凭据用于同时建立与无线网络的连接并登录到域。  

2. **用户通过启动无线配置文件，来手动配置无线计算机，然后加入域。** 使用此方法，用户使用手动配置其无线计算机根据 IT 管理员的说明启动无线配置文件。 启动无线配置文件允许用户建立无线连接，然后将计算机加入到域。 将计算机加入到域并重新启动计算机之后, 用户可以登录到域使用无线连接和其域帐户凭据。

若要部署无线访问，请参阅[无线访问部署](e-wireless-access-deployment.md)。
