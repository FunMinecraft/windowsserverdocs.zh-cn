# <a name="configure-network-policies"></a>配置网络策略

>适用于：Windows 服务器 （半年频道），Windows Server 2016

本主题可用于在 NPS 中配置网络策略。

## <a name="add-a-network-policy"></a>添加网络策略

网络策略服务器\(NPS\)使用网络策略和要确定是否授权连接请求以连接到网络的用户帐户的拨入属性。

此过程可用于在 NPS 控制台或远程访问控制台中配置新的网络策略。

### <a name="performing-authorization"></a>执行授权

当 NPS 执行连接请求授权时，它会将该请求与策略，从第一个策略，然后将配置的策略列表中向下移动的有序列表中每个网络策略进行比较。 如果 NPS 发现其条件符合连接请求策略，NPS 使用匹配策略和用户帐户的拨入属性执行授权。 如果用户帐户的拨入属性配置为授予访问权或控制访问权限通过网络策略和连接请求进行授权，NPS 会应用到连接的网络策略中配置的设置。

如果 NPS 找不到与连接请求匹配网络策略，除非上的用户帐户的拨入属性设置为授予访问权限被拒绝连接请求。

如果用户帐户的拨入属性设置为拒绝访问，是由 NPS 拒绝连接请求。

### <a name="key-settings"></a>密钥设置

当您使用新建网络策略向导创建网络策略中指定的值**网络连接方法**用于自动配置**策略类型**条件： 

- 如果保留未指定的默认值，您创建的网络策略由 NPS 评估使用任何类型的网络访问服务器 (NAS) 的所有网络连接类型。
- 如果指定了网络连接方法时，NPS 将评估网络策略，仅当连接请求源自您指定的网络访问服务器的类型。

上**访问权限**页上，必须选择**授予访问权限**如果你想要允许用户连接到网络的策略。 如果你想要阻止用户连接到网络，请选择的策略**访问被拒绝**。 

如果您希望由用户帐户拨入属性在 Active Directory 中的访问权限&reg;域服务\(AD DS\)，可以选择**访问由用户拨入属性**复选框。

必须至少具有 **Domain Admins** 中的成员身份或同等身份才能完成此过程。

### <a name="to-add-a-network-policy"></a>若要添加的网络策略 

1. 打开 NPS 控制台中，并双击**策略**。

2. 在控制台树中，右键单击**网络策略**，然后单击**新建**。 此时将打开新建网络策略向导。

3. 使用新建网络策略向导创建的策略。

## <a name="create-network-policies-for-dial-up-or-vpn-with-a-wizard"></a>为拨号或 VPN 使用向导创建网络策略

此过程可用于创建连接请求策略和网络策略，需要部署拨号服务器或虚拟专用网络\(VPN\)服务器作为远程身份验证拨入用户服务\(RADIUS\)到 NPS RADIUS 服务器的客户端。

>[!NOTE]
>客户端计算机，如便携式计算机和运行客户端操作系统，其他计算机不是 RADIUS 客户端。 RADIUS 客户端是网络访问服务器，如无线访问点、 802.1 X 身份验证交换机、 虚拟专用网络\(VPN\)服务器和拨号服务器，因为这些设备使用 RADIUS 协议进行通信与如 NPSs RADIUS 服务器。

此过程说明如何在 NPS 中打开新建拨号或虚拟专用网络连接向导。

运行向导后，将创建以下策略：

- 一个连接请求策略
- 一个网络策略

每次你需要为拨号服务器和 VPN 服务器创建新策略时，可以运行新建拨号或虚拟专用网络连接向导。

运行新建拨号或虚拟专用网络连接向导不是作为到 NPS RADIUS 客户端部署拨号或 VPN 服务器所需的唯一步骤。 这两个网络访问方法都需要部署其他硬件和软件组件。

必须至少具有 **Domain Admins** 中的成员身份或同等身份才能完成此过程。

### <a name="to-create-policies-for-dial-up-or-vpn-with-a-wizard"></a>若要使用向导为拨号或 VPN 创建策略

1. 打开 NPS 控制台。 如果尚未选中，单击**NPS\(本地\)**。 如果你想要在远程 NPS 上创建策略，选择的服务器。

2. 在中**Getting Started**并**标准配置**，选择**用于拨号或 VPN 连接的 RADIUS 服务器**。 文本和文本更改以反映你的选择下面的链接。

3. 单击**配置 VPN 或拨号一个向导**。 打开新建拨号或虚拟专用网络连接向导。

4. 按照向导中的说明，以完成创建新策略。

## <a name="create-network-policies-for-8021x-wired-or-wireless-with-a-wizard"></a>创建网络策略对 802.1 X 有线或无线一个向导

可以使用下列步骤创建连接请求策略和将 802.1 X 身份验证交换机或 802.1x 无线访问点部署作为到 NPS 的远程身份验证拨入用户服务 (RADIUS) 客户端所需的网络策略RADIUS 服务器。

此过程说明如何在 NPS 中启动新建 IEEE 802.1x 安全有线和无线连接向导。

运行向导后，将创建以下策略：

- 一个连接请求策略
- 一个网络策略

每次需要新建 802.1x 访问策略时，可以运行新建 IEEE 802.1x 安全有线和无线连接向导。

运行新建 IEEE 802.1x 安全有线和无线连接向导不是部署 802.1x 作为到 NPS RADIUS 客户端身份验证交换机和无线访问点所需的唯一步骤。 这两个网络访问方法都需要部署其他硬件和软件组件。

必须至少具有 **Domain Admins** 中的成员身份或同等身份才能完成此过程。

### <a name="to-create-policies-for-8021x-wired-or-wireless-with-a-wizard"></a>若要使用向导创建策略为 802.1x 有线或无线

1. 在 NPS 中，在服务器管理器中，单击**工具**，然后单击**网络策略服务器**。 将打开 NPS 控制台。 

2. 如果尚未选中，单击**NPS\(本地\)**。 如果你想要在远程 NPS 上创建策略，选择的服务器。

3. 在中**Getting Started**并**标准配置**，选择**用于 802.1x 无线或有线连接的 RADIUS 服务器**。 文本和文本更改以反映你的选择下面的链接。

4. 单击**使用向导配置 802.1x**。 将打开新建 IEEE 802.1x 安全有线和无线连接向导。

5. 按照向导中的说明，以完成创建新策略。

## <a name="configure-nps-to-ignore-user-account-dial-in-properties"></a>将 NPS 配置为忽略用户帐户拨入属性

使用此过程来配置要在授权过程中忽略 Active Directory 中的用户帐户的拨入属性的 NPS 网络策略。 在 Active Directory 用户和计算机的用户帐户具有拨入属性的授权过程中计算结果为 NPS，除非**网络访问权限**的用户帐户的属性设置为**控件通过 NPS 网络策略访问**。 

有两种情况下您可能想要将 NPS 配置为忽略 Active Directory 中的用户帐户的拨入属性：

- 当你想要使用网络策略，简化 NPS 授权，但并非所有用户帐户都**网络访问权限**属性设置为**通过 NPS 网络策略控制访问**。 例如，可能有一些用户帐户**网络访问权限**的用户帐户的属性设置为**拒绝访问**或**允许访问**。

- 当其他用户帐户的拨入属性都不适用于在网络策略中配置该连接类型。 例如，属性以外的其他**网络访问权限**设置仅适用于拨号或 VPN 连接，但要创建网络策略是针对无线或身份验证交换机的连接。

此过程可用于将 NPS 配置为忽略用户帐户拨入属性。 如果连接请求匹配网络策略选中此复选框的位置，NPS 不使用用户帐户的拨入属性来确定是否用户或计算机有权访问网络。仅在网络策略中的设置用于确定授权。

中的成员身份**管理员**，或等效身份是完成此过程所需的最小。

1. 在 NPS 中，在服务器管理器中，单击**工具**，然后单击**网络策略服务器**。 将打开 NPS 控制台。

2. 双击**策略**，单击**网络策略**，然后在详细信息窗格中双击你想要配置的策略。

3. 在策略中**属性**对话框中，在**概述**选项卡上，在**访问权限**，选择**忽略用户帐户拨入属性**复选框，然后依次**确定**。

## <a name="configure-nps-for-vlans"></a>将 NPS 配置为 Vlan

通过使用 Windows Server 2016 中识别 VLAN 的网络访问服务器和 NPS，可以仅对适用于其安全权限的网络资源具有访问权限提供用户的组。 例如，您可以访问者与无线访问到 Internet 但不允许他们访问你的组织网络。 

此外，Vlan 还允许到逻辑组的网络资源存在于不同的物理位置或不同的物理子网上。 例如，你的销售部门和它们的网络资源，如客户端计算机、 服务器和打印机的成员可能位于多个不同建筑物在你的组织，但可以将所有这些资源放在一个使用相同的 IP 的 VLAN 上地址范围。 VLAN 然后，此函数，从最终用户角度来看，作为单个子网。

当你想要隔离不同的用户组之间的网络时，还可以使用 Vlan。 确定你想要定义你的组之后，可以在 Active Directory 用户和计算机管理单元中，创建安全组，然后将成员添加到组。

### <a name="configure-a-network-policy-for-vlans"></a>配置适用于 Vlan 的网络策略

此过程可用于配置将用户分配给 VLAN 的网络策略。 在使用 VLAN 的网络硬件，如路由器、 交换机和访问控制器时可以配置网络策略，以指示访问服务器放置在特定 Vlan 上的特定 Active Directory 组的成员。 此功能组逻辑上与 Vlan 的网络资源提供了设计和实施网络解决方案时的灵活性。

在使用 Vlan 配置使用 NPS 网络策略的设置，必须配置属性**隧道介质类型**，**隧道 Pvt 组 ID**，**隧道类型**，并**隧道标记**。 

此过程提供指导原则是;你的网络配置可能需要不同的设置如下所述。

中的成员身份**管理员**，或等效身份是完成此过程所需的最小。

### <a name="to-configure-a-network-policy-for-vlans"></a>若要配置适用于 Vlan 的网络策略

1. 在 NPS 中，在服务器管理器中，单击**工具**，然后单击**网络策略服务器**。 将打开 NPS 控制台。

2. 双击**策略**，单击**网络策略**，然后在详细信息窗格中双击你想要配置的策略。

3. 在策略中**属性**对话框中，单击**设置**选项卡。

4. 在策略中**属性**，在**设置**，在**RADIUS 属性**，，请确保**标准**处于选中状态。

5. 在详细信息窗格中，在**特性**，则**服务类型**属性配置的默认值**需要不带**。 默认情况下，为策略的 VPN 和拨号访问方法**帧协议**属性配置值为**PPP**。 若要指定所需的 Vlan 的其他连接属性，请单击**添加**。 **添加标准的 RADIUS 属性**对话框随即打开。

6. 在中**添加标准的 RADIUS 属性**，属性，向下滚动到并添加以下属性：

    - **隧道媒体类型**。 选择适合于以前的策略所做的选择的值。 例如，如果要配置的网络策略的无线策略，请选择**值：802 （包括所有 802 介质和以太网规范格式）**。

    - **隧道 Pvt 组 ID**。 输入一个整数，表示要向其分配组成员的 VLAN 号码。 

    - **隧道类型**。 选择**虚拟 Lan (VLAN)**。


7. 在中**添加标准的 RADIUS 属性**，单击**关闭**。 

8. 如果您的网络访问服务器 (NAS) 需要使用**Tunnel-tag**属性，请使用以下步骤添加**隧道标记**向网络策略属性。 如果 NAS 文档不会提及此属性，请不将其添加到策略。 如果需要，添加属性，如下所示：

    - 在策略中**属性**，在**设置**，在**RADIUS 属性**，单击**供应商特定**。 

    - 在详细信息窗格中，单击**添加**。 **添加供应商特定属性**对话框随即打开。

    - 在中**特性**，向下滚动到并选择**隧道标记**，然后单击**添加**。 **属性信息**对话框随即打开。 

    - 在中**属性值**，键入从硬件文档获取的值。

## <a name="configure-the-eap-payload-size"></a>配置 EAP 有效负载大小

在某些情况下，路由器或防火墙丢弃的数据包因为它们已配置为放弃需要碎片的数据包。

当你使用可扩展的身份验证协议的网络策略与部署 NPS \(EAP\)使用传输层安全性\(TLS\)，或 EAP-TLS，作为身份验证方法，默认最大值传输单位\(MTU\) NPS 用于 EAP 有效负载为 1500 个字节。 

此最大大小为 EAP 有效负载可以创建需要碎片被路由器或防火墙 NPS 和 RADIUS 客户端之间的 RADIUS 消息。 这种情况，如果路由器或防火墙放置于 RADIUS 客户端和 NPS 之间可能会以无提示方式放弃某些片段，从而导致身份验证失败和访问客户端无法连接到网络。

使用以下过程来降低 NPS 通过调整为的值不应超过 1344年的网络策略中的不带 MTU 属性使用 EAP 有效负载的最大大小。

中的成员身份**管理员**，或等效身份是完成此过程所需的最小。

### <a name="to-configure-the-framed-mtu-attribute"></a>若要配置需要不带 MTU 属性

1. 在 NPS 中，在服务器管理器中，单击**工具**，然后单击**网络策略服务器**。 将打开 NPS 控制台。
 
2. 双击**策略**，单击**网络策略**，然后在详细信息窗格中双击你想要配置的策略。

3. 在策略中**属性**对话框中，单击**设置**选项卡。

4. 在中**设置**，在**RADIUS 属性**，单击**标准**。 在详细信息窗格中，单击**添加**。 **添加标准的 RADIUS 属性**对话框随即打开。

5. 在**特性**，向下滚动并单击**需要不带 MTU**，然后单击**添加**。 **属性信息**对话框随即打开。

6. 在中**属性值**，键入一个值小于或等于比**1344年**。 单击**确定**，单击**关闭**，然后单击**确定**。
