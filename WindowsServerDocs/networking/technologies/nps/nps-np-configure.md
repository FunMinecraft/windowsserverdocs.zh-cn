# <a name="configure-network-policies"></a>配置网络策略

>适用于：Windows Server（半年通道），Windows Server 2016

你可以使用本主题中 NPS 配置网络策略。

## <a name="add-a-network-policy"></a>添加网络策略

网络策略服务器 \(NPS\) 使用网络策略和授权的用户帐户，以确定是否连接请求拨号中属性连接到该网络。

可以使用此过程中 NPS 控制台或远程访问控制台配置新的网络策略。

### <a name="performing-authorization"></a>执行授权

在 NPS 执行授权连接请求时，它将在排序列表的策略，开头的第一个策略，并配置策略的列表中向下移动每个网络策略与该请求进行比较。 NPS 发现的条件与匹配连接请求的策略，如果 NPS 使用匹配的策略和用户帐户拨号中的属性执行授权。 如果连接请求已授权的用户帐户拨号中属性配置为授予通过网络策略的访问或控制访问权限，NPS 适用配置中连接到的网络策略设置。

如果 NPS 找不到网络策略符合请求的连接，连接请求被拒绝，除非拨号中的用户帐户设置授予访问权限。

如果拨号中的用户帐户设置来拒绝访问，连接请求被拒绝 nps。

### <a name="key-settings"></a>键设置

当你使用新的网络策略向导创建网络策略，在中指定的值**网络连接方法**用于自动配置**策略类型**条件： 

- 确保未指定的默认值，如果你创建网络策略计算 nps 正在使用的网络访问权限服务器 (NAS) 的任何类型的所有网络的连接类型。
- 如果你指定网络连接方法，NPS 评估网络策略，仅当连接请求都源自你指定的网络访问权限服务器的类型。

在**访问权限**页上，你必须选择**授予访问权限**如果你想要允许用户连接到你的网络的策略。 如果你希望策略阻止用户连接到你的网络，选择**拒绝访问**。 

如果你希望访问权限，可由用户帐户拨号中属性 Active Directory 中&reg;域服务 \(AD DS\)，你可以选择**访问由用户拨号中属性**复选框。

在会员**域管理员**，或等效的最低要求完成此过程。

### <a name="to-add-a-network-policy"></a>若要添加网络策略 

1. 打开 NPS 主机，然后双击**策略**。

2. 控制台树中，右键单击**网络策略**，然后单击**新建**。 打开新的网络策略的向导。

3. 使用新的网络策略向导创建的策略。

## <a name="create-network-policies-for-dial-up-or-vpn-with-a-wizard"></a>拨号或使用向导 VPN 创建网络策略

此过程用于创建连接请求策略和作为远程身份验证拨入用户服务 \(RADIUS\) NPS RADIUS 服务器的客户部署拨号服务器或虚拟专用网络 \(VPN\) 服务器所需的网络策略。

>[!NOTE]
>客户端计算机，如笔记本电脑和其他运行客户端操作系统的计算机不是 RADIUS 客户端。 RADIUS 客户端是网络的访问权限服务器，如无线接入点，802.1 X 身份验证交换机、虚拟专用网络 \(VPN\) 服务器和拨号服务器，因为这些设备使用 RADIUS 协议如服务器 NPS RADIUS 服务器与其通信。

此过程介绍如何在 NPS 打开新拨号或虚拟专用网络连接向导。

运行向导后，会创建以下策略：

- 连接一个请求策略
- 一个网络策略

每当你需要创建新为拨号服务器和 VPN 服务器的策略，你可以运行新拨号或虚拟专用网络连接向导。

运行新拨号或虚拟专用网络连接向导不是唯一部署拨号或 VPN 服务器服务器 NPS RADIUS 客户端为所需的步骤。 这两个网络的访问权限方法要求你部署附加的硬件和软件组件。

在会员**域管理员**，或等效的最低要求完成此过程。

### <a name="to-create-policies-for-dial-up-or-vpn-with-a-wizard"></a>使用向导创建拨号或 VPN 策略

1. 打开 NPS 主机。 如果它未选中，请单击**NPS \(Local\)**。 如果你想要创建策略 NPS 远程服务器上，依次选择服务器。

2. 在**入门**和**标准配置**、选择**拨号或 VPN 连接 RADIUS 服务器**。 文本和下更改，以反映你选择的链接。

3. 单击**配置 VPN 或使用向导拨号**。 打开新的拨号或虚拟专用网络连接向导。

4. 按照该向导中的说明完成创建新的策略。

## <a name="create-network-policies-for-8021x-wired-or-wireless-with-a-wizard"></a>创建用于 802.1 网络策略有线或无线用向导 X

此过程用于创建连接请求策略和需要为 NPS RADIUS 服务器远程身份验证拨入用户服务 (RADIUS) 的客户部署 802.1 X 身份验证交换机或 802.1 X 的无线接入点的网络策略。

此过程介绍如何 NPS 中启动新 IEEE 802.1 X 安全有线和无线连接向导。

运行向导后，会创建以下策略：

- 连接一个请求策略
- 一个网络策略

你可以在每次你需要先创建新的策略 802.1 X 访问运行新 IEEE 802.1 X 安全有线和无线连接向导。

运行新 IEEE 802.1 X 安全有线和无线连接向导不是唯一部署 802.1 X 身份切换和无线接入点验证作为服务器 NPS RADIUS 客户端所需的步骤。 这两个网络的访问权限方法要求你部署附加的硬件和软件组件。

在会员**域管理员**，或等效的最低要求完成此过程。

### <a name="to-create-policies-for-8021x-wired-or-wireless-with-a-wizard"></a>若要使用向导有线或无线创建 802.1 X 的策略

1. 在 NPS 服务器，在服务器管理器中，单击**工具**，然后单击**网络策略服务器**。 打开 NPS 主机。 

2. 如果它未选中，请单击**NPS \(Local\)**。 如果你想要创建策略 NPS 远程服务器上，依次选择服务器。

3. 在**入门**和**标准配置**、选择**802.1 X 无线或有线连接 RADIUS 服务器**。 文本和下更改，以反映你选择的链接。

4. 单击**配置 802.1 X 使用向导**。 打开新 IEEE 802.1 X 安全有线和无线连接的向导。

5. 按照该向导中的说明完成创建新的策略。

## <a name="configure-nps-to-ignore-user-account-dial-in-properties"></a>配置 NPS 忽略用户帐户拨号属性

使用此过程配置 NPS 网络策略，在授权过程忽略 Active Directory 中的用户帐户拨号中属性。 Active Directory 用户和计算机中的用户帐户已 NPS 评估授权过程中，除非拨号中属性**网络访问权限**的用户帐户的属性设置为**控制 NPS 网络策略访问**。 

有两种情况，你可能需要配置 NPS 忽略 Active Directory 中的用户帐户拨号中属性：

- 当你想要使用网络策略简化 NPS 授权，但不是所有用户帐户能够**网络访问权限**属性设置为**控制 NPS 网络策略访问**。 例如，可能有一些用户帐户**网络访问权限**的用户帐户属性设置为**拒绝访问**或**允许访问**。

- 当用户帐户的其他拨号中属性并非适用于配置在网络策略的连接类型。 例如，以外的其他属性**网络访问权限**设置将仅适用于拨或 VPN 连接，但你创建网络策略不适用于无线或身份验证开关连接。

你可以使用此过程配置 NPS 忽略用户帐户拨号属性。 如果连接请求相符网络策略了选中此复选框，NPS 不使用的用户帐户拨号中属性以确定是否用户或计算机有权访问网络;仅在网络策略设置用于确定授权。

在会员**管理员**，或等效的最低要求完成此过程。

1. 在 NPS 服务器，在服务器管理器中，单击**工具**，然后单击**网络策略服务器**。 打开 NPS 主机。

2. 双击**策略**，单击**网络策略**，然后双击详细信息窗格中的你想要配置的策略。

3. 在策略**属性**对话框中，在**概述**选项卡上，在**访问权限**、选择**忽略用户帐户拨号属性**复选框，然后依次单击**确定**。

## <a name="configure-nps-for-vlans"></a>配置 Vlan NPS

通过使用 Windows Server 2016 中的 VLAN 注意到的网络访问权限服务器和 NPS，你可以提供只能访问网络资源适合他们的安全性权限的组的用户。 例如，你可以提供访客与无线接入到 Internet 但不允许他们访问你的组织的网络。 

此外，Vlan 允许你的逻辑组网络资源存在在不同的物理位置，或在不同的物理个子网。 例如，你销售部门以及它们的网络资源的客户端，服务器和打印机，如成员可能位于在你的组织的几个不同的建筑，但您可以在使用相同的 IP 地址范围的一个 VLAN 放置所有这些资源。 VLAN 然后功能，从最终用户的角度，单个子网作为。

你还可以使用 Vlan 时你希望将网络之间不同组的用户的分隔。 你确定你想要定义组后，你可以创建安全组中的 Active Directory 用户和计算机贴靠中，并将成员添加到这些组。

### <a name="configure-a-network-policy-for-vlans"></a>配置 Vlan 网络策略

你可以使用此过程配置分配用户 VLAN 到的网络策略。 当你使用 VLAN 注意到的网络硬件，例如路由器、开关和访问控制器可以配置网络策略指示访问服务器，在特定 Vlan 放置特定 Active Directory 组中的成员。 灵活性设计和实现网络解决方案时提供了与 Vlan 的逻辑组网络资源功能。

与 Vlan 配置为使用 NPS 网络策略设置时，你必须配置属性**隧道中等类型**，**隧道 Pvt 组 ID**，**隧道类型**，并**隧道标记**。 

此过程提供原则;你的网络配置可能需要比，述不同的设置。

在会员**管理员**，或等效的最低要求完成此过程。

### <a name="to-configure-a-network-policy-for-vlans"></a>若要配置 Vlan 网络策略

1. 在 NPS 服务器，在服务器管理器中，单击**工具**，然后单击**网络策略服务器**。 打开 NPS 主机。

2. 双击**策略**，单击**网络策略**，然后双击详细信息窗格中的你想要配置的策略。

3. 在策略**属性**对话框中，单击**设置**选项卡。

4. 在策略**属性**，请在**设置**中**RADIUS 属性**，确保**标准**选择。

5. 在详细信息窗格中，在**属性**、**服务类型**属性配置默认值为**帧**。 默认情况下，访问方法 VPN 拨号，以及使用的策略**帧协议**属性配置值为**PPP**。 若要指定 Vlan 所需的其他连接属性，请单击**添加**。 **添加标准 RADIUS 属性**对话框中打开。

6. 在**添加标准 RADIUS 属性**、在属性，向下滚动到和添加以下属性：

    - **隧道中等类型**。 选择与以前的策略所做的选择相应的值。 例如，如果你正在配置网络策略无线策略，请选择**值：802（包括所有 802 媒体以及以太网 canonical 格式）**。

    - **隧道 Pvt 组 ID**。 输入表示将分配组成员 VLAN 数。 

    - **隧道类型**。 选择**虚拟 Lan (VLAN)**。


7. 在**添加标准 RADIUS 属性**，单击**关闭**。 

8. 如果你的网络访问权限服务器 (NAS) 要求使用**隧道标记**特性，请使用以下步骤添加**隧道标记**特性到的网络策略。 如果你的 NAS 文档不指出此特性，不要添加它策略。 如果需要，添加属性，如下所示：

    - 在策略**属性**中**设置**中**RADIUS 属性**，单击**供应商特定**。 

    - 在详细信息窗格中，单击**添加**。 **添加供应商特定属性**对话框中打开。

    - 在**属性**、向下滚动并选择**隧道标记**，然后单击**添加**。 **属性信息**对话框中打开。 

    - 在**特性值**，键入你购买了从硬件文档的值。

## <a name="configure-the-eap-payload-size"></a>配置 EAP 负载大小

在某些情况下，路由器或防火墙放数据包因为配置放弃需要碎片的数据包。

传输层安全性 \(TLS\) 或 EAP-TLS，身份验证方法为使用扩展验证协议 \(EAP\) 的网络策略部署 NPS 时，默认大传输 \(MTU\) NPS 用于 EAP 负载的 1500 字节。 

为 EAP 负载此最大大小可以创建 RADIUS 邮件需要通过路由器或之间服务器 NPS RADIUS 客户端防火墙碎片。 如果是这种情况，路由器或光标 RADIUS 客户端和服务器 NPS 之间的防火墙可能会自行放弃导致身份验证失败再访问客户端无法连接到该网络的某些碎片。

使用以下步骤来减小用于通过调整中值不超过 1344 年到的网络策略的帧 MTU 属性负载 EAP NPS 的最大大小。

在会员**管理员**，或等效的最低要求完成此过程。

### <a name="to-configure-the-framed-mtu-attribute"></a>若要配置帧 MTU 属性

1. 在 NPS 服务器，在服务器管理器中，单击**工具**，然后单击**网络策略服务器**。 打开 NPS 主机。
 
2. 双击**策略**，单击**网络策略**，然后双击详细信息窗格中的你想要配置的策略。

3. 在策略**属性**对话框中，单击**设置**选项卡。

4. 在**设置**中**RADIUS 属性**，单击**标准**。 在详细信息窗格中，单击**添加**。 **添加标准 RADIUS 属性**对话框中打开。

5. 在**属性**、向下滚动到和单击**帧 MTU**，然后单击**添加**。 **属性信息**对话框中打开。

6. 在**特性值**，键入值等于或小于比**1344 年**。 单击**确定**，单击**关闭**，然后单击**确定**。
