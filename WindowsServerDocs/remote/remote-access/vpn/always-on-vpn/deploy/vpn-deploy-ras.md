# <a name="step-3-configure-the-remote-access-server-for-always-on-vpn"></a>步骤 3： 为始终启用 VPN 配置远程访问服务器

>适用于：Windows Server （半年频道），Windows Server 2016 中，Windows Server 2012 R2、 Windows 10

- [**上一：** 步骤 2：配置服务器基础结构](vpn-deploy-server-infrastructure.md)
- [**上一：** 步骤 4：安装和配置网络策略服务器 (NPS)](vpn-deploy-nps.md)

RRAS 旨在执行以及路由器和远程访问服务器，因为它支持广泛的功能。 此部署的目的，并要求只有一小部分的这些功能： 对 IKEv2 VPN 连接和 LAN 路由的支持。

IKEv2 VPN 隧道协议注释 7296 针对 Internet 工程任务组请求中所述。 IKEv2 的主要优点是它不必完全在基础网络连接中断。 例如，如果连接暂时中断或用户客户端计算机从一个网络移到另一个，IKEv2 时自动还原 VPN 连接重新建立网络连接 — 所有这些都无需用户干预。

配置 RRAS 服务器以支持 IKEv2 连接时禁用未使用的协议，从而降低服务器的安全占用空间。 此外，配置要从静态地址池将地址分配给 VPN 客户端的服务器。 从池或 DHCP 服务器; 一点分配地址但是，使用 DHCP 服务器会增加复杂性与设计和实现了最小优势。

>[!IMPORTANT]
>它十分重要：
>- 在物理服务器上安装两个以太网网络适配器。 如果在 VM 上安装 VPN 服务器，则必须创建两个外部虚拟交换机，一个用于每个物理网络适配器;然后使用每个网络适配器连接到一个虚拟交换机的 VM，创建两个虚拟网络适配器。
>
>- 使用一个网络适配器连接到内部的外围网络和一个网络适配器连接到外部的外围网络中，边缘和内部防火墙之间在外围网络上安装服务器。

>[!WARNING]
>在开始之前，请确保在 VPN 服务器上启用 IPv6。 否则为不能建立连接，并显示一条错误消息。

## <a name="install-remote-access-as-a-ras-gateway-vpn-server"></a>RAS 网关 VPN 服务器作为安装远程访问

在此过程中，单租户 RAS 网关 VPN 服务器作为安装远程访问角色。 有关详细信息，请参阅[远程访问](../../../Remote-Access.md)。

### <a name="install-the-remote-access-role-by-using-windows-powershell"></a>通过使用 Windows PowerShell 安装远程访问角色

1. 打开 Windows PowerShell 作为**管理员**。

2. 输入并运行以下 cmdlet:

   ```powershell
   Install-WindowsFeature DirectAccess-VPN -IncludeManagementTools
   ```

   安装完成后，在 Windows PowerShell 中显示以下消息。

   ```powershell
   | Success | Restart Needed | Exit Code |               Feature Result               |
   |---------|----------------|-----------|--------------------------------------------|
   |  True   |       No       |  Success  | {RAS Connection Manager Administration Kit |
   ```

### <a name="install-the-remote-access-role-by-using-server-manager"></a>通过使用服务器管理器安装远程访问角色

可以使用以下过程来安装使用服务器管理器的远程访问角色。

1. 在 VPN 服务器上，在服务器管理器中，选择**管理**，然后选择**添加角色和功能**。
   
   将打开“添加角色和功能向导”。

2. 在之前的准备开始页上，选择**下一步**。

3. 在选择安装类型页上选择**基于角色或基于功能的安装**选项，然后选择**下一步**。

4. 在选择目标服务器页上选择**从服务器池中选择服务器**选项。

5. 在服务器池下选择本地计算机，然后选择**下一步**。

6. 在选择服务器角色页上，在**角色**，选择**远程访问**，然后**下一步**。

7. 在选择功能页上选择**下一步**。

8. 在远程访问页上，选择**下一步**。

9.  在选择角色服务页面中**角色服务**，选择**DirectAccess 和 VPN (RAS)** 。

   **添加角色和功能向导**对话框随即打开。

11. 在添加角色和功能对话框中，选择**添加功能**然后选择**下一步**。

12. 在 Web 服务器角色 (IIS) 页上，选择**下一步**。

13. 在选择角色服务页上选择**下一步**。

14. 在确认安装选择页上，查看你的选择，然后选择**安装**。

15. 安装完成后，选择**关闭**。

## <a name="configure-remote-access-as-a-vpn-server"></a>配置远程访问作为 VPN 服务器

在本部分中，可以配置允许 IKEv2 VPN 连接，拒绝来自其他 VPN 协议，连接并将颁发的 IP 地址的静态 IP 地址池分配给连接已授权的 VPN 客户端的远程访问 VPN。

1. 在 VPN 服务器上，在服务器管理器中，选择**通知**标志。

2. 在中**任务**菜单中，选择**打开开始向导**

   配置远程访问向导将打开。

   >[!NOTE]
   >配置远程访问向导可能会打开服务器管理器后面。 如果您认为该向导时间太长，若要打开、 移动或最小化服务器管理器，以了解其背后的向导是否。 如果没有，等待向导来初始化。

3. 选择**仅部署 VPN**。

    路由和远程访问 Microsoft 管理控制台 (MMC) 将打开。

4. 右键单击 VPN 服务器，然后选择**配置并启用路由和远程访问**。

   路由和远程访问服务器安装向导将打开。

5. 在路由和远程访问服务器安装向导欢迎，选择**下一步**。

6. 在中**配置**，选择**自定义配置**，然后选择**下一步**。

7. 在中**自定义配置**，选择**VPN 访问**，然后选择**下一步**。

   正在完成该路由和远程访问服务器安装向导将打开。

8. 选择**完成**若要关闭向导，然后选择**确定**以关闭路由和远程访问对话框。

9. 选择**启动服务**启动远程访问。

10. 在远程访问 MMC 中，右键单击 VPN 服务器，然后选择**属性**。

11. 在属性中，选择**安全**选项卡上，执行：

    a. 选择**身份验证提供程序**，然后选择**RADIUS 身份验证**。

    b. 选择**配置**。

       RADIUS 身份验证对话框将打开。

    c. 选择 **添加** 。

       此时将打开添加 RADIUS 服务器对话框。

    d. 在中**服务器名称**，输入你的组织/企业网络上的完全限定域 (FQDN) 的 NPS 服务器。
    
       例如，如果您的 NPS 服务器的 NetBIOS 名称是 NPS1 和你的域名为 corp.contoso.com，则输入**NPS1.corp.contoso.com**。

    e. 在中**共享机密**，选择**更改**。

       更改密码对话框将打开。

    f. 在中**新的机密**，输入文本字符串。

    g. 在中**确认新机密**，输入相同的文本字符串，然后选择**确定**。

    >[!IMPORTANT]
    >保存此文本字符串。 当组织/企业网络上配置 NPS 服务器时，您将添加此 VPN 服务器的 RADIUS 客户端。 在该配置，您将使用此相同的共享的机密，以便 NPS 和 VPN 服务器可以进行通信。

12. 在中**添加 RADIUS 服务器**，查看默认设置为：

    - **Time-out**

    - **初始分数**

    - **Port**

13. 如有必要，更改这些值以匹配你的环境和选择的要求**确定**。

    NAS 是访问的提供权限较大网络某些级别的设备。 使用 RADIUS 基础结构的 NAS 还是 RADIUS 客户端，将连接请求和记帐消息发送到 RADIUS 服务器进行身份验证、 授权和记帐。

14. 查看为设置**记帐提供程序**:

    |                    如果你希望...                     |                                                     则…                                                      |
    |-----------------------------------------------------------|----------------------------------------------------------------------------------------------------------------|
    | 记录远程访问服务器上的远程访问活动 |                               请确保**Windows 记帐**处于选中状态。                               |
    |        NPS 执行 VPN 的帐户服务         | 更改**记帐提供程序**到**RADIUS 记帐**然后将 NPS 配置为记帐提供程序。 |

15. 选择**IPv4**选项卡上，执行：

    a. 选择**静态地址池**。

    b. 选择**添加**若要配置的 IP 地址池。

       静态地址池应包含内部外围网络中的地址。 这些地址是 VPN 服务器，而不是公司网络上的面向内部的网络连接上。

    c. 在中**起始 IP 地址**，输入你想要分配给 VPN 客户端范围内的起始 IP 地址。

    d. 在中**结束 IP 地址**，输入你想要分配给 VPN 客户端，或在范围内的结束 IP 地址**地址数**，输入想要提供的地址数。 如果为此子网使用 DHCP，请确保 DHCP 服务器上配置相应地址排除。

    e. （可选）如果要使用 DHCP，则选择**适配器**，在结果列表中选择连接到内部外围网络的以太网适配器。

16. （可选）*如果你要配置针对 VPN 连接的条件性访问*，从**证书**下拉列表中，在**SSL 证书绑定**，选择 VPN 服务器身份验证。

17. （可选）*如果你要配置针对 VPN 连接的条件性访问*，在 NPS MMC 中，展开**策略\\网络策略**和执行操作： 

    a. 右**连接到 Microsoft 路由和远程访问服务器**网络策略，然后选择**属性**。

    b. 选择**授予访问权限。如果连接请求满足此策略授予访问权限**选项。

    c. 在网络访问服务器类型下选择**远程访问服务器 （VPN 拨号向上）** 从下拉列表。

18. 在路由和远程访问 MMC 中，右键单击**端口**，然后选择**属性**。 
    
    端口属性对话框将打开。

19. 选择**WAN 小型端口 (SSTP)** ，然后选择**配置**。 配置设备-WAN 小型端口 (SSTP) 对话框将打开。

    a. 清除**远程访问连接 （仅入站）** 并**的请求拨号路由连接 （入站和出站）** 复选框。

    b. 选择“确定”  。

20. 选择**WAN 小型端口 (L2TP)** ，然后选择**配置**。 配置设备-WAN 小型端口 (L2TP) 对话框将打开。

    a. 在中**最多端口**，输入要匹配的最大数的同时你想要支持的 VPN 连接的端口号。

    b. 选择“确定”  。

21. 选择**WAN 小型端口 (PPTP)** ，然后选择**配置**。 配置设备-WAN 小型端口 (PPTP) 对话框将打开。

    a. 在中**最多端口**，输入要匹配的最大数的同时你想要支持的 VPN 连接的端口号。

    b. 选择“确定”  。

22. 选择**WAN 小型端口 (IKEv2)** ，然后选择**配置**。 配置设备-WAN 小型端口 (IKEv2) 对话框将打开。

     a. 在中**最多端口**，输入要匹配的最大数的同时你想要支持的 VPN 连接的端口号。

     b. 选择“确定”  。

23. 如果系统提示，请选择**是**以确认重启该服务器，然后选择**关闭**重启服务器。
