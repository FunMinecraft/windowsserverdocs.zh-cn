# 步骤 3： 为始终启用 VPN 配置远程访问服务器

>适用于： Windows Server （半年频道）、 Windows Server 2016、 Windows Server 2012 R2、 Windows 10

& #171; [**以前：** 步骤 2。配置服务器基础结构](vpn-deploy-server-infrastructure.md)<br>
& #187; [**以前：** 步骤 4。安装和配置网络策略服务器 (NPS)](vpn-deploy-nps.md)


RRAS 旨在执行以及路由器和远程访问服务器，因为它支持种类多样的功能。 对于此部署的目的，你需要只有一小部分的这些功能： IKEv2 VPN 连接和 LAN 路由的支持。

IKEv2 VPN 隧道协议的注释 7296 Internet 工程任务强制请求中所述。 IKEv2 主要优势在于它能够规避基础的网络连接中断。 例如，如果连接是暂时丢失，或者如果用户将从一个网络客户端计算机移到另一台，IKEv2 自动还原 VPN 连接时重新建立网络连接 — 所有无需用户干预。

RRAS 服务器配置为支持 IKEv2 连接时禁用未使用的协议，这减少了服务器的安全占用空间。 此外，将服务器配置为从静态地址池将地址分配给 VPN 客户端。 你可以有效地从一个池或 DHCP 服务器; 分配地址但是，使用 DHCP 服务器添加到设计的复杂性，可提供最少的优势。


>[!Important]
>很重要：
>- 在物理服务器上安装两个以太网网络适配器。 如果你要在 VM 上安装的 VPN 服务器，你必须创建两个外部虚拟交换机，一个用于每个物理网络适配器。与每个网络适配器连接到一台虚拟交换机，然后创建用于 VM，两个虚拟网络适配器。
>
>- 连接到外部的外围网络中，一个网络适配器和一个网络适配器连接到内部外围网络外围网络内部防火墙，你边缘之间上安装服务器。


>[!Warning]
>在开始之前，请确保在 VPN 服务器上启用 IPv6。 否则为不能建立的连接，并显示一条错误消息。

## 远程访问安装为 RAS 网关 VPN 服务器

在此过程中，将远程访问角色安装为一个租户 RAS 网关 VPN 服务器。 有关详细信息，请参阅[远程访问](../../../Remote-Access.md)。


### 使用 Windows PowerShell 安装远程访问角色

1. 以**管理员身份**打开 Windows PowerShell。

2. 键入以下命令，然后按**ENTER**:

   `Install-WindowsFeature DirectAccess-VPN -IncludeManagementTools`

   安装完成后，在 Windows PowerShell 中出现以下消息。
    
   | 成功 | 需要重新启动 | 退出代码 | 功能结果                             |
   |---------|----------------|-----------|--------------------------------------------|
   | True    | 否             | 成功   | {RAS 连接管理器管理工具包 |
   ---

### 使用服务器管理器安装远程访问角色

你可以使用以下过程安装远程访问角色使用服务器管理器。

1.  在 VPN 服务器上，在服务器管理器中，单击**管理**，然后单击**添加角色和功能**。 <p>添加角色和功能向导将打开。

2.  之前在你开始页面，请单击**下一步**。

3.  在选择安装类型页上选择**基于角色或基于功能的安装**选项，然后单击**下一步**。

4.  在选择目标服务器页中，选择**选择从服务器池中服务器**选项。

5.  在服务器池下选择本地计算机，然后单击**下一步**。

6.  在选择服务器角色页上，在**角色**，单击**远程访问**，然后**下一步**。

7.  在功能选择页面上，单击**下一步**。

8.  在远程访问页上，单击**下一步**。

9.  在选择的角色服务页面中**角色服务**，单击**DirectAccess 和 VPN (RA)**。<p>打开**添加角色和功能向导**对话框。

10. 在添加角色和功能对话框上，单击**添加功能**，然后单击**下一步**。

11. 在 Web 服务器角色 (IIS) 页上，单击**下一步**。

12. 在选择的角色服务页上，单击**下一步**。

13. 在确认安装选择页面上查看你的选择，然后单击**安装**。

14. 安装完成后，单击**关闭**。

## 为 VPN 服务器配置远程访问

在此部分中，你可以配置远程访问 VPN 允许 IKEv2 VPN 连接、 拒绝连接其他 VPN 协议，并将颁发的 IP 地址静态 IP 地址池分配给连接授权的 VPN 客户端。

1.  在 VPN 服务器上，在服务器管理器中，单击**通知**标志。

2.  在**任务**菜单中，单击**打开开始向导**。<p>配置远程访问向导将打开。 

    >[!NOTE] 
    >配置远程访问向导可能会打开服务器管理器后面。 如果你认为向导时间太长，以打开、 移动或最大程度减少服务器管理器，以了解该向导是否隐藏它。 否则，等待向导来初始化。

3.  单击**部署 VPN 仅**。<p>路由和远程访问 Microsoft 管理控制台 (MMC) 将打开。

4.  右键单击的 VPN 服务器，然后单击**配置并启用路由和远程访问**。<p>路由和远程访问服务器安装向导将打开。

5.  在路由和远程访问服务器安装向导欢迎，单击**下一步**。

6.  在**配置**中，单击**自定义配置**，然后单击**下一步**。

7.  在**自定义配置**中，单击**VPN 访问**，然后单击**下一步**。<p>完成路由和远程访问服务器安装向导将打开。

8.  单击**完成**以关闭向导，然后单击**确定**关闭路由和远程访问对话框。

9.  单击**开始菜单服务**启动远程访问。

10. 在远程访问 MMC 中，右键单击的 VPN 服务器，然后单击**属性**。

11. 在属性中，单击**安全**选项卡，并执行操作：

    a. 单击**身份验证提供程序**，然后单击**RADIUS 身份验证**。
    
    b. 单击**配置**。<p>打开 RADIUS 身份验证对话框。
    
    c. 单击**添加**。<p>打开添加 RADIUS 服务器对话框。
    
    d. 在**服务器名称**中，键入组织/企业网络上的完全限定域名 (FQDN) NPS 服务器的。<p>例如，如果你的域名是 corp.contoso.com NPS 服务器的 NetBIOS 名称是 NPS1，键入**NPS1.corp.contoso.com**。
    
    e. 在**共享机密**，单击**更改**。<p>打开更改密码对话框。
    
    f. 在**新的密码**，键入的文本字符串。
    
    g. 在**确认新的密码**，键入相同的文本字符串，并单击**确定**。

    >[!IMPORTANT] 
    >保存此文本字符串。 在你的组织/企业网络上配置 NPS 服务器时，你将为 RADIUS 客户端添加此 VPN 服务器。 该配置期间，你将使用此相同的共享的密钥，以便 NPS 和 VPN 服务器可以进行通信。

12. 在**添加 RADIUS 服务器**中，查看的默认设置：

    - **超时**
    
    - **初始分数**
    
    - **端口**

13. 如有必要，更改值以匹配你的环境的要求，并单击**确定**。<p>NAS 是提供一定程度的较大的网络访问权限的设备。 使用半径基础结构 NAS 也是 RADIUS 客户端将连接请求和记帐消息发送到 RADIUS 服务器进行身份验证、 和记帐。

14. 查看适用于**记帐提供程序**的设置：

    | 如果你希望...  | 则…             |
    |---------------------|-------------------|
    | 登录远程访问服务器的远程访问活动 | 请确保**Windows 记帐**处于选中状态。      |
    | NPS 执行 vpn 记帐服务   | 将**记帐提供商**更改为**RADIUS 记帐**，并作为记帐提供商配置 NPS。 |
    ---

15. 单击**IPv4**选项卡并执行：

    a. 单击**静态地址池**。
    
    b. 单击**添加**配置的 IP 地址池。<p>静态地址池应包含从内部外围网络地址。 这些地址是面向内部的网络连接的 VPN 服务器上，不是公司网络上。
    
    c. 在**开始菜单的 IP 地址**中，键入你想要分配给 VPN 客户端的范围内的起始 IP 地址。
    
    d. 在**结束 IP 地址**，在你想要分配给 VPN 客户端的范围或**地址数**键入结束的 IP 地址，键入你想要提供的地址的数量。 如果你使用 DHCP 此子网，请确保你在你的 DHCP 服务器上配置一个相应的地址排除。
    
    e. （可选）如果你使用的 DHCP，单击**适配器**，然后在结果列表中，单击连接到内部外围网络的以太网适配器。

16. （可选）*如果你要配置用于 VPN 连接的条件访问*，从**证书**下拉列表中，在**SSL 证书绑定**，选择的 VPN 服务器身份验证。

17. （可选）*如果你要配置用于 VPN 连接的条件访问*，在 NPS MMC 中，展开**Policies\\Network 策略**，并执行操作： 

    a. 右**与 Microsoft 路由和远程访问服务器连接**网络策略，并选择**属性**。
    
    b. 选择**授予访问权限。授予的访问权限，如果连接请求匹配此策略**选项。
    
    c. 在网络访问服务器的类型，从下拉列表中选择**远程访问服务器 （VPN-拨号）** 。

3.  在路由和远程访问 MMC 中，右键单击**端口**，然后单击**属性**。 <p>打开端口属性对话框。

4.  单击**WAN 微型端口 (SSTP)** ，然后单击**配置**。 配置设备中的 WAN 微型端口 (SSTP) 对话框随即打开。

    a. 清除**远程访问连接 （仅入站）** 和**拨号路由连接 （入站和出站）** 复选框。
    
    b. 单击**确定**。

5.  单击**WAN 微型端口 (L2TP)** ，然后单击**配置**。 配置设备中的 WAN 微型端口 (L2TP) 对话框随即打开。

    a. 在**最多端口**，键入端口来匹配你想要支持的同时 VPN 连接的最大数目数。
    
    b. 单击**确定**。

6.  单击**WAN 微型端口 (PPTP)** ，然后单击**配置**。 配置设备中的 WAN 微型端口 (PPTP) 对话框随即打开。

    a. 在**最多端口**，键入端口来匹配你想要支持的同时 VPN 连接的最大数目数。
    
    b. 单击**确定**。
    
7. 单击**WAN 微型端口 (IKEv2)** ，然后单击**配置**。 配置设备中的 WAN 微型端口 (IKEv2) 对话框随即打开。

    a. 在**最多端口**，键入端口来匹配你想要支持的同时 VPN 连接的最大数目数。
    
    b. 单击**确定**。

7.  如果出现提示，请单击**是**以确认重启服务器，然后单击**关闭**重启服务器。
