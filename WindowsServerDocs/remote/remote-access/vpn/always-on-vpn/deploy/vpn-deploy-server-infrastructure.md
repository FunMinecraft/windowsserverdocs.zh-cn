# <a name="step-2-configure-the-server-infrastructure"></a>步骤 2： 配置服务器基础结构

>适用于：Windows Server （半年频道），Windows Server 2016 中，Windows Server 2012 R2、 Windows 10

&#171;  [**上一：** 步骤 1：计划始终打开 VPN 部署](always-on-vpn-deploy-planning.md)<br>
&#187;  [**下一步：** 步骤 3：配置的远程访问服务器的 Always On VPN](vpn-deploy-ras.md)


此步骤中，安装并配置支持的 VPN 所必需的服务器端组件。 服务器端组件包括配置要分发所用的用户、 VPN 服务器和 NPS 服务器证书的 PKI。  您还可以配置 RRAS 以支持建立的 IKEv2 连接和要为 VPN 连接执行授权的 NPS 服务器。

## <a name="configure-certificate-autoenrollment-in-group-policy"></a>在组策略中配置证书自动注册
在此过程中，您配置组策略在域控制器上，以便域成员自动请求用户和计算机证书。 执行此操作允许 VPN 用户请求并检索自动 VPN 连接进行身份验证的用户证书。 同样，此策略允许 NPS 服务器以请求服务器身份验证证书自动。 

手动注册 VPN 服务器上的证书。

>[!TIP]
>非域加入的计算机，请参阅[CA 配置为非域加入计算机](#ca-configuration-for-non-domain-joined-computers)下面。 RRAS 服务器不是已加入域，因为不能使用自动注册来注册 VPN 网关证书。  因此，使用脱机证书请求过程。 


1.  在域控制器上，打开组策略管理。

2.  在导航窗格中，右键单击你的域 (例如，corp.contoso.com)，然后单击 **在此域中创建 GPO 并在此处链接**。

3.  在新建 GPO 对话框中，键入**自动注册策略**，然后单击**确定**。

4.  在导航窗格中，右键单击**自动注册策略**，然后单击**编辑**。

5.  在组策略管理编辑器中，完成以下步骤来配置计算机证书自动注册：

    1.  在导航窗格中，单击**计算机配置\\策略\\Windows 设置\\安全设置\\公钥策略**。

    2.  在细节窗格中，右键单击 **证书服务客户端-自动注册**，然后单击**属性**。

    3.  在证书服务客户端-自动注册属性对话框中，在**配置模型**，单击**已启用**。

    4.  选择 **续订过期证书、 更新未决证书并删除吊销的证书** 并 **更新使用证书模板的证书**。

    5.  单击 **确定**。

6.  在组策略管理编辑器中，完成以下步骤以配置用户证书自动注册：

    1.  在导航窗格中，单击**用户配置\\策略\\Windows 设置\\安全设置\\公钥策略**。

    2.  在细节窗格中，右键单击 **证书服务客户端-自动注册**，然后单击**属性**。

    3.  在证书服务客户端-自动注册属性对话框中，在**配置模型**，单击**已启用**。

    4.  选择 **续订过期证书、 更新未决证书并删除吊销的证书** 并 **更新使用证书模板的证书**。

    5.  单击 **确定**。

    6.  关闭“组策略管理编辑器”。

7.  关闭“组策略管理”。

### <a name="ca-configuration-for-non-domain-joined-computers"></a>非域加入的计算机的 CA 配置

RRAS 服务器不是已加入域，因为不能使用自动注册来注册 VPN 网关证书。  因此，使用脱机证书请求过程。 

1. 在 RRAS 服务器上生成名为的文件**VPNGateway.inf**基于示例证书策略请求在附录中提供 （部分 0） 和自定义了以下项： 

   - 在 [NewRequest] 部分中，将替换为用于使用者名称与所选的 vpn.contoso.com [_客户_] VPN 终结点的 FQDN。

   - 在 [扩展] 部分中，将替换为用于使用者备用名称与所选的 vpn.contoso.com [_客户_] VPN 终结点的 FQDN。

2. 保存或复制**VPNGateway.inf**到所选位置的文件。

3. 从提升的命令提示符，导航到包含的文件夹**VPNGateway.inf**文件，并键入：

   ```
   certreq -new VPNGateway.inf VPNGateway.req
   ```

4. 将新创建的复制**VPNGateway.req**到证书颁发机构服务器或特权访问工作站 (PAW) 的输出文件。 

5. 保存或复制**VPNGateway.req**文件到所选位置上的证书颁发机构服务器或特权访问工作站 (PAW)。

6. 从提升的命令提示符，导航到包含中的上一步和类型创建的 VPNGateway.req 文件的文件夹： 

   ```
   certreq -attrib “CertificateTemplate:[Customer]VPNGateway” -submit VPNgateway.req VPNgateway.cer
   ```

7. 如果系统提示您通过证书颁发机构列表窗口中，选择适当的企业 CA，证书请求提供服务。

8. 将新创建的复制**VPNGateway.cer**到 RRAS 服务器的输出文件。 

9. 保存或复制**VPNGateway.cer** RRAS 服务器上的所选位置的文件。

10. 从提升的命令提示符，导航到包含中的上一步和类型创建的 VPNGateway.cer 文件的文件夹：
   
   ```
   certreq -accept VPNGateway.cer
   ```

11. 运行证书 MMC 管理单元中所述[这里](https://docs.microsoft.com/dotnet/framework/wcf/feature-details/how-to-view-certificates-with-the-mmc-snap-in)选择**计算机帐户**选项。

12. 请确保有效的证书存在用于 RRAS 服务器具有以下属性：

   - **预期的目的：** 服务器身份验证，IP 安全 IKE 中级 

   - **证书模板：** [_客户_] VPN 服务器

#### <a name="example-vpngatewayinf-script"></a>例如：VPNGateway.inf 脚本

这里可以看到用于请求使用带外进程的 VPN 网关证书的证书请求策略的示例脚本。

>[!TIP]
>VPN 提供 IP 工具包中的证书请求策略文件夹下，可以找到一份 VPNGateway.inf 脚本。 只需更新主题和\_继续\_与客户特定的值。

```
[Version] 

Signature="$Windows NT$"

[NewRequest]
Subject = "CN=vpn.contoso.com"
Exportable = FALSE   
KeyLength = 2048     
KeySpec = 1          
KeyUsage = 0xA0      
MachineKeySet = True
ProviderName = "Microsoft RSA SChannel Cryptographic Provider"
RequestType = PKCS10 

[Extensions]
2.5.29.17 = "{text}"
_continue_ = "dns=vpn.contoso.com&"

```

## <a name="create-the-vpn-users-vpn-servers-and-nps-servers-groups"></a>创建 VPN 用户、 VPN 服务器和 NPS 服务器组

在此过程中，可以添加新的 Active Directory (AD) 组包含允许使用 VPN 连接到你的组织网络的用户。

此组具有两个用途：

-   它定义了自动注册用户证书 VPN 需要允许哪些用户。

-   它定义 NPS 授权 VPN 访问的用户。

通过使用自定义组，如果你想要撤销用户的 VPN 访问权限，您可以删除该用户从组。

您还可以添加包含 VPN 服务器的组和包含 NPS 服务器的另一个组。 使用这些组来限制对其成员的证书请求。

>[!NOTE] 
>我们建议 VPN DMA/外围网络中驻留的服务器不是已加入域的。 但是，如果您愿意让已加入域的更好的可管理性的 VPN 服务器 （组策略，备份/监视代理没有本地用户来管理和其他操作），然后添加 AD 组到 VPN 服务器证书模板。

### <a name="configure-the-vpn-users-group"></a>配置 VPN 用户组

1.  在域控制器上，打开 Active Directory 用户和计算机。

2.  右键单击某个容器或组织单位中，单击**新建**然后单击**组**。

3.  在中**组名称**，类型**VPN 用户**，然后单击**确定**。

4.  右键单击**VPN 用户**，然后单击**属性**。

5.  上**成员**选项卡的 VPN 用户属性对话框中，单击**添加**。

6.  在选择用户对话框中，添加需要 VPN 访问权限和单击所有用户**确定**。

7.  关闭“Active Directory 用户和计算机”。

### <a name="configure-the-vpn-servers-and-nps-servers-groups"></a>配置 VPN 服务器和 NPS 服务器组

1.  在域控制器上，打开 Active Directory 用户和计算机。

2.  右键单击某个容器或组织单位中，单击**新建**然后单击**组**。

3.  在中**组名称**，类型**VPN 服务器**，然后单击**确定**。

4.  右键单击**VPN 服务器**，然后单击**属性**。

5.  上**成员**选项卡的 VPN 服务器属性对话框中，单击**添加**。

6.  单击**对象类型**，选择**计算机**复选框，然后单击**确定**。

7.  在中**输入要选择的对象名称**、 键入 VPN 服务器的名称，然后单击**确定**。

8.  单击**确定**以关闭 VPN 服务器属性对话框。

9.  NPS 服务器组重复前面的步骤。

10. 关闭“Active Directory 用户和计算机”。

## <a name="create-the-user-authentication-template"></a>创建用户身份验证模板

在此过程中，配置自定义客户端-服务器身份验证模板。 此模板是必需的因为你想要提高通过选择升级后兼容性级别的证书的整体安全性并选择 Microsoft 平台加密提供程序。 此最后一个更改可让你在客户端计算机上使用 TPM 保护的证书。 有关 TPM 的概述，请参阅[受信任的平台模块技术概述](https://docs.microsoft.com/windows/device-security/tpm/trusted-platform-module-overview)。

**过程：**

1.  在 CA 上，打开证书颁发机构。

2.  在导航窗格中，右键单击 **证书模板**，然后单击 **管理**。

3.  在证书模板控制台中，右键单击 **用户**，然后单击 **复制模板**。

    >[!WARNING]
    >不要单击**Apply**或**确定**在步骤 10 之前的任何时间。  如果在输入所有参数之前单击这些按钮，很多选择将成为固定且不再可编辑。 例如，在**加密**选项卡上，如果_旧版加密存储提供程序_显示在提供程序类别字段中，它将被禁用，防止任何后续更改。 唯一的备用方法是删除该模板并重新创建它。  

4.  在新模板的属性对话框中，在 **常规** 选项卡上，完成以下步骤：

    1.  在中**模板显示名称**，类型**VPN 用户身份验证**。

    2.  清除**在 Active Directory 中的发布证书**复选框。

5.  上 **安全** 选项卡上，完成以下步骤：

    1.  单击**添加**。

    2.  在选择用户、 计算机、 服务帐户或组对话框中，键入**VPN 用户**，然后单击**确定**。

    3.  在中**组或用户名**，单击**VPN 用户**。

    4.  在中**VPN 用户的权限**，选择**注册**并**自动注册**中的复选框**允许**列。

       >[!TIP]
       >请确保选中了读取复选框。 换句话说，需要读取权限注册。 

    5.  在**组或用户名**，单击**Domain Users**，然后单击**删除**。

6.  上**兼容性**选项卡上，完成以下步骤：

    1.  在中**证书颁发机构**，单击**Windows Server 2012 R2**。

    2.  上**产生的更改**对话框中，单击**确定**。

    3.  在中**证书接收人**，单击**Windows 8.1 / Windows Server 2012 R2**。

    4.  上**产生的更改**对话框中，单击**确定**。

7.  上**请求处理**选项卡上，清除**允许导出私钥**复选框。

8.  上**加密**选项卡上，完成以下步骤：

    1.  在中**提供程序类别**，单击**密钥存储提供程序**。

    2.  单击**请求必须使用以下提供程序之一**。

    3.  选择**Microsoft 平台加密提供程序**复选框。

9.  上**使用者名称**选项卡上，如果你没有清除所有的用户帐户上列出的电子邮件地址**在使用者名称中的包含电子邮件名称**并**电子邮件名称**复选框。

10. 单击 **确定**保存 VPN 用户身份验证证书模板。

11. 关闭证书模板控制台。

12. 在证书颁发机构管理单元中的导航窗格中，右键单击 **证书模板**，单击**新建**，然后单击 **颁发的证书模板**。

13. 单击 **VPN 用户身份验证**，然后单击 **确定**。

14. 关闭证书颁发机构管理单元中。

## <a name="create-the-vpn-server-authentication-template"></a>创建 VPN 服务器身份验证模板

在此过程中，可以为你的 VPN 服务器配置新的服务器身份验证模板。 添加 IP 安全 (IPsec) IKE 中间应用程序策略允许筛选器证书服务器，如果具有服务器身份验证提供了多个证书扩展密钥用法。

>[!IMPORTANT] 
>VPN 客户端从公共 Internet 访问此服务器，因为使用者和备用名称是不同于内部服务器名称。 因此，不能自动注册此证书在 VPN 服务器上。

**先决条件：**<p>
已加入域的 VPN 服务器

**过程：** 

1.  在 CA 上，打开证书颁发机构。

2.  在导航窗格中，右键单击**证书模板**，然后单击 **管理**。

3.  在证书模板控制台中，右键单击 **RAS 和 IAS 服务器**，然后单击 **复制模板**。

4.  在新模板的属性对话框中，在 **常规** 选项卡上，在**模板显示名称**，输入 VPN 服务器的描述性名称，例如， **VPN 服务器身份验证**或**RADIUS 服务器**。

5.  上 **扩展** 选项卡上，完成以下步骤：

    1.  单击 **应用程序策略**，然后单击 **编辑**。

    2.  在中**编辑应用程序策略扩展**对话框中，单击 **添加**。

    3.  上**添加应用程序策略**对话框中，单击**IP 安全 IKE 中级**，然后单击**确定**。<p>添加 IP 安全 IKE 中级到 EKU 可帮助在多个服务器身份验证证书，VPN 服务器存在的情况下。 如果使用 IP 安全 IKE 中级，IPSec 仅使用这两个 EKU 选项使用的证书。 如果没有，IKEv2 身份验证可能会失败，错误 13801:IKE 身份验证的凭据是不可接受。

    4.  单击**确定**回到 **新模板的属性** 对话框。

6.  上 **安全** 选项卡上，完成以下步骤：

    1.  单击**添加**。

    2.  上**选择用户、 计算机、 服务帐户或组**对话框中，键入**VPN 服务器**，然后单击**确定**。

    3.  在中**组或用户名**，单击**VPN 服务器**。

    4.  在中**VPN 服务器的权限**，选择**注册**中的复选框**允许**列。

    5.  在中**组或用户名**，单击**RAS 和 IAS 服务器**，然后单击**删除**。

7.  上**使用者名称**选项卡上，完成以下步骤：

    1.  单击**在请求中提供**。

    2.  上**证书模板**警告对话框中，单击**确定**。

8.  （可选）*如果你要配置针对 VPN 连接的条件性访问*，单击**请求处理**选项卡，然后单击**允许导出私钥**以将其选中。

9.  单击 **确定**保存 VPN 服务器证书模板。

10. 关闭证书模板控制台。

11. 在证书颁发机构管理单元中的导航窗格中，右键单击 **证书模板**，单击**新建**，然后单击 **颁发的证书模板**。

12. 重新启动证书颁发机构服务。

13. 选择在上面，步骤 4 中选择的名称，然后单击 **确定**。

14. 关闭证书颁发机构管理单元中。

## <a name="create-the-nps-server-authentication-template"></a>创建 NPS 服务器身份验证模板

若要创建的第三个和最后一个证书模板是 NPS 服务器身份验证模板。 NPS 服务器身份验证模板是先前在本部分中创建的 NPS 服务器组中保护的 RAS 和 IAS 服务器模板的简单的复制。 

将配置为自动注册此证书。

**过程：**

1.  在 CA 上，打开证书颁发机构。

2.  在导航窗格中，右键单击**证书模板**，然后单击 **管理**。

3.  在证书模板控制台中，右键单击 **RAS 和 IAS 服务器**，然后选择**复制模板**。

4.  在新模板的属性对话框中，在 **常规** 选项卡上，在**模板显示名称**，类型**NPS 服务器身份验证**。

5.  上 **安全** 选项卡上，完成以下步骤：

    1.  单击**添加**。

    2.  上**选择用户、 计算机、 服务帐户或组**对话框中，键入**NPS 服务器**，然后单击**确定**。

    3.  在中**组或用户名**，单击**NPS 服务器**。

    4.  在中**NPS 服务器的权限**，选择**注册**并**自动注册**中的复选框**允许**列。

    5.  在中**组或用户名**，单击**RAS 和 IAS 服务器**，然后单击**删除**。

6.  单击 **确定**来保存 NPS 服务器证书模板。

7.  关闭证书模板控制台。

8.  在证书颁发机构管理单元中的导航窗格中，右键单击 **证书模板**，单击**新建**，然后单击 **颁发的证书模板**。

9.  单击 **NPS 服务器身份验证**，然后单击 **确定**。

10. 关闭证书颁发机构管理单元中。

## <a name="enroll-and-validate-the-user-certificate"></a>注册和验证的用户证书

由于您将组策略自动注册用户证书，你只需更新策略，并且 Windows 10 自动注册正确的证书的用户帐户。 随后可以验证证书控制台中的证书。

**过程：**

1.  作为的成员登录到已加入域的客户端计算机**VPN 用户**组。

2.  按 Windows 键 + R，类型**gpupdate /force**，然后按 Enter。

3.  在开始菜单中，键入**certmgr.msc**，然后按 Enter。

4.  在证书管理单元中下,**个人**，单击**证书**。 详细信息窗格中显示你的证书。

5.  右键单击该证书具有当前域用户名，然后依次**打开**。

6.  上**常规**选项卡上，确认在列出的日期**有效起始日期**是今天的日期。 如果不是这样，你可能会选择错误的证书。

7.  单击**确定**，并关闭证书管理单元中。

## <a name="enroll-and-validate-the-server-certificates"></a>注册并验证服务器证书

与不同的用户证书，你必须手动注册 VPN 服务器的证书。 已注册后，它通过使用相同的过程用于用户证书进行验证。 用户证书，如 NPS 服务器自动注册其身份验证证书，因此您需要做的就是对其进行验证。

>[!NOTE] 
>您可能需要重新启动 VPN 和 NPS 服务器，以允许他们更新其组成员身份才能完成这些步骤。

### <a name="enroll-and-validate-the-vpn-server-certificate"></a>注册并验证 VPN 服务器证书

1.  在 VPN 服务器的开始菜单上，键入**certlm.msc**，然后按 Enter。

2.  右键单击**个人**，单击**的所有任务**，然后单击**申请新证书**以启动证书注册向导。

3.  在开始之前页上，单击**下一步**。

4.  在选择证书注册策略页上，单击**下一步**。

5.  在申请证书页上，单击 VPN 服务器以选择它旁边的复选框。

6.  在 VPN 服务器复选框，单击**详细信息是必需的**打开证书属性对话框并完成以下步骤：

    1.  单击**使用者**选项卡上，单击**公用名**下**使用者名称**，在**类型**。

    2.  下**使用者名称**，在**值**，输入的外部域的客户端用于连接到 VPN 例如 vpn.contoso.com，并单击名称**添加**。

    3.  下**备用名称**，在**类型**，单击**DNS**。

    4.  下**备用名称**，在**值**，输入的所有客户端用于连接到 VPN，例如服务器名称、 vpn.contoso.com、 vpn、 132.64.86.2。

    5.  单击**添加**后输入的每个名称。

    6.  完成后，单击**确定**。

7.  单击**注册**。

8.  单击 **“完成”**。

9.  在证书管理单元中下,**个人**，单击**证书**。<p>详细信息窗格中显示你列出的证书。

10. 右键单击该证书具有 VPN 服务器的名称，然后依次**打开**。

11. 上**常规**选项卡上，确认在列出的日期**有效起始日期**是今天的日期。 如果不是这样，您可能选择不正确的证书。

12. 上**详细信息**选项卡上，单击**增强型密钥用法**，并验证**IP 安全 IKE 中级**并**服务器身份验证**在列表中显示。

13. 单击**确定**关闭该证书。

14. 关闭证书管理单元中。

### <a name="validate-the-nps-server-certificate"></a>验证 NPS 服务器证书

1.  重新启动 NPS 服务器。

2.  在 NPS 服务器的开始菜单上，键入**certlm.msc**，然后按 Enter。

3.  在证书管理单元中下,**个人**，单击**证书**。<p>详细信息窗格中显示你列出的证书。

4.  右键单击该证书具有 NPS 服务器的名称，然后依次**打开**。

5.  上**常规**选项卡上，确认在列出的日期**有效起始日期**是今天的日期。 如果不是这样，您可能选择不正确的证书。

6.  单击**确定**关闭该证书。

7.  关闭证书管理单元中。
