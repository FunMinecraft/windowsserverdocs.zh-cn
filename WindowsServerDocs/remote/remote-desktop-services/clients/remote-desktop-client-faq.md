## 连接、 网关和网络

### 为什么无法连接使用远程桌面？

下面是一些可能的解决方案的尝试连接到远程电脑时可能遇到的常见问题。 如果这些解决方案不起作用，你可以在[Microsoft 社区网站](https://go.microsoft.com/fwlink/p/?LinkId=242079)上找到更多帮助。

- **找不到远程电脑。** 请确保具有正确的电脑名称，然后进行检查以查看是否正确输入该名称。 如果仍无法连接，请尝试使用远程电脑的 IP 地址，而不电脑名称。
- **没有网络问题。** 请确保你具有 internet 连接。 
- **可能通过防火墙阻止使用远程桌面端口。** 如果你使用 Windows 防火墙，请按照下列步骤：

   1. 打开 Windows 防火墙。 
   2. 单击**允许应用或通过 Windows 防火墙的功能**。 
   3. 单击**更改设置**。 系统可能会要求你提供管理员密码或确认你的选择。
   4. 在**允许的应用和功能**，选择**远程桌面**，然后点击或单击**确定**。

   如果你使用其他防火墙，请确保远程桌面 (通常为 3389) 端口已打开。
- **远程连接可能不会设置远程电脑上。** 若要解决此问题，滚动设置回[如何设置电脑设置为远程桌面？](#how-do-i-set-up-a-pc-for-remote-desktop)本主题中的问题。
- **远程电脑可能仅允许电脑连接已设置的网络级别身份验证。** 
- **远程电脑可能已关闭。** 你无法连接到已关闭、 睡眠，电脑或休眠状态，因此请确保设置睡眠和休眠远程电脑上的设置为**从不**（休眠不可用所有电脑上。）。

### 为什么无法找到或连接到我的电脑？
检查以下：
- 是的电脑上并且唤醒状态？
- 你没有输入的适当的名称或 IP 地址？

   > [!IMPORTANT]
   > 使用电脑名称要求网络正确通过 DNS 名称解析。 在许多家庭网络中，你必须使用的 IP 地址而不是主机名连接。
- 位于不同的网络上的电脑？ 未配置电脑以使外部通过连接？  有关帮助，请查看[允许你从网络外部的电脑访问](remote-desktop-allow-outside-access.md)。
- 你是否连接到支持的 Windows 版本？ 

   > [!NOTE]
   > 没有第三方软件的情况下不支持 Windows XP 家庭版、 Windows Media Center 版本、 Windows Vista 家庭版和 Windows 7 家庭版或简易版。

### 为什么我无法登录到远程电脑？
如果你可以看到远程电脑的登录屏幕，但无法登录，你可能不已添加到远程桌面用户组或使用远程电脑上的管理员权限的任何组。 要求系统管理员来为你执行此操作。

### 公司网络支持的连接方法？
如果你想要访问 office 桌面从你的公司网络之外，你的公司必须提供一种远程访问。 远程桌面客户端当前支持以下功能：

- 终端服务器网关或远程桌面网关
- 远程桌面 Web 访问
- VPN （通过 iOS 内置 VPN 选项）

### VPN 不起作用

VPN 问题可以有几个原因。 第一步是验证 VPN 的工作电脑或 Mac 计算机与在同一网络上。 如果你不能与电脑或 Mac 测试，你可以尝试访问公司 intranet 网页使用你设备的浏览器。

若要查看其他事项：
- **3g 网络阻止或损坏 VPN。** 有多个世界中提供的 3g 商合作为阻止或损坏 3g 流量看起来。 验证 VPN 连接正常工作的一分钟内。
- **L2TP 或 PPTP Vpn。** 如果在 VPN 使用 L2TP 或 PPTP，请设置发送所有流量都为 ON 在 VPN 配置。
- **VPN 配置错误。** 错误配置的 VPN 服务器可以是 VPN 连接永远不会处理或一段时间后停止工作的原因。 请确保测试 ios 设备的 web 浏览器或在电脑或 Mac 同一网络上，如果发生这种情况。

### 如何测试是否 VPN 正常运行？
验证你的设备上启用了 VPN。 你可以通过在内部网络上转到网页或使用 web 服务该服务仅可通过 VPN 测试 VPN 连接。

### 如何配置 L2TP 或 PPTP VPN 连接？
如果在 VPN 使用 L2TP 或 PPTP，请确保**将所有流量都发送**到**等**中设置的 VPN 配置。

## Web 客户端

### 可以使用哪个浏览器？

Web 客户端支持 Microsoft Edge、 Internet Explorer 11，Mozilla Firefox (v55.0 及更高版本)，旅行和 Google Chrome。

### 可以使用哪些电脑访问 web 客户端？

Web 客户端支持 Windows、 macOS、 Linux 以及 ChromeOS。 目前不支持移动设备。

### 可以在远程桌面部署而无需网关使用 web 客户端？

不需要。 客户端需要远程桌面网关连接。 不知道这意味着？ 你的管理员询问它。

### 远程桌面 web 客户端替换为远程桌面 Web 访问页面？

不需要。 远程桌面 web 客户端托管在不同的远程桌面 Web 访问网页的 URL。 你可以使用 web 客户端或 Web 访问页在浏览器中查看远程资源。

### 可以在其他网页中嵌入 web 客户端？

目前不支持此功能。

## 监视器、 音频和鼠标

### 如何使用我的监视器的所有？
若要使用两个或多个屏幕，请执行以下操作：

1. 右键单击你想要启用多个屏幕，远程桌面，然后单击**编辑**。
2. 启用**使用的所有监视器**和**全屏显示**。

### 是双向的声音支持？
远程桌面客户端不支持声音上游 （从客户端到服务器，麦克风）。

### 如果不会播放声音，我可以做什么？
登录开会话 （不只是断开连接，一直注销），然后再次登录。

## Mac 客户端-硬件问题
### 视网膜分辨率是否受支持？
是，远程桌面客户端支持视网膜分辨率。

### 如何启用辅助右键单击？
为了使在有三个选项打开会话内右键单击使用：

- 标准电脑两个按钮 USB 鼠标
- Apple 幻鼠标： 若要启用，右键单击，单击**系统首选项**中扩展坞中，单击**鼠标**，，然后启用**辅助单击**。
- Apple 幻触控或 MacBook 触控： 以启用右键单击，单击**系统首选项**扩展坞中，单击**鼠标**，然后启用**辅助单击**。

### AirPrint 是否受支持？
否，远程桌面客户端不支持 AirPrint。 （这是适用于适用于 Mac 和 iOS 客户端。

### 为什么在会话中显示不正确的字符？
如果你使用的国际键盘，你可能会看到问题在会话中显示的字符是其中的字符执行匹配 Mac 键盘上键入。

这可以在以下情况下发生：

- 你正在使用远程会话无法识别的键盘。 当远程桌面不识别键盘时，它默认为上次使用与远程电脑的语言。
- 要连接到远程电脑上已断开连接会话和远程电脑使用不同的键盘语言与语言当前正在使用。

可以通过手动设置远程会话的键盘语言来解决此问题。 请参阅下一节中的步骤。

### 语言设置如何影响键盘在远程会话中？
有多种类型的 Mac 键盘布局。 其中某些是 Mac 特定布局或为其可能不可用版本的 Windows 的远程处理到上一个精确匹配的自定义布局。 远程会话将键盘映射到的最佳匹配的远程电脑上可用的键盘语言。 

如果 Mac 键盘布局设置为应只需工作的电脑版本语言键盘 （例如，法语 – PC） 应正确映射所有密钥和键盘。

如果 Mac 键盘布局设置为键盘 （例如，法语） 的 Mac 版本远程会话将映射你为法语的电脑版本。 在使用 OSX 上使用的 Mac 键盘快捷键将不起作用远程 Windows 会话中。

如果你的键盘布局设置为的变体的语言 （例如，加拿大法语） 和远程会话无法将你映射到该精确的变体，远程会话将映射你为最接近的语言 （例如，法语）。 在使用 OSX 上使用的 Mac 键盘快捷键将不起作用远程 Windows 会话中。

如果键盘布局设置为远程会话不能完全匹配的布局，在远程会话将默认为你提供与该电脑上次使用的语言。 在此情况下，或者在需要更改以匹配 Mac 键盘，在远程会话的语言的情况下你可以手动到与你想要使用如下所示的一个最接近匹配的语言在远程会话中设置键盘语言。

请按照以下说明来更改在远程桌面会话内的键盘布局：

**在 Windows 10 或 Windows 8:**

1. 从在远程会话中，打开区域和语言。 单击**开始菜单 > 设置 > 时间和语言**。 打开**区域和语言**。
2. 添加你想要使用的语言。 然后关闭区域和语言窗口。
3. 现在，在远程会话中，你将看到语言之间切换的能力。 （在右侧远程会话，时钟附近。）单击你想要切换到 （如**Eng**) 的语言。

你可能需要关闭并重新启动应用程序当前使用键盘更改才能生效。


## 特定错误

### 为什么要获取"权限不足"错误？
不允许你访问你想要连接到的会话。 最可能的原因是你尝试连接到管理会话。 只有管理员才可以连接到控制台。 验证关闭的控制台交换机中的远程桌面的高级设置。 如果这不是问题的源，请联系你的系统管理员获取更多帮助。

### 客户端为什么说，如果没有任何 CAL？
当远程桌面客户端连接到远程桌面的服务器时，服务器会发出远程桌面服务客户端访问许可证 (RDS CAL) 存储的客户端。 每当客户端连接时，它将重新使用其 RDS CAL 和服务器不会发出另一个许可证。 如果在设备上 RDS CAL 丢失或损坏，服务器将发出另一个许可证。 当达到最大许可设备数服务器不会发出新 RDS Cal。 与网络管理员联系以寻求帮助。

### 为什么未收到"访问被拒绝"错误？
连接尝试时生成通过远程桌面网关并不正确的凭据的结果"的访问权限被拒绝"错误。 验证用户名和密码。 如果连接处理之前并最近发生错误，你可能已更改 Windows 用户帐户密码，尚未它尚未更新中的远程桌面设置。

### "RPC 错误 23014"或"错误 0x59e6"意味着会如何？
如果**RPC 错误 23014**或**错误 0x59E6 等待几分钟后重试**，RD 网关服务器已达到的最大活动连接数。 在 RD 网关上运行的最大连接数不同具体取决于 Windows 版本： Windows Server 2008 R2 标准版实现限制到 250 连接数。 Windows Server 2008 R2 Foundation 实现限制为 50 连接数。 所有其他 Windows 实现允许无限的数量的连接。

### "无法解析 NTLM 质询"错误是什么意思？
远程电脑上的配置不正确导致此错误。 请确保远程电脑上的 RDP 安全级别设置设置为"客户端兼容。" （与你的系统管理员需要帮助执行此操作。）

### 不"TS_RAP 不允许连接到给定主机"，确定意味着？
网关服务器上的资源授权策略停止你连接到远程电脑的用户名称时，会发生此错误。 这可以在以下情况下发生：

- 远程电脑名称是网关的名称相同。 然后，当你尝试连接到远程电脑，连接会转至网关相反，这可能不具有访问权限。 如果你需要连接到该网关，不要使用外部网关名称作为计算机名称。 改为使用"localhost"的 IP 地址 (127.0.0.1) 或内部服务器名称。
- 你的用户帐户不是远程访问的用户组的成员。
