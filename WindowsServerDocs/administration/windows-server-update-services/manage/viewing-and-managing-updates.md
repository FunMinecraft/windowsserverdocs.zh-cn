# <a name="viewing-and-managing-updates"></a>查看和管理更新

>适用于：Windows 服务器 （半年频道），Windows Server 2016 中，Windows Server 2012 R2、 Windows Server 2012

可以使用 WSUS 控制台来查看和管理更新。

## <a name="viewing-updates"></a>查看更新
上**更新**页上，您可以执行以下操作：

-   查看更新。 更新概述显示已与你的 WSUS 服务器从更新源中同步并可供批准的更新。

-   筛选更新。 默认视图中，您可以筛选更新的批准状态和安装状态。 默认设置为为未批准的更新的所需的某些客户端或某些客户端上已安装失败。 可以通过更改的批准状态和安装状态筛选器，然后单击更改此视图**刷新**。

-   创建新的更新视图。 在中**操作**窗格中，单击**新更新视图**。 您可以筛选按分类、 产品、 针对其它们具有已批准的组和同步日期的更新。 可以通过单击标题栏中相应的列标题对列表进行排序。

-   搜索更新。 按标题、 说明、 知识库文章或更新的 Microsoft 安全响应中心号，可以搜索单个更新或组的更新。

-   查看详细信息、 状态和每个更新的修订版本历史记录。

-   批准和拒绝更新。

#### <a name="to-view-updates"></a>若要查看更新

1.  在 WSUS 管理控制台中，展开更新节点，然后单击所有更新。

2.  默认情况下，使用其标题、 分类、 安装/未适用百分比和批准状态显示更新。 如果你想要显示详细的或不同的更新属性，右键单击列标题栏并选择相应的列。

3.  若要按不同的条件，如下载状态、 标题、 分类、 发布日期或批准状态，请单击相应的列标题。

#### <a name="to-filter-the-list-of-updates-displayed-on-the-updates-page"></a>若要筛选显示在更新页上更新的列表

1.  在 WSUS 管理控制台中，展开**更新**，然后单击**的所有更新**。

2.  在中心窗格中下, 一步**审批**，选择所需的批准状态和到下的一步**状态**选择所需的安装状态。 单击**刷新**。

#### <a name="to-create-a-new-update-view-on-wsus"></a>若要创建新的更新视图上 WSUS

1.  在 WSUS 管理控制台中，展开**更新**，然后单击**的所有更新**。

2.  在中**操作**窗格中，单击**新更新视图**。

3.  在中**添加更新视图**窗口下**步骤 1： 选择属性**，选择您需要更新视图进行筛选的属性：

    -   选择更新是在要作为筛选依据更新属于特定分类中，为一个或多个更新分类。

    -   选择更新是针对特定产品要作为筛选依据的一个或多个产品或产品系列更新。

    -   选择要筛选一个或多个计算机组批准更新的特定组批准更新。

    -   选择要作为筛选依据在特定时间同步更新在特定时间段内已同步更新。

    -   选择更新是 WSUS 更新对 WSUS 更新进行筛选。

4.  下**步骤 2： 编辑属性**，请单击带下划线的词来选取所需的值。

5.  下**步骤 3:指定一个名称**，为您的新视图提供一个名称。

6.  单击 **“确定”** 。

您的新视图将显示在更新下的树视图窗格中。 它将显示，如处于选中状态时的中心窗格中的标准视图。

#### <a name="to-search-for-an-update"></a>搜索更新

1.  选择**更新**节点 （或其下的任何节点）。

2.  在中**操作**窗格中，单击**搜索**。

3.  在中**搜索**窗口，然后在**更新**选项卡上，输入您的搜索条件。 可以使用中的文本**标题**，**说明**，并**Microsoft 知识库 (KB) 文章编号**字段。 其中每一项是属性上列出**详细信息**更新属性中的选项卡。

#### <a name="to-view-the-properties-for-an-update"></a>若要查看更新属性

1.  在 WSUS 管理控制台中，展开**更新**，然后单击**的所有更新**。

2.  在更新列表中，单击你想要查看的更新。

3.  在下部窗格中，你将看到不同的属性部分：

    -   标题栏会显示标题的更新;例如，安全更新的 Windows Media Player 9 (KB911565)。

    -   状态部分将显示安装状态 （它需要在其安装的计算机，计算机在其中已安装但出现错误，计算机在其其已安装或不适用和未报告的计算机的更新更新的状态），以及 （KB 和 MSRC 编号发布日期等） 的一般信息。

    -   描述部分显示更新的简短说明。

    -   更多详细信息部分显示以下信息：

        -   Update （它是可移除，请求重新启动，需要用户输入，或必须以独占方式安装） 安装行为。

        -   更新是否有 Microsoft 软件许可条款

        -   更新应用产品

        -   取代此更新的更新

        -   此更新所取代的更新

        -   更新支持的语言

        -   更新 ID

请注意，您可以在一次只有一个更新执行此过程。 如果选择多个更新，仅在列表中的首个更新将显示在**属性**窗格。

## <a name="managing-updates-with-wsus"></a>使用 WSUS 管理更新
更新用于更新或软件的计算机上安装提供了完整的文件。 每个更新都可在 Microsoft Update 是由两个组件构成：

-   元数据：提供有关更新的信息。 例如，元数据提供的属性的更新，从而使你可找到适合的更新是有用的信息。 元数据还包括 Microsoft 软件许可条款。 下载的更新的元数据包是通常比实际更新文件包小得多。

-   更新文件：所需的计算机上安装更新的实际文件。

将更新同步到 WSUS 服务器时，元数据和更新文件存储在两个不同的位置。 元数据存储在 WSUS 数据库中。 更新文件可以存储在 WSUS 服务器上或 Microsoft Update 在服务器上，具体取决于同步选项的配置方式。 如果您选择 Microsoft 更新服务器上存储更新文件，在同步; 时，将下载仅元数据你批准通过 WSUS 控制台中，更新，然后客户端计算机获取更新文件直接从 Microsoft 更新在安装时。 有关存储更新选项的详细信息，请参阅部分[1.3。选择 WSUS 存储策略](../plan/plan-your-wsus-deployment.md#13-choose-a-wsus-storage-strategy)的步骤 1:WSUS 部署指南中为你的 WSUS 部署做好准备。

您将设置和运行同步过程中，添加计算机和计算机组和部署上定期更新。 以下列表提供了可能会在更新的计算机使用 WSUS 时需要执行的常规任务的示例。

-   确定整体更新管理计划根据您的网络拓扑和带宽、 公司需求和组织结构。

    -   是否要设置的 WSUS 服务器层次结构和层次结构的组织结构。

    -   若要创建，哪台计算机进行分组以及如何将计算机分配到其 （服务器端或客户端的目标）。

    -   要用于更新元数据 （例如，Windows 内部数据库，SQL Server） 的数据库。

    -   是否自动，以及在什么时候应该同步更新。

-   设置同步选项，如更新源、 产品和更新分类、 语言、 连接设置、 存储位置和同步计划。

-   获取更新和关联的元数据在 WSUS 服务器上通过同步从 Microsoft 更新或上游 WSUS 服务器。

-   批准或拒绝更新。 必须是允许用户安装这些更新本身 （如果它们是其客户端计算机上的本地管理员） 的选项。

-   配置自动审批。 此外可以配置想要启用自动审批对现有更新的修订的还是手动批准修订。 如果你选择手动批准修订版本，你的 WSUS 服务器将继续使用较旧版本，直到您手动批准的新修订版本。

-   检查更新的状态。 可以查看更新状态、 打印状态报表，或配置为常规状态报告的电子邮件。

## <a name="update-products-and-classifications"></a>更新产品和分类
在 Microsoft Update 上可用的更新进行区分的产品 （或产品系列） 和分类。

产品是指特定版本的操作系统或应用程序，例如，Windows Server 2012。 产品家族是指从中派生单个产品的基本操作系统或应用程序。 产品系列的一个示例是 Microsoft Windows，Windows Server 2012 的成员。 可以选择的产品或产品系列为其所需服务器以同步更新。 您可以指定产品系列或系列中的各个产品。 选择任何产品或产品系列将获取产品的当前和未来版本的更新。

更新分类表示更新的类型。 对于任何给定的产品或产品系列，更新可能会在多个更新分类 （例如，Windows 7 系列关键更新和安全更新） 之间可用。 下表列出了更新分类。

| 更新分类  | 描述   |
|--|--|
|关键更新|广泛发布的特定问题解决重要的、 非安全相关的 bug 修补程序。|
|定义更新|更新病毒或其他定义文件。|
|Drivers|设计为支持新硬件的软件组件。|
|功能包|新功能的版本中，通常回滚到下一版本的产品。|
|安全更新|广泛发布的特定产品，解决安全问题的修补程序。|
|Service Pack|所有修补程序、 安全更新、 关键更新的累积集合和产品发布以来创建的更新。 Service pack 还可能包含有限的数量的客户请求的设计更改或功能。|
|工具|实用程序或功能来协助完成任务或一组任务。|
|更新汇总|一组累积的修补程序、 安全更新、 关键更新和其他更新，它们打包在一起以便于部署。 汇总通常针对特定区域中的，如安全性或特定组件，如 Internet 信息服务 (IIS)。|
|更新|广泛发布的特定问题解决非关键的、 与安全相关的 bug 修补程序。|
