# <a name="manage-the-local-server-and-the-server-manager-console"></a>Manage the Local Server and the Server Manager Console

>适用于：Windows 服务器 （半年频道），Windows Server 2016 中，Windows Server 2012 R2、 Windows Server 2012

在 Windows Server 中，服务器管理器可用于管理本地服务器 （如果您运行服务器管理器在 Windows 服务器上，而不是在基于 Windows 的客户端操作系统） 和运行 Windows Server 2008 和 Windows 的较新版本的远程服务器服务器操作系统。

**本地服务器**页在服务器管理器将显示服务器属性、 事件、 服务和性能计数器数据，以及最佳做法分析器 (BPA) 结果针对本地服务器。 事件、服务、BPA 和性能磁贴的功能与它们在角色和服务器组页面的功能相同。 有关配置这些磁贴所显示数据的详细信息，请参阅[查看和配置性能、事件和服务数据](view-and-configure-performance-event-and-service-data.md)和[运行最佳做法分析器扫描并管理扫描结果](run-best-practices-analyzer-scans-and-manage-scan-results.md)。

菜单命令和在服务器管理器控制台标题栏中的设置对服务器池中的所有服务器执行全局都应用，让你使用服务器管理器来管理整个服务器池。

本主题包含以下部分。

-   [关闭本地服务器](#BKMK_shutdown)

-   [配置服务器管理器属性](#BKMK_props)

-   [管理服务器管理器控制台。](#BKMK_managesm)

-   [自定义工具菜单中显示的工具](#BKMK_tools)

-   [管理角色主页上的角色](#BKMK_roles)

## <a name="BKMK_shutdown"></a>关闭本地服务器
**任务**本地服务器中的菜单**属性**磁贴允许在本地服务器上，启动 Windows PowerShell 会话中打开**计算机管理**mmc 管理单元中，或打开mmc 管理单元的角色或功能的安装在本地服务器上。 你还可以使用 **“任务”** 菜单中的 **“关闭本地服务器”** 命令来关闭本地服务器。 **“关闭本地服务器”** 命令还可用于 **“所有服务器”** 页面上的 **“服务器”** 磁贴中的本地服务器，或者代表本地服务器的任何角色或组页面。

使用此方法与关闭 Windows Server 2016 从方法关闭本地服务器**启动**屏幕上，打开**关闭 Windows**对话框，从中可以指定关闭原因在中**关闭事件跟踪程序**区域。

> [!NOTE]
> 只有管理员组的成员才能关闭或重启服务器。 标准用户不能关闭或重启服务器。 单击 **“关闭本地服务器”** 命令可注销标准用户的服务器会话。 这与普通用户从服务器桌面运行 **Alt+F4** 关闭命令的效果相同。

## <a name="BKMK_props"></a>配置服务器管理器属性
你可以在 **“本地服务器”** 页面的 **“属性”** 磁贴中查看或更改以下设置。 若要更改设置的值，请单击该设置的超链接值。

> [!NOTE]
> 通常，本地服务器 **“属性”** 磁贴中显示的属性只能在本地服务器上更改。 无法从远程计算机更改本地服务器属性，使用服务器管理器，因为**属性**磁贴只能获取本地计算机，而非远程计算机的信息。
> 
> 由于许多属性显示在**属性**工具不是一部分的服务器管理器 （控制面板，例如），由控制磁贴变为**属性**设置并不总是显示在**属性**磁贴。 默认情况下，**“属性”** 磁贴中的数据每两分钟刷新一次。 若要刷新**属性**磁贴数据中，单击**刷新**服务器管理器地址栏中。

## <a name="BKMK_managesm"></a>管理服务器管理器控制台。
在服务器管理器控制台窗口顶部的标题栏中找到全局设置，将应用于整个服务器管理器控制台，并已添加到服务器管理器服务器池的所有远程服务器。

### <a name="add-servers-to-server-manager"></a>将服务器添加到服务器管理器
打开的命令可**将服务器添加**对话框中，并允许您对服务器管理器服务器池添加远程物理或虚拟服务器、 处于**管理**菜单中的服务器管理器控制台。 有关如何添加服务器的详细信息，请参阅[将服务器添加到服务器管理器](add-servers-to-server-manager.md)。

### <a name="refresh-data-that-is-displayed-in-server-manager"></a>刷新服务器管理器中显示的数据
可以配置数据显示在服务器管理器上的刷新间隔**服务器管理器属性**对话框中，从打开**管理**菜单。

##### <a name="to-configure-the-refresh-interval-in-server-manager"></a>配置服务器管理器刷新间隔时间的步骤

1.  上**管理**菜单上，在服务器管理器控制台中，单击**服务器管理器属性**。

2.  在中**服务器管理器属性**对话框框中，指定一个时间段，以分钟为单位，所需要的显示在服务器管理器中的数据刷新间隔时间量。 默认值为 10 分钟。 完成后单击“确定”。

#### <a name="refresh-limitations"></a>刷新限制
刷新将全局应用于数据的所有服务器的已添加到服务器管理器服务器池。 你无法为单个服务器、角色或组来刷新数据或配置不同的刷新间隔时间。

群集中的服务器添加到服务器管理器中，无论它们是物理计算机或虚拟机，数据在首次刷新都会失败或只显示数据为群集对象主服务器。 之后再进行刷新时便会显示服务器群集中的物理或虚拟服务器的准确数据。

为远程桌面服务、 IP 地址管理以及文件和存储服务角色主页在服务器管理器中显示的数据不会自动刷新。 刷新数据显示在这些页中手动，通过按**F5**或单击**刷新**在服务器管理器控制台标题，而你位于这些页面上。

### <a name="add-or-remove-roles-or-features"></a>添加或删除角色或功能
打开添加角色和功能向导和删除角色和功能向导，并让您中添加或删除角色、 角色服务和功能的服务器到服务器池的命令**管理**的服务器管理器菜单控制台中，并**任务**菜单**角色和功能**角色或组页上磁贴。 有关如何添加或删除角色或功能的详细信息，请参阅[安装或卸载角色、 角色服务或功能](install-or-uninstall-roles-role-services-or-features.md)。

在服务器管理器中，角色和功能的数据显示在系统中，也称为系统默认 GUI 语言或在操作系统安装过程中选择的语言的基本语言。

### <a name="create-server-groups"></a>创建服务器组
打开的命令可**创建服务器组**对话框中，并允许您创建自定义服务器组，处于**管理**菜单中的服务器管理器控制台。 有关如何创建服务器组的详细信息，请参阅[创建和管理服务器组](create-and-manage-server-groups.md)。

### <a name="prevent-server-manager-from-opening-automatically-at-logon"></a>防止登录时自动启动服务器管理器
**在不启动服务器管理器会自动登录**中的复选框**服务器管理器属性**框可以控制是否服务器管理器会自动打开在登录时的成员本地服务器上的 administrators 组。 作为远程服务器管理工具的一部分运行在 Windows 10 上时，此设置不会影响服务器管理器行为。 有关配置此设置的详细信息，请参阅[服务器管理器](server-manager.md)。

### <a name="zoom-in-or-out"></a>放大或缩小
若要放大或缩小的服务器管理器控制台视图，您可以使用**Zoom**上的命令**视图**菜单中或按**Ctrl + 加号 （+）** 来缩放和**Ctrl + 减号 （-）** 缩小。

## <a name="BKMK_tools"></a>自定义工具菜单中显示的工具
**工具**菜单在服务器管理器中包括快捷方式软链接**管理工具**文件夹中的**控制面板/系统和安全**。 **管理工具**文件夹包含快捷方式或 LNK 文件可用的管理工具，例如 mmc 管理单元的列表。服务器管理器填充**工具**菜单，其中包含指向这些快捷方式，并将复制的文件夹结构**管理工具**文件夹**工具**菜单。 默认情况下，“管理工具”文件夹中的工具以无层次列表形式显示，以及按类型和名称排序。 在服务器管理器**工具**菜单中，仅按名称，不按类型排序项。

若要定制该 **“工具”** 菜单，可将要使用的工具或脚本快捷方式复制到 **“管理工具”** 文件夹。 你还可以组织文件夹中的快捷方式，为 **“工具”** 菜单创建级联菜单。 此外，如果你想要限制访问的自定义工具上**工具**菜单中，您可以设置用户访问权限在管理工具，自定义工具文件夹或直接在原始工具或脚本文件。

我们不建议重新组织系统和管理工具以及与本地服务器上安装的角色和功能相关的任何管理工具。 必要时可通过移动角色和功能管理工具来阻止这些管理工具卸载。 角色或功能卸载后，虽然工具快捷方式已删除，但 **“工具”** 菜单中仍可能存在一个该工具的无效链接。 如果你重新安装该角色，**“工具”** 菜单中还会生成一个指向同一工具的副本链接，但其中一个链接将无效。

然而，在基于 Windows 客户端的计算机上作为远程服务器管理工具的一部分进行安装的角色和功能工具，可以放入自定义文件夹。 卸载父级角色或功能不起作用运行 Windows 10 的远程计算机上的可用工具快捷方式。

以下过程描述如何创建一个名为的示例文件夹*MyTools*，并将两个 Windows PowerShell 脚本的快捷方式移动到可以从服务器管理器工具菜单上的文件夹。

#### <a name="to-customize-the-tools-menu-by-adding-shortcuts-in-administrative-tools"></a>通过在“管理工具”中添加快捷方式来定制“工具”菜单的步骤

1.  创建一个名为的新文件夹*MyTools*在方便的位置。

    > [!NOTE]
    > 由于 **“管理工具”** 文件夹具有访问权限限制，你无法直接在 **“管理工具”** 文件夹中新建文件夹；你必须在其他位置（例如桌面）新建文件夹，然后将新文件夹复制到 **“管理工具”** 文件夹。

2.  移动或复制*MyTools*到**控制面板/系统和安全 / 管理工具**。 默认情况下，你必须是计算机管理员组的成员，才能更改 **“管理工具”** 文件夹。

3.  如果不需要限制用户对自定义工具快捷方式的访问权限，请转到步骤 6。 否则，右键单击工具文件（或 *MyTools* 文件夹），然后单击 **“属性”**。

4.  上**安全**的文件的选项卡**属性**对话框中，单击**编辑**。

5.  对于你想要限制访问工具的用户，清除复选框**读取 & 执行**，**读取**，并**编写**权限。 **“管理工具”** 文件夹中的工具快捷方式会继承这些权限。

    如果用户使用服务器管理器时编辑用户的访问权限 （或服务器管理器时打开），则所做的更改不会显示在**工具**直到用户重新启动服务器管理器的菜单。

    > [!NOTE]
    > 如果你已复制到管理工具的整个文件夹限制访问，受限制的用户可以看到该文件夹或其内容在服务器管理器**工具**菜单。
    > 
    > 编辑中的文件夹的权限**管理工具**文件夹。 由于管理工具中隐藏的文件和文件夹始终显示在服务器管理器**工具**菜单中，不要使用**Hidden**上的文件或文件夹的设置**属性**对话框中，若要限制用户对自定义工具快捷方式访问。
    > 
    > **“拒绝”** 权限总是覆盖 **“允许”** 权限。

6.  右键单击原始工具、 脚本或可执行文件，您要为其上添加项**工具**菜单，并单击**创建快捷方式**。

7.  将移到快捷*MyTools*管理工具中的文件夹。

8.  刷新或重新启动服务器管理器中，如有必要，若要查看你的自定义工具快捷方式**工具**菜单。

## <a name="BKMK_roles"></a>管理角色主页上的角色
将服务器添加到服务器管理器服务器池和服务器管理器收集有关在您池中的服务器的清单数据后，服务器管理器会将页面添加到的托管服务器发现的角色的导航窗格中。 角色页面上的 **“服务器”** 磁贴列出正在运行角色的托管服务器。 默认情况下，“事件” 、“最佳做法分析器” 、“服务” 和“性能”  磁贴显示所有运行角色的服务器的数据；选择“服务器”  磁贴中的特定服务器会让事件、服务、性能计数器和 BPA 结果的范围仅限于所选服务器。 通常在服务器管理器控制台中提供了管理工具**工具**菜单中，已安装或在托管服务器上发现的角色或功能后。 你还可以右键单击角色或组的“服务器”磁贴中的服务器条目，然后启动要使用的管理工具。

在 Windows Server 2016 中，以下角色和功能的以页面形式集成到服务器管理器控制台的管理工具。

-   **文件和存储服务。** 文件和存储服务页面包括用于管理卷、共享、iSCSI 虚拟磁盘和存储池的自定义磁贴和命令。 当打开文件和存储服务角色主页中服务器管理器中，一个收缩窗格将打开，显示文件和存储服务自定义管理页面。 有关部署和管理文件和存储服务的详细信息，请参阅 [文件和存储服务](https://go.microsoft.com/fwlink/p/?LinkId=241530)。

-   **远程桌面服务。** 远程桌面服务页面包括用于管理会话、许可证、网关和虚拟机的自定义磁贴和命令。 有关部署和管理远程桌面服务的详细信息，请参阅[远程桌面服务 (rdS)](https://go.microsoft.com/fwlink/p/?LinkId=241532)。

-   **IP 地址管理 (IPAM)。** IPAM 角色页面包括一个自定义“欢迎”磁贴，该磁贴包含常规 IPAM 配置和管理任务的链接，其中包括预配 IPAM 服务器的向导。 IPAM 主页还包含用来查看托管网络、配置摘要和计划任务的磁贴。

    有一些限制的 IPAM 管理服务器管理器中。 与常见角色和组页面不同，IPAM 没有 **“服务器”**、**“事件”**、**“性能”**、**“最佳做法分析器”** 或 **“服务”** 磁贴。 没有可用于 IPAM; 的最佳做法分析器模型不支持在 IPAM 上的最佳做法分析器扫描。 若要访问运行 IPAM 的服务器池中的服务器，请为运行 IPAM 的服务器创建一个自定义组，然后从自定义组页面的 **“服务器”** 磁贴访问服务器列表。 或者，还可从 **“所有服务器”** 组页面的 **“服务器”** 磁贴访问 IPAM 服务器。

    比起其他角色和组缩略图，仪表板缩略图还会显示有限的 IPAM 行。 单击 IPAM 缩略图行，可查看运行 IPAM 的服务器的事件、性能数据和可管理性状态通知。 可以从包含 IPAM 服务器的服务器组页面（例如 **“所有服务器”** 组的页面）管理 IPAM 相关服务。

    有关部署和管理 IPAM 的详细信息，请参阅[IP 地址管理 (IPAM)](https://go.microsoft.com/fwlink/p/?LinkId=241533)。
