# <a name="mapping-design-requirements-to-forest-design-models"></a>设计要求映射到林设计模型

>适用于：Windows Server 2016 中，Windows Server 2012 R2、 Windows Server 2012

在你的组织中的大多数组可以共享单个组织林，且由单一信息技术 (IT) 小组和，其中包含的用户帐户和共享在林中的组的所有资源。 此共享的林，名为初始组织林，是对组织的林设计模型的基础。  

因为初始组织林可以承载组织中的多个组，林所有者必须建立与每个组的服务级别协议，以便所有参与方了解它们的预期。 这可通过建立商定的服务期望的各个组和林所有者。  

如果未在组织中的组的所有可以共享单个组织林，必须扩展林设计以满足不同的组的需求。 这涉及到标识适用于基于其需求的自治性和隔离以及它们具有受限连接网络的组的设计要求，然后确定可用于适应这些林模型系统要求。 下表列出了林设计模型方案基于自治性、 隔离性和连接性因素。 在确定最符合要求的林设计方案后，确定是否需要进行任何其他的决策，以满足您的设计规范。  

> [!NOTE]  
> 如果身份被列为不适用，则不需要考虑因为其他要求也适用该系数。

## <a name="BKMK_1"></a>方案 1:联接数据自主性的现有林  

只需通过托管在现有组织林中的组织单位 (Ou) 中的组，可以满足数据自主权的必要条件。 从该组，以实现数据自治委派给数据管理员控制的 Ou。 有关使用 Ou 委派控制的详细信息，请参阅[创建组织单位设计](../../ad-ds/plan/Creating-an-Organizational-Unit-Design.md)。  
  
## <a name="BKMK_2"></a>方案 2:组织林或域用于服务自治性  

如果你的组织中的组标识服务自治性作为一项要求，我们建议您首先重新考虑此要求。 实现了服务自治性创建更多的管理开销和组织的其他成本。 请确保服务自治性的要求不只是为方便起见，越是中满足此要求产生的成本。  
  
可以通过执行以下任一操作来满足所必需的服务自治性：  

- 创建组织的林。 将用户、 组和计算机组到一个单独的组织林所需服务自治性。 将个人分配组中要为林所有者。 如果组都需要与组织中其他林的访问权限或共享资源，它们可以建立其组织的林与其他林之间建立信任。  

- 使用组织的域。 将用户、 组和计算机放在单独的域中现有的组织林。 此模型提供域级服务自治性以及不能用于全文服务自治服务隔离或需要数据隔离。  

有关使用组织的域的详细信息，请参阅[使用组织域林模型](../../ad-ds/plan/../../ad-ds/plan/Using-the-Organizational-Domain-Forest-Model.md)。  

## <a name="BKMK_3"></a>方案 3:使用的服务隔离组织林或资源林  

可以通过执行以下任一操作来满足所必需的服务隔离：  

- 使用组织的林。 将用户、 组和计算机需要服务隔离到一个单独的组织目录林中的组。 将个人分配组中要为林所有者。 如果组都需要与组织中其他林的访问权限或共享资源，它们可以建立其组织的林与其他林之间建立信任。 但是，我们不建议此方法因为在目录林信任方案中很大程度限制通过通用组的资源访问权限。  

- 使用资源林。 将资源和服务帐户放入单独的资源林，将用户帐户保留在现有组织林。 如有必要，可以在要访问资源林中的资源，如果组织林变得不可用的资源林中创建备用帐户。 备用帐户必须具有登录到资源林和维护资源的控制，直到组织林已重新联机所需的颁发机构。  

   建立资源与组织的林之间建立信任，以便使用其普通用户帐户时，用户可以访问林中的资源。 此配置使，从而允许用户组织林不可用时回退到备用帐户资源林中的用户帐户的集中式的管理。  

服务隔离的注意事项包括：

- 创建的服务隔离的林可以信任来自其他林的域，但不能包含其他林中的用户在其服务管理员组中。 如果在独立的林的管理组中包括其他林中的用户，独立的林的安全性可能会受到影响因为林中的服务管理员不具有独占控件。  

- 域控制器可访问网络上，只要它们是遭受攻击 （例如拒绝服务攻击） 从该网络上的恶意软件。 您可以执行以下操作来抵御攻击的可能性：  

   - 仅在被视为安全的网络上的主机域控制器。  

   - 限制对网络或托管的域控制器的网络访问。  

- 服务隔离需要其他林的创建。 评估是否维护基础结构以支持其他林的成本远远超过丢失由于变得不可用的组织林的资源的访问权限的情况相关的成本。  

## <a name="BKMK_4"></a>方案 4:使用组织林或受限制的访问林进行数据隔离  

可以通过执行以下任一操作来实现数据隔离：  

- 使用组织的林。 将用户、 组和计算机需要数据隔离到一个单独的组织目录林中的组。 将个人分配组中要为林所有者。 如果组都需要与组织中其他林的访问权限或共享资源，建立组织的林与其他林之间建立信任。 在新组织林中存在仅需要对已分类的信息的访问的用户。 用户具有信任关系通过用来访问分类自己的林结构中的数据和其他林中的未分类的数据的一个帐户。  

- 使用受限制的访问林。 这是包含受限制的数据和用于访问该数据的用户帐户的单独林中。 在用于访问网络上不受限制的资源的现有组织林维护单独的用户帐户。 受限制的访问林和企业中的其他林之间创建没有信任关系。 可以进一步通过将部署在单独的物理网络的林中限制林，以便它无法连接到其他林。 如果部署在单独的网络的林中，用户必须具有两个工作站： 一个用于访问受限制的林，另一个用于访问网络的 nonrestricted 的区域。  

创建进行数据隔离的林的注意事项包括：  

- 针对数据隔离创建的组织林可以信任来自其他林的域，但其他林中的用户必须不包含以下任何方面：  

  - 负责服务管理或组可以管理服务管理员组的成员身份的组  

  - 具有对计算机上存储受保护的数据的管理控制的组  

  - 组有权访问受保护的数据或组负责的用户对象或有权访问的组对象管理的受保护的数据  

    如果任何这些组中包含来自另一个林的用户，另一个林的安全威胁可能会导致危及安全的独立的林和受保护数据的泄露。  

- 可以配置其他林信任组织的林，以便在独立的林的用户可以访问其他林中的资源创建的数据隔离。 但是，从独立的林的用户必须永远不会以交互方式登录到在信任林的工作站。 信任林中的计算机可能会泄露了恶意软件，并可用于捕获用户的登录凭据。  

   > [!NOTE]
   > 若要防止模拟从独立的林的用户，然后访问独立林中的资源中的信任林的服务器，林所有者可以禁用委派的身份验证或使用受约束的委派功能。 有关委派的身份验证和约束的委派的详细信息，请参阅[委派身份验证](https://go.microsoft.com/fwlink/?LinkId=106614)。  

- 您可能需要限制对其林之外的信息的用户访问组织中建立组织的林与其他林之间的防火墙。  

- 尽管创建单独的林中使数据隔离，只要在独立的林和计算机的域控制器的受保护的主机信息是在网络上可访问，但不会从该网络上的计算机发起的攻击。 确定攻击的风险太高，或攻击或安全违规的后果是过大的组织需要限制对网络或托管的域控制器的网络访问和托管的计算机受保护的数据. 可以通过使用防火墙和 Internet 协议安全 (IPsec) 等技术来限制访问权限。 在极端情况下，组织可能会选择维护组织中有不到任何其他网络的物理连接的独立网络上的受保护的数据。  

   > [!NOTE]  
   > 受限制的访问林与另一个网络之间存在任何网络连接，可能存在受限制区域传送到另一个网络中的数据。  

## <a name="BKMK_5"></a>方案 5:使用组织的林，或重新配置防火墙以允许有限的连接  

若要满足连接受到限制的要求时，可以执行下列任一操作：  

- 将用户放入现有组织林，然后打开防火墙不足以允许 Active Directory 流量通过。  

- 使用组织的林。 将用户、 组和组仅限于连接的计算机放入单独的组织林。 将个人分配组中要为林所有者。 组织林提供一个单独的环境的另一端的防火墙。 在林中包含的林中，管理用户帐户和资源，以便用户不需要通过防火墙来完成其日常任务。 特定用户或应用程序可能有特殊需求，需要通过防火墙联系其他林的功能。 通过包括所需的任何信任关系才能正常那些在防火墙中打开相应的接口，可以单独满足这些需求。  

有关配置防火墙以使用与 Active Directory 域服务 (AD DS) 的详细信息，请参阅[防火墙分段的网络中的 Active Directory](https://go.microsoft.com/fwlink/?LinkId=37928)。  

## <a name="BKMK_6"></a>方案 6:使用组织林或域，并使用有限的连接重新配置防火墙以允许服务自治性  

如果你的组织中的组标识服务自治性作为一项要求，我们建议您首先重新考虑此要求。 实现了服务自治性创建更多的管理开销和组织的其他成本。 请确保服务自治性的要求不只是为方便起见，越是中满足此要求产生的成本。  

如果连接受到限制时出现问题，并且必须所必需的服务自治性，可以执行下列任一操作：  

- 使用组织的林。 将用户、 组和计算机组到一个单独的组织林所需服务自治性。 将个人分配组中要为林所有者。 组织林提供一个单独的环境的另一端的防火墙。 在林中包含的林中，管理用户帐户和资源，以便用户不需要通过防火墙来完成其日常任务。 特定用户或应用程序可能有特殊需求，需要通过防火墙联系其他林的功能。 通过包括所需的任何信任关系才能正常那些在防火墙中打开相应的接口，可以单独满足这些需求。  

- 将用户、 组和计算机放在单独的域中现有的组织林。 此模型提供域级服务自治性以及不能用于全文服务自治服务隔离或需要数据隔离。 其他林中的组必须信任其信任林所有者的程度等同于新域的服务管理员。 出于此原因，我们不建议此方法。 有关使用组织的域的详细信息，请参阅[使用组织域林模型](../../ad-ds/plan/../../ad-ds/plan/Using-the-Organizational-Domain-Forest-Model.md)。  

此外需要打开防火墙足够，使 Active Directory 流量通过。 有关配置防火墙以使用 AD ds 的详细信息，请参阅[防火墙分段的网络中的 Active Directory](https://go.microsoft.com/fwlink/?LinkId=37928)。  

## <a name="BKMK_7"></a>方案 7:使用资源林，并使用有限的连接重新配置防火墙以允许服务隔离  

如果连接受到限制时出现问题，并且您具有必需的服务隔离，可以执行下列任一操作：  

- 使用组织的林。 将用户、 组和计算机需要服务隔离到一个单独的组织目录林中的组。 将个人分配组中要为林所有者。 组织林提供一个单独的环境的另一端的防火墙。 在林中包含的林中，管理用户帐户和资源，以便用户不需要通过防火墙来完成其日常任务。 特定用户或应用程序可能有特殊需求，需要通过防火墙联系其他林的功能。 通过包括所需的任何信任关系才能正常那些在防火墙中打开相应的接口，可以单独满足这些需求。  

- 使用资源林。 将资源和服务帐户放入单独的资源林，将用户帐户保留在现有组织林。 可能需要在要保持对资源林访问权限，如果组织林变得不可用的资源林中创建一些备用用户帐户。 备用帐户必须具有登录到资源林和维护资源的控制，直到组织林已重新联机所需的颁发机构。  

   建立资源与组织的林之间建立信任，以便使用其普通用户帐户时，用户可以访问林中的资源。 此配置使，从而允许用户组织林不可用时回退到备用帐户资源林中的用户帐户的集中式的管理。  

服务隔离的注意事项包括：  

- 创建的服务隔离的林可以信任来自其他林的域，但不能包含其他林中的用户在其服务管理员组中。 如果在独立的林的管理组中包括其他林中的用户，独立的林的安全性可能会受到影响因为林中的服务管理员不具有独占控件。  

- 域控制器可访问网络上，前提是它们受到攻击 （例如拒绝服务攻击） 的计算机从网络上。 您可以执行以下操作来抵御攻击的可能性：  

   - 仅在被视为安全的网络上的主机域控制器。  

   - 限制对网络或托管的域控制器的网络访问。  

- 服务隔离需要其他林的创建。 评估是否维护基础结构以支持其他林的成本远远超过丢失由于变得不可用的组织林的资源的访问权限的情况相关的成本。  

   特定用户或应用程序可能有特殊需求，需要通过防火墙联系其他林的功能。 通过包括所需的任何信任关系才能正常那些在防火墙中打开相应的接口，可以单独满足这些需求。  

有关配置防火墙以使用 AD ds 的详细信息，请参阅[防火墙分段的网络中的 Active Directory](https://go.microsoft.com/fwlink/?LinkId=37928)。
