# <a name="audit-policy-recommendations"></a>建议审核策略

>适用于：Windows Server 2016，Windows Server 2012 R2、Windows Server 2012

解决了 Windows 默认审核策略设置此部分中，但基准建议审核策略设置和其他危害程度大的建议来自 Microsoft，工作站和服务器产品。  

显示在这里，以帮助检测到的危害，我们建议设置以及 SCM 基准建议只希望将管理员起始基准指南。 每一个组织必须进行自己有关他们遇到的威胁，他们接受风险公差，应哪些审核策略类别或子类别中启用的决策。 有关威胁的详细信息，请参阅[威胁并对策指南](https://technet.microsoft.com/library/hh125921(v=ws.10).aspx)。 鼓励开始使用此处，建议设置没有认真审核策略就地管理员将然后修改并测试之前在他们生产的环境中实现。  

对于企业级的计算机，Microsoft 将定义为有平均安全要求，并且要求的运行很高的计算机是功能的推荐。 实体需要更高版本的安全要求应该考虑其他危害程度大审核策略。  

> [!NOTE]  
> Microsoft Windows 的默认值，以及从执行基准建议[Microsoft 安全合规性管理器工具](https://technet.microsoft.com/library/cc677002.aspx)。  

正常安全计算机不知道要下活动、 成功攻击确定的对手或恶意软件，以下基准审核策略设置被推荐。

## <a name="set-audit-policy-on-workstations-and-servers"></a>设置工作站和服务器上的审核策略  
所有事件日志管理计划应都监视工作站和服务器。 常见错误是仅监视服务器或域控制器。 由于恶意通常最初黑客发生工作站上，因此不监视工作站忽视信息的最佳和最早的来源。  

管理员应该仔细查看和测试他们生产环境中实现之前任何审核策略。  

## <a name="events-to-monitor"></a>监视器的事件  
要生成安全警告完美事件 ID 应包含以下属性：  

-   高的可能性此匹配项指示未经授权的活动  

-   误报低数  

-   发生应导致调查/鉴定响应  

应监视和提醒两种类型的事件：  

1.  这些事件在这甚至出现一次指示未经授权的活动  

2.  上方的预期并接受基准事件的累积  

第一件事情的一个示例是：  

登录到计算机未域控制器禁止域管理员 (DAs)，如果发生一次登录到最终用户工作站 DA 成员应该生成一个警报，进行调查。 这种类型的警报很容易生成通过使用审核特殊的登录活动 4964 （特殊组已分配给新登录）。 单实例通知的其他示例包括：  

-   如果服务器 A 应永远不会连接到服务器 B，连接到彼此时的警报。  

-   如果正常最终用户帐户意外添加到敏感安全组，警报。  

-   如果在夜晚格外美丽警报用户在登录时午夜的员工出厂位置 A 永远不会工作。  

-   如果未经授权的服务上域控制器安装，警报。  

-   如果定期最终用户尝试直接登录到他们拥有完成该操作无清除原因 SQL Server，调查。  

-   如果你不有任何成员 DA 组中，其他人添加了自行存在，请立即检查。  

第二个事件的一个示例是：  

异常失败登录数指示猜测攻击密码。 为提供的非常高的失败登录数警报企业，他们必须先了解标准基准之前恶意安全事件他们的环境中的失败登录。  

你应该包括危害的迹象监视时的事件的完整列表，请参阅[到监视器附录 l： 事件](../../../ad-ds/plan/Appendix-L--Events-to-Monitor.md)。

## <a name="general-list-of-security-event-id-recommendation-criticalities"></a>常规列表中的安全事件 ID 建议 Criticalities  
推荐的所有事件 ID 都附有严重程度 rating，如下所示：  

**高：**事件 Id 高严重程度分级应始终和立即提醒和调查。  

**中等：**中等严重程度分级事件 ID 指示恶意的活动，但必须伴随一些异常 （例如，异常数发生在特定时间段、 意外的发生或通常将不会记录的事件的计算机上的发生。）。 此外可能中严重程度事件 r 进行作为指标收集和随着时间的推移比较。  

**低：**记录事件 ID 严重性较低事件，应该不会获得关注或导致警告，除非了中等或高严重程度事件相关。  

这些建议旨在提供基准指南管理员。 推荐的所有应完全擦除查看之前生产环境中实现。
