# <a name="implementing-secure-administrative-hosts"></a>实现安全管理主机

>适用于：Windows Server 2016 中，Windows Server 2012 R2、 Windows Server 2012

安全管理主机是工作站还是服务器已配置专用于创建从其特权的帐户可以执行管理任务在 Active Directory 或域控制器上的安全平台的目的已加入域的系统，并在已加入域的系统上运行的应用程序。 在这种情况下，"特权的帐户"引用的 Active Directory 中的最高特权组的成员的帐户不仅对到任何已被委派的权限的帐户和权限，允许执行管理任务。  
  
这些帐户可能已为大多数帐户用于管理 DNS 记录和区域或用于配置管理的帐户在域中，用户重置密码的功能的支持人员帐户。 安全管理主机专用于管理功能，并且它们不运行如 Microsoft Office 电子邮件应用程序、 web 浏览器或工作效率软件等软件。  
  
尽管"最高特权的"帐户和组应相应地为最严格受保护，但这并不能不排除需要保护所有帐户和已向上面的标准用户的权限授予帐户的组。  
  
安全管理主机可以仅用于管理任务，运行远程桌面网关服务器角色的成员服务器的专用的工作站和哪些 IT 用户连接到要执行的目标主机或运行的服务器管理HYPER-V 角色，并提供唯一的虚拟机的每个 IT 用户使用其管理任务。 在许多环境中，可能会实现所有这三种方法的组合。  
  
实现安全管理主机需要规划和配置，而是与你组织的大小、 管理做法，风险偏好和预算保持一致。 注意事项和实现安全管理主机的选项供你在开发适用于你的组织的管理策略使用此处提供。  
  
## <a name="principles-for-creating-secure-administrative-hosts"></a>用于创建安全的管理主机的原则  
若要有效地保护系统免受攻击，应记住保留几个一般原则：  
  
1.  永远不应从太受信任的主机 （即，未受保护程度作为它所管理的系统的工作站） 管理受信任的系统 （即，例如域控制器的安全服务器）。  
  
2.  您不应依赖于单一身份验证因素时执行特权的活动;也就是说，用户和密码组合不应将可接受的身份验证因为只有一个因素 （一种您知道） 表示形式。 应考虑是生成和缓存或管理方案中存储凭据。  
  
3.  虽然当前的威胁环境中的大多数攻击利用恶意软件和恶意黑客，不要忽略物理安全性设计和实现安全管理主机时。  
  
### <a name="account-configuration"></a>帐户配置  
即使你的组织当前不使用智能卡，应考虑实现用于特权的帐户和安全管理主机。 管理主机应配置为通过修改 GPO 链接到包含管理主机的 Ou 中的以下设置的所有帐户需要智能卡登录：  
  
**计算机配置 \ 策略 \windows 设置 \ 本地策略 \ 安全 Options\Interactive 登录：需要智能卡**  
  
此设置后，必须使用智能卡，而不考虑在单个帐户在 Active Directory 中配置的所有交互式登录。  
  
您还应该配置为仅由授权的帐户，可以在中配置允许登录的安全管理主机：  
  
**计算机配置 \ 策略 \windows 设置 \ 本地策略 \ 安全设置 \ 本地策略 \ 用户权限分配**  
  
此授予交互式 （并在适用时，远程桌面服务） 仅向已授权用户的安全管理主机的登录权限。  
  
### <a name="physical-security"></a>物理安全性  
对于被视为可信任的管理主机，它们必须配置并为他们管理的系统的程度等同于受保护。 中提供的建议的大多数[针对攻击保护域控制器](../../../ad-ds/plan/security-best-practices/Securing-Domain-Controllers-Against-Attack.md)还有适用于用来管理域控制器和 AD DS 数据库的主机。 在大多数环境中实现安全管理系统的挑战之一是可以更难实现，因为这些计算机通常驻留在不是服务器托管在数据中心，如尽可能安全的区域中物理安全性管理用户的桌面。  
  
物理安全包括控制对管理主机的物理访问。 在小型组织中，这可能意味着你始终在 office 或在不使用时的桌子抽屉中保留的专用管理工作站锁定。 也可能表示，当您需要执行的 Active Directory 或你的域控制器管理，您直接登录到域控制器。  
  
在中型组织中，可以考虑实施安全管理"跳转服务器"，位于在办公室内的安全位置，并需要管理的 Active Directory 或域控制器时使用。 您也可能实现在安全位置不在使用时，使用或跳转服务器不锁定的管理工作站。  
  
在大型组织中，你可以部署到 Active Directory; 提供的严格受控的访问的数据中心存放的跳转服务器域控制器;和文件、 打印或应用程序服务器。 跳转服务器体系结构的实现是最有可能在大型环境中包括安全的工作站和服务器的组合。  
  
无论你的组织的大小和管理主机的设计，你应保护物理计算机免受未经授权的访问或被盗，并应使用 BitLocker 驱动器加密来加密和保护管理主机上的驱动器. 通过在管理主机上实现 BitLocker，即使主机被盗或删除其磁盘时，您可以确保驱动器上的数据都无法访问未经授权的用户。  
  
### <a name="operating-system-versions-and-configuration"></a>操作系统版本和配置  
所有管理主机是否服务器或工作站，应运行最新操作系统中使用你的组织中的本文档前面所述的原因。 通过运行当前的操作系统，您的管理人员可以利用新的安全功能、 完整的供应商支持和其他操作系统中引入的功能。 此外，当您正在评估新的操作系统，通过将其部署到管理第一个主机，你将需要自己熟悉的新功能、 设置和管理机制提供，随后可利用在规划更广泛的操作系统部署。 那时，您组织中最复杂的用户也将是熟悉新的操作系统和最方便地进行支持它的用户。  
  
### <a name="microsoft-security-configuration-wizard"></a>Microsoft 安全配置向导  
如果你实现跳转服务器作为管理的主机策略的一部分，应使用内置的安全配置向导来配置服务、 注册表、 审核和防火墙设置，以减少服务器的受攻击面。 时的安全配置向导配置设置已收集并配置后，可将设置转换用于强制实施所有跳转服务器上的一致的基线配置的 GPO。 您可以进一步编辑 GPO 以实现安全设置特定于要跳转服务器，并可以组合的所有其他基准设置提取从 Microsoft Security Compliance Manager 设置。  
  
### <a name="microsoft-security-compliance-manager"></a>Microsoft Security Compliance Manager  
[Microsoft Security Compliance Manager](https://technet.microsoft.com/library/cc677002.aspx)是免费提供的工具集成 Microsoft，根据操作系统版本和角色配置推荐的安全配置和将在这些数据收集这一个工具，可用于创建和配置基线域控制器的安全设置的 UI。 Microsoft 安全合规管理器模板可以组合使用以生成全面的配置基线的跳转服务器的部署和部署了在哪些跳转服务器是的 Ou 的 Gpo 的强制实施的安全配置向导设置位于 Active Directory 中。  
  
> [!NOTE]  
> 撰写本文时，Microsoft Security Compliance Manager 不包括设置特定于跳转服务器或其他安全管理主机，但 Security Compliance Manager (SCM) 仍可用于创建为你管理的初始基线主机的主机。 但是，若要正确保护的主机，应该应用其他安全设置适用于高度安全的工作站和服务器。  
  
### <a name="applocker"></a>AppLocker  
使用脚本、 工具和应用程序允许列表通过 AppLocker 或第三方应用程序限制软件配置管理主机和虚拟 machinesshould。 任何管理应用程序或实用程序，不符合保护设置应升级或替换为要遵循的安全开发和管理做法的工具。 当新的或其他工具需要管理的主机上时，应全面测试应用程序和实用程序，以及是否适用于管理主机上的部署工具，它可以添加到系统的允许列表。  
  
### <a name="rdp-restrictions"></a>RDP 限制  
特定的配置将会有所不同，具体取决于管理系统的体系结构，但应包含的帐户和计算机可用于建立远程桌面协议 (RDP) 连接到托管系统的限制例如，使用远程桌面网关 （RD 网关） 跳转服务器来控制对域控制器的访问和经过授权的用户和系统中其他托管的系统。  
  
您应允许获得授权的用户的交互式登录和应删除或甚至阻止其他不需要的服务器的访问权限的登录类型。  
  
### <a name="patch-and-configuration-management"></a>修补程序和配置管理  
较小的组织可能依赖于产品/服务，Windows 更新或[Windows Server Update Services](https://technet.microsoft.com/windowsserver/bb332157) (WSUS) 来管理 Windows 系统的更新部署，而较大的组织可能会实现企业修补程序和System Center Configuration Manager 等软件配置管理。 无论您使用将更新部署到你的常规服务器和工作站总体的机制，应考虑高度安全的系统，例如域控制器、 证书颁发机构和管理主机的单独的部署。 通过隔离这些系统的常规管理基础结构，如果管理软件或服务帐户已泄露，这种损害无法轻松地扩展到您的基础结构中的最安全系统。  
  
尽管不应实现安全系统的手动更新过程，应配置单独的基础结构，对安全系统进行更新。 甚至在超大型组织中，此基础结构可以通常实现通过专用的 WSUS 服务器和 Gpo 的安全系统。  
  
### <a name="blocking-internet-access"></a>阻止 Internet 访问  
管理主机不应允许访问 Internet，也不应能够浏览组织的 intranet。 不应在管理主机上允许 web 浏览器和类似应用程序。 为安全主机通过外围防火墙设置、 WFAS 配置和安全的主机上的"黑洞"代理配置的组合，可以阻止 Internet 访问权限。 应用程序允许列表还可用于防止管理主机上使用 web 浏览器。  
  
### <a name="virtualization"></a>虚拟化  
如有可能，请考虑实施虚拟机作为在管理主机。 使用虚拟化，您可以创建每个用户管理系统的集中存储和管理，，其中可以轻松地关闭时不在使用中，确保，凭据不保持活动状态的管理系统上。 此外可以要求的虚拟管理主机重置为初始快照后每次使用，确保虚拟机保持原始。 在以下部分提供有关管理主机的虚拟化选项的详细信息。  
  
## <a name="sample-approaches-to-implementing-secure-administrative-hosts"></a>示例方法可以实现安全管理主机  
而不考虑如何设计和部署你管理的主机基础结构时，您应该牢记于心本主题前面的"原则创建安全管理主机的"中提供的指南。 此处所介绍的方法的每个提供有关如何可以分隔"管理"和"生产力"系统由您的 IT 员工的常规信息。 工作效率系统是 IT 管理员使用要检查电子邮件，请浏览 Internet，并且使用诸如 Microsoft Office 之类的常规工作效率软件的计算机。 管理系统是强化和专用于使用的 IT 环境的日常管理的计算机。  
  
实现安全管理主机的最简单方法是使用安全工作站，从中他们可以执行管理任务提供您的 IT 员工。 在仅限工作站的实现中，每个管理工作站用于启动管理工具和管理服务器和其他基础结构的 RDP 连接。 仅限工作站的实现是很有效的较小的组织，虽然更大、 更复杂的基础结构可能会受益于分布式设计为在其中管理主机专用管理服务器和工作站的使用，但作为本主题后面的"实现安全管理工作站和跳转服务器"中所述。  
  
### <a name="implementing-separate-physical-workstations"></a>实现单独的物理工作站  
可以实现管理主机的一种方法是发出两个工作站的每个 IT 用户。 多个工作站的"常规"用户帐户用于执行活动，例如查看电子邮件和使用的工作效率应用程序，而第二个工作站严格限于管理功能。  
  
工作效率工作站，IT 人员可以为给定的常规用户帐户，而不是使用特权的帐户登录到未受保护计算机上。 管理工作站应使用严格受控配置进行配置和 IT 人员应使用不同的帐户登录到管理工作站上。  
  
如果已实现智能卡，应将管理工作站配置为需要智能卡登录和 IT 人员应提供独立的帐户供管理使用，还配置为以进行交互式登录需要智能卡。 管理主机，应加以强化按前面所述，并仅指定的 IT 用户有权在本地登录到管理工作站。  
  
#### <a name="pros"></a>优点  
通过实现单独的物理系统，可以确保每台计算机为其角色适当地配置和 IT 用户不能无意中公开的风险管理系统。  
  
#### <a name="cons"></a>缺点  
  
-   实现的单独物理计算机会增加硬件成本。  
  
-   登录到一台物理计算机与用于管理远程系统的凭据缓存在内存中的凭据。  
  
-   如果不安全地存储管理工作站，它们可能容易受到威胁通过如物理硬件击键记录器或其他物理攻击的机制。  
  
### <a name="implementing-a-secure-physical-workstation-with-a-virtualized-productivity-workstation"></a>实现虚拟化的工作效率工作站的安全物理工作站  
在此方法中，IT 为用户提供从中他们可以执行日常管理功能的安全管理工作站的责任范围内他们使用远程服务器管理工具 (RSAT) 或 RDP 连接到服务器。 当 IT 用户需要执行的工作效率的任务时，它们可以通过 RDP 进行连接到远程工作效率工作站作为虚拟机运行。 应为每个工作站使用单独的凭据，应实现控件，如智能卡。  
  
#### <a name="pros"></a>优点  
  
-   管理工作站和工作效率工作站被隔离。  
  
-   使用安全工作站连接到的工作效率工作站的 IT 人员可以使用单独的凭据和智能卡和特权的凭据都未将存储在安全级别较低的计算机上。  
  
#### <a name="cons"></a>缺点  
  
-   实现该解决方案需要设计和实现工作和可靠的虚拟化选项。  
  
-   如果不安全地存储物理工作站，它们可能受到物理攻击的破坏的硬件或操作系统并使其成为可能的通信被截取。  
  
### <a name="implementing-a-single-secure-workstation-with-connections-to-separate-productivity-and-administrative-virtual-machines"></a>实现单独的"生产力"和"管理"的虚拟机的连接的单一安全工作站  
在这种方法，可以发出 IT 用户单个物理工作站的锁住按前面所述，并在其上 IT 用户不具有特权的访问。 可以提供远程桌面服务连接到专用服务器上托管的虚拟机提供一个电子邮件和其他工作效率应用程序中，运行的虚拟机和第二个虚拟机配置为用户的 IT 人员专用管理主机。  
  
您应该要求智能卡或其他多因素登录虚拟机使用单独的帐户，而不用于登录到物理计算机上的帐户。 IT 用户登录到物理计算机后，他们可以使用自己的工作效率的智能卡连接到远程工作效率计算机和单独的帐户和智能卡连接到其远程管理计算机。  
  
#### <a name="pros"></a>优点  
  
-   IT 用户可以使用单个物理工作站。  
  
-   通过为虚拟的主机和使用远程桌面服务连接到虚拟机需要单独的帐户，不会在本地计算机上的内存中缓存 IT 用户的凭据。  
  
-   可以管理主机，从而降低的本地计算机受到安全威胁的可能性程度保护物理主机。  
  
-   在其中某个 IT 用户的工作效率虚拟机或其管理的虚拟机可能已泄露的情况下，虚拟机可以轻松地重置为"已知的有用"状态。  
  
-   如果物理计算机遭到破坏，无特权的凭据将被缓存在内存中，并将智能卡可以阻止通过击键记录器的凭据的侵害。  
  
#### <a name="cons"></a>缺点  
  
-   实现该解决方案需要设计和实现工作和可靠的虚拟化选项。  
  
-   如果不安全地存储物理工作站，它们可能受到物理攻击的破坏的硬件或操作系统并使其成为可能的通信被截取。  
  
### <a name="implementing-secure-administrative-workstations-and-jump-servers"></a>实现安全管理工作站和跳转服务器  
作为替代方法来安全管理工作站，或与它们结合使用，可实现安全的跳转服务器，并管理用户可以连接到使用 RDP 和智能卡执行管理任务的跳转服务器。  
  
跳转服务器应配置为运行远程桌面网关角色来允许你连接到跳转服务器并将从其管理的目标服务器上实现限制。 如果可能，应该还安装 HYPER-V 角色，并创建[个人虚拟桌面](https://technet.microsoft.com/library/dd759174.aspx)或管理用户使用其任务跳转服务器上的其他每个用户虚拟机。  
  
通过在跳转服务器上为管理用户提供每个用户的虚拟机，为管理工作站，提供物理安全性和管理用户可以重置或关闭在不使用其虚拟机。 如果不想要管理的同一主机上安装 HYPER-V 角色和远程桌面网关角色，你可以在单独的计算机上安装它们。  
  
如有可能，应使用远程管理工具来管理服务器。 应用户的虚拟机 （或跳转服务器，如果您不想实现每个用户以进行管理的虚拟机） 上安装远程服务器管理工具 (RSAT) 功能和管理人员应连接通过 rdp 连接到其虚拟机，以执行管理任务。  
  
在情况下时管理用户必须通过 RDP 连接到目标服务器，以直接管理 RD 网关应配置为允许连接到相应的用户和计算机用于建立到目标的连接才成为服务器。 执行工具的 RSAT （或类似） 应在未指定管理系统的系统上如常规使用的工作站和成员的服务器不跳转服务器禁止。  
  
#### <a name="pros"></a>优点  
  
-   创建跳转服务器允许您将映射到"区域"（具有类似的配置、 连接和安全要求的系统的集合） 在网络中的特定服务器，并要求每个区域管理通过管理人员从安全管理主机到指定的"区域"服务器连接。  
  
-   通过将跳转服务器映射到区域中，您可以实现的连接属性和配置要求的精细控制，并可以轻松地识别尝试从未经授权的系统连接。  
  
-   通过跳接服务器上实现每个管理员虚拟机，您强制执行关闭和重置到已知的干净状态的虚拟机的管理任务完成时。 通过完成管理任务时，强制执行关闭 （或重新启动） 的虚拟机，虚拟机不能为目标的攻击者，也不是可行的凭据被盗攻击，因为内存中缓存的凭据不会持久保留之外重新启动。  
  
#### <a name="cons"></a>缺点  
  
-   无论是物理还是虚拟专用的服务器需要跳转服务器。  
  
-   实现指定跳转服务器和管理工作站需要进行仔细的规划和配置映射到在环境中配置任何安全区域。
