# <a name="identifying-the-deployment-project-participants"></a>标识部署项目参与者

>适用于：Windows Server 2016，Windows Server 2012 R2、Windows Server 2012

建立部署项目 Active Directory 域服务 (广告 DS) 的第一步是建立设计，将负责管理设计阶段和部署阶段 Active Directory 项目的部署项目团队重启。 此外，你必须识别的个人和将负责拥有和维护目录部署完成后的组。  
  
-   [定义特定于项目角色](#BKMK_1)  
  
-   [所有者和管理员建立](#BKMK_2)  
  
-   [构建项目团队](#BKMK_3)  
  
## <a name="BKMK_1"></a>定义特定于项目角色  
建立项目团队重要步是找出要按项目特定角色的人员。 其中包括执行发起人项目设计师、，项目经理。 这些人负责运行 Active Directory 部署项目。  
  
你指定的项目设计师和项目经理后，这些人建立整个组织进行通信的频道、 安排项目和识别将项目团队的成员开头各种所有者的人员。  
  
### <a name="executive-sponsor"></a>执行发起人  
部署基础结构，如广告 DS 会会对组织广泛造成影响。 出于此原因，很重要有行政发起人理解部署的业务值支持官，在项目，可以帮助解决在组织内冲突。  
  
### <a name="project-architect"></a>项目设计师  
每个 Active Directory 部署项目需要项目设计师管理 Active Directory 设计和部署决策过程。 设计师提供的技术协助设计和部署广告 DS 的过程。  
  
> [!NOTE]  
> 如果你的组织中没有现有的人员体验目录设计，你可能希望雇佣是方面的专家 Active Directory 设计和部署的外部顾问。  
  
Active Directory 项目设计师责任如下所示：  
  
-   拥有 Active Directory 设计  
  
-   了解和录制主要设计决策的原因  
  
-   确保设计满足组织的业务需要  
  
-   建立共识之间设计、 部署和运营团队  
  
-   了解广告 DS 集成应用程序需要  
  
最终的 Active Directory 设计必须反映商业目标和 technical 决策的组合。 因此，项目设计师必须先检查设计决策，以确保他们对准商业目标。  
  
### <a name="project-manager"></a>项目经理  
项目经理便于合作跨设备业务和技术管理组之间。 理想、 Active Directory 部署项目经理是由熟悉这两个运营和的策略 IT 组准备部署广告 DS 组的设计要求组织中的人从。 项目经理开头设计，一直到实现，监督整个部署项目中，并确保该项目保持按计划并在 budget。 项目经理责任如下所示：  
  
-   提供基本的项目，如日程安排和预算计划  
  
-   驾驶进度 Active Directory 设计和部署项目  
  
-   确保相应个人所涉及的设计过程中的每个部分  
  
-   作为单一 Active Directory 部署项目的联系人的房子  
  
-   建立之间设计、 部署和运营团队通信  
  
-   建立和维护部署项目整个执行发起人通信  
  
## <a name="BKMK_2"></a>所有者和管理员建立  
在 Active Directory 部署项目中，有所有者人士负责由持有管理，以确保该完成任务的部署和规范满足组织该 Active Directory 设计。 所有者不一定执行有权访问或直接操作目录基础结构。 管理员是负责完成所需的部署任务的每个人。 管理员拥有网络访问权限必要目录和其基础结构进行操作。  
  
负责人的角色是战略和管理。 所有者负责通信管理员到所需的 Active Directory 设计，如新的域控制器树林中创建的实现的任务。 管理员是负责实现设计，设计规范根据网络上。  
  
在较大的组织中，其他个人填充所有者和管理员的角色：但是，某些小型企业中的同一个人可能作为所有者和管理员。  
  
### <a name="service-and-data-owners"></a>所有者服务和数据  
管理每天广告 DS 涉及两种类型的所有者：  
  
-   同样负责 Active Directory 基础结构的规划和长期维护以及确保目录继续运行，并建立 service 级别协议中的目标维护服务所有者  
  
-   数据负责目录中存储的信息的维护的所有者。 这包括用户和计算机帐户管理和如成员服务器和工作站本地资源的管理。  
  
请务必，以便他们可以参与尽可能多的尽可能设计过程初期识别 Active Directory 服务和数据所有者。 服务和数据的所有者负责目录长期维护部署项目完毕后，因为务必为这些人提供有关的公司需要输入和熟悉方式和原因进行了某些设计决策。 所有者服务包括森林所有者、 Active Directory 域名系统 (DNS) 所有者以及网站拓扑所有者。 所有者数据包括部门 (OU) 所有者。  
  
### <a name="service-and-data-administrators"></a>服务和数据管理员  
广告 DS 操作涉及的管理员两种类型： 管理员和管理员数据提供服务。 服务管理员实现由服务所有者策略决策，并处理与维护目录服务和基础结构的日常任务。 这包括管理承载目录服务，管理其他网络服务，如广告 DS，控制配置的森林范围的设置，并确保目录始终可用，需要具备的 DNS 域控制器。  
  
还有负责完成持续 Active Directory 部署任务所需的初始 Windows Server 2008 Active Directory 部署过程完成后的服务管理员。 例如，根据目录增加的要求，服务管理员创建其他域控制器和建立域，之间信任根据需要或删除。 出于此原因，需要包括服务管理员 Active Directory 部署团队。  
  
你必须要小心，若要仅为受信任的人组织中分配服务管理员角色。 由于这些人修改域控制器上的系统文件的功能，因此他们可以更改 DS 广告的行为。 你必须确保你的组织的服务管理员个人熟悉运营和安全策略，在网络上的位置并谁了解需执行这些的策略。  
  
管理员数据是域的在域中同时维护广告 DS 如用户和组帐户中存储的数据以及维护的成员，各自的计算机负责用户。 管理员数据控制的对象目录中子集，无法安装或目录服务配置的控制。  
  
默认情况下不提供数据管理员帐户。 设计团队确定资源的方式组织进行管理之后，域的所有者必须创建数据管理员帐户并将它们委派根据的对象即将负责管理员套相应的权限。  
  
最好是限制，才能确保基础结构继续运行的最低号码组织中的服务管理员数。 管理工作大部分可以完成由管理员数据。 服务管理员要求更宽的技能，设置，因为它们负责维护目录和支持该功能的基础结构。 管理员数据只需管理其一部分目录所需的技巧。 划分以这种方式工作分配会导致组织成本，因为仅少量管理员需要培训运行和维护整个目录并提供其基础结构。  
  
例如的服务管理员需要了解如何添加到林域。 其中包括如何安装该软件转换域控制器服务器以及如何操作 DNS 环境，以便域控制器可以无缝合并为 Active Directory 环境。 只需知道如何管理它们负责如为新的员工其部门创建新的用户帐户的特定数据数据管理员。  
  
部署广告 DS 需要协调和网络的基础结构的操作所涉及的许多不同组之间进行通信。 这些组应指定的服务和数据负责表示设计和部署过程的各个组的所有者。  
  
完成部署项目后，这些服务和数据所有者继续负责由其组的基础结构的部分。 在 Active Directory 环境，这些所有者是森林所有者、 广告 DS 所有者 DNS、 网站拓扑所有者和 OU 所有者。 这些服务和数据负责人的角色是以下部分中所述。  
  
#### <a name="forest-owner"></a>森林所有者  
森林所有者通常是负责 Active Directory 部署过程，并且谁最终负责部署完成后保持林中服务交付组织中的高级信息 (IT) 技术经理。 森林所有者分配个人以填充的其他所有权角色通过识别组织中的关键人员可以提供关于网络的基础结构和管理需求必需的信息。 森林所有者负责如下：  
  
-   部署的森林根域创建森林  
  
-   每个域创建森林所需的域中的第一个域控制器的部署  
  
-   会员身份的服务管理员域中的组所有的森林  
  
-   森林中的每个域 OU 结构设计的创建  
  
-   管理授权委派给 OU 所有者  
  
-   对模式的更改  
  
-   更改树林配置设置  
  
-   某些使用组策略策略设置包括域用户帐户的策略，如精细的密码和帐户锁定策略实现  
  
-   适用于域控制器的业务策略设置  
  
-   任何其他域级别应用组策略设置  
  
森林所有者通过整个森林具有机构。 这是森林所有者责任组策略和业务策略设置并选择是服务管理员的人员。 森林所有者是服务所有者。  
  
#### <a name="dns-for-ad-ds-owner"></a>广告 DS 所有者 DNS  
广告 DS 所有者 DNS 不具有现有 DNS 基础结构彻底的了解和组织现有命名空间的人员。  
  
广告 DS 所有者 DNS 负责以下事项：  
  
-   作为设计团队和当前拥有 DNS 基础结构 IT 组之间联系  
  
-   提供有关现有 DNS 空间要帮助创建的新的 Active Directory 命名空间的组织的信息  
  
-   使用部署团队，以确保 DNS 新的基础结构部署根据设计团队的规范且正常工作  
  
-   管理用于广告 DS 基础结构，包括 DNS 服务器服务和 DNS 数据 DNS  
  
广告 DS 所有者 DNS 是服务所有者。  
  
#### <a name="site-topology-owner"></a>网站拓扑所有者  
网站拓扑所有者是熟悉组织网络，包括映射的单个子网、 路由器和 slow 链接通过连接的网络方面的物理结构。 网站拓扑所有者负责以下事项：  
  
-   了解物理网络拓扑以及该法规如何影响广告 DS  
  
-   了解如何 Active Directory 部署将影响该网络  
  
-   如何确定要创建 Active Directory 逻辑站点  
  
-   添加一个子网时，更新域控制器站点对象修改或删除  
  
-   创建网站的链接、 站点链接桥和手动连接对象  
  
网站拓扑所有者是服务所有者。  
  
#### <a name="ou-owner"></a>OU 所有者  
OU 所有者管理目录中存储的数据负责。 此个人需要熟悉运营和在网络上的位置中的安全策略。 OU 所有者可以执行的服务管理员，委派给他们的这些任务，并且他们可以对分配到华丽绚烂执行这些的任务。 可能分配给 OU 所有者任务，如下所示：  
  
-   执行其指定 OU 内的所有帐户的管理任务  
  
-   工作站和属于其指定 OU 成员服务器管理  
  
-   委派给其指定 OU 内的本地管理员权限  
  
OU 所有者是数据所有者。  
  
## <a name="BKMK_3"></a>构建项目团队  
Active Directory 项目团队是临时负责完成 Active Directory 设计和部署任务的组。 完成 Active Directory 部署项目时，所有者负责目录，并可以 disband 项目团队。  
  
根据组织的大小的大小项目团队而异。 小型企业，在一个人可以键盘盖上的项目团队负责的多个领域和参与部署的多个阶段。 较大的团队，有不同的人或甚至不同涵盖责任的不同区域的团队，可能需要大型企业。 只要分配的所有方面的责任，并且满足组织的设计的目标，并不重要团队的大小。  
  
### <a name="identifying-potential-forest-owners"></a>确定潜在森林所有者  
确定你的组织中的组拥有或网络上的用户提供目录服务所需的资源的控制。 这些组算是潜在的森林所有者。  
  
服务和数据管理广告 DS 中的分隔可 IT 组 （或多个组） 的组织管理目录服务在每个组中的本地管理员管理属于自己组的数据的基础结构。 潜在森林所有者通过网络基础结构，以将其部署和支持广告 DS 具有所需的权限。  
  
有一个集中基础结构 IT 组的组织的 IT 组通常是森林所有者和，因此，对于任何将来部署潜在森林所有者。 包含大量的组织的独立基础结构 IT 组具有大量潜在森林所有者。 如果您的组织已经已经之后 Active Directory 的基础结构，任何当前森林所有者也会潜在用于部署新的森林所有者。  
  
选择一个潜在的森林所有者，使其作为正在考虑以便进行部署每个森林的森林所有者。 这些潜在的森林所有者负责处理设计团队，以确定将实际上部署其森林或备用采取的操作 （如加入另一个现有森林） 如果可用资源更好地使用，并且仍能满足他们的需求。 在你的组织的森林所有者 （或所有者） 是 Active Directory 设计团队的成员。  
  
### <a name="establishing-a-design-team"></a>建立设计团队  
Active Directory 设计团队负责收集做出 Active Directory 逻辑结构设计有关所需的所有信息。  
  
设计团队的职责如下所示：  
  
-   确定多少林和域都需要和关系的是域林还是  
  
-   数据所有者以确保他们的安全和管理要求满足设计处理  
  
-   与当前的网络管理员以确保当前的网络基础结构支持设计，设计不会产生负面影响现有的应用程序部署到该网络上的工作  
  
-   使用代表的组织的安全该组，以确保设计满足建立的安全策略  
  
-   设计允许相应级别的保护和颁发机构正确委派给数据所有者的 OU 结构  
  
-   使用部署团队测试设计在实验环境，以确保它充当计划和修改设计为所需解决发生的任何问题  
  
-   创建站点拓扑设计，以满足阻止的可用带宽重载时的森林复制要求。 有关设计站点拓扑的详细信息，请参阅[设计网站拓扑为 Windows Server 2008 广告 DS](https://technet.microsoft.com/library/cc772013.aspx)。  
  
-   使用部署团队，以确保正确实现设计  
  
设计团队包含以下成员：  
  
-   潜在森林所有者  
  
-   项目设计师  
  
-   项目经理  
  
-   个人负责建立和维护网络上的安全策略  
  
过程中逻辑结构设计，设计团队标识其他所有者。 这些人必须开始参与设计过程尽快对它们进行标识。 部署项目传递到部署团队后，设计团队负责监督部署过程，以确保正确实现设计。 设计团队还可根据测试反馈的设计进行更改。  
  
### <a name="establishing-a-deployment-team"></a>建立部署团队  
Active Directory 部署团队负责测试和实现 Active Directory 逻辑结构设计。 这包括以下任务：  
  
-   建立足够模拟生产环境测试环境  
  
-   通过在验证符合的每个角色所有者目标实验环境中实现提出的森林和域结构测试设计  
  
-   开发和测试任意迁移方案提出的实验环境中的设计  
  
-   确保在每个所有者登录上测试进程以确保测试正确的设计功能  
  
-   测试在先导环境中部署操作  
  
完成后设计和测试任务，部署团队将执行以下任务：  
  
-   森林并域的 Active Directory 逻辑结构设计根据创建  
  
-   站点上创建和根据需要的站点链接对象根据站点拓扑设计  
  
-   确保 DNS 基础结构配置支持广告 DS 和任何新命名空间集成组织现有命名空间  
  
Active Directory 部署团队包含以下成员：  
  
-   森林所有者  
  
-   广告 DS 所有者 DNS  
  
-   网站拓扑所有者  
  
-   所有者 OU  
  
在部署阶段，以确保操作团队的成员熟悉新设计，部署团队适用于的服务和数据管理员。 这有助于部署操作完成后，请确保顺利过渡拥有成本。 部署过程完成后，在维护新 Active Directory 环境责任将传递到操作团队。  
  
### <a name="documenting-the-design-and-deployment-teams"></a>记录设计和部署的团队  
记录的名称，并将参与的设计和部署广告 DS 的人员联系信息。 确定谁将每个角色有关设计和部署的团队负责。 最初，此列表将包括潜在的森林所有者，项目经理，项目设计师。 当你确定将部署的森林数时，你可能需要创建新的设计团队的更多森林。 请注意，你将需要更新你的文档，随着团队成员更改以及您在设计过程识别各种 Active Directory 所有者。 对于要向你提供帮助记录每个林设计和部署团队中工作表，下载 Job_Aids_Designing_and_Deploying_Directory_and_Security_Services.zip ([https://go.microsoft.com/fwlink/?LinkID=102558](https://go.microsoft.com/fwlink/?LinkID=102558))，然后打开"设计和部署团队的信息"(DSSLOGI_1.doc)。
