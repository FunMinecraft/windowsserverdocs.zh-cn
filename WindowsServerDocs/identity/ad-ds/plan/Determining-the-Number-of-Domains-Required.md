# <a name="determining-the-number-of-domains-required"></a>确定所需的域数量

>适用于：Windows Server 2016 中，Windows Server 2012 R2、 Windows Server 2012

每个林启动与单一域。 最慢的链接，必须在考虑域控制器和可用带宽，你想要分配给 Active Directory 域服务 (AD DS) 之间的复制为基础的单个域林中可以包含的用户数上限。 下表列出了建议的用户的域可以包含基于单个域林中，速度最慢的链接，并想要保留复制的带宽的百分比数目的最大值。 此信息适用于林包含最多 100000 个用户和已建立了连接，为 28.8 千位 / 秒 (Kbps) 或更高版本。 将应用于包含 100,000 多个用户或小于 28.8 Kbps 的连接的林的建议，请咨询有经验的 Active Directory 设计器。 下表中的值基于在具有以下特征的环境中生成的复制流量：  
  
- 新用户加入每 20 年 %的速度的林。  
- 用户将保留在林中每 15 年 %的费率收费。  
- 每个用户是五个全局组和五个通用组的成员。  
- 访问计算机的用户的比率是 1:1。  
- 使用 active Directory 集成域名系统 (DNS)。  
- 使用 DNS 清理。  

> [!NOTE]  
> 下表中列出的数字是近似值。 复制流量的数量很大程度上取决于在给定的时间内对目录所做的更改数。 确认你的网络通过部署你的域前在实验室中测试的估计的数量和速率上您的设计的更改可以适应您的复制流量。  
  
|最慢的链接连接的域控制器 (Kbps)|用户如果 1%的带宽可用的最大数量|用户如果 5%的带宽可用的最大数量|用户如果 10%的带宽可用的最大数量|  
| --- | --- | --- | --- |  
|28.8|10,000|25,000|40,000|  
|32|10,000|25,000|50,000|  
|56|10,000|50,000|100,000|  
|64|10,000|50,000|100,000|  
|128|25,000|100,000|100,000|  
|256|50,000|100,000|100,000|  
|512|80,000|100,000|100,000|  
|1,500|100,000|100,000|100,000|  

若要使用此表：  

1. 在中**速度最慢的链接连接的域控制器**列中，找到匹配的速度最慢的链接，AD DS 将在你的域中复制的值。  

2. 在行中对应于最慢的链接速度，找到的列，表示你想要分配到 AD DS 的百分比带宽。 在该位置的值是可以包含单个域林中的域的用户的最大数目。  

如果您确定在您的林中的用户总数小于最大可包含你的域的用户数，可以使用单个域。 请确保以适应计划未来的增长时做出此判断。 如果您确定在您的林中的用户总数大于最大可包含你的域的用户数，您需要保留更高版本的复制，带宽的百分比提高链接速度，或将划分到你的组织地区域。  
  
## <a name="dividing-the-organization-into-regional-domains"></a>除以单位分成若干个地区域

如果您不能容纳所有单个域中的用户，则必须选择区域域模型。 将您的单位分成若干个区域对你的组织和现有的网络有意义的方式。 例如，可以创建基于洲际边界的区域。  
  
请注意，因为您需要创建您建立每个区域的 Active Directory 域，我们建议你尽量减小的定义适用于 AD DS 的区域数量。 尽管可以包含无数个域的林中，但对于可管理性建议一个林包含不超过 10 个域。 必须建立优化复制带宽和除组织到地区域时最大程度减少管理复杂性之间的适当平衡。  
  
首先，确定的最大可以托管你的林的用户数。 这根据林中的域控制器将复制速度最慢的链接和你想要分配给 Active Directory 复制的平均带宽量。 下表列出了建议的一个林中可以包含的用户数的最大值。 这基于速度最慢的链接和想要保留复制的百分比带宽。 此信息适用于林包含最多 100000 个用户和已连接的 28.8 Kbps 或更高版本。 下表中的值基于以下假设：  

- 所有域控制器都是全局编录服务器。  
- 新用户加入每 20 年 %的速度的林。  
- 用户将保留在林中每 15 年 %的费率收费。  
- 用户是五个全局组和五个通用组的成员。  
- 访问计算机的用户的比率是 1:1。  
- 使用 active Directory 集成的 DNS。

|最慢的链接连接的域控制器 (Kbps)|用户如果 1%的带宽可用的最大数量|用户如果 5%的带宽可用的最大数量|用户如果 10%的带宽可用的最大数量|  
| --- | --- | --- | --- |  
|28.8|10,000|50,000|75,000|

如果需要托管的用户数大于最大可以托管你的林的用户数，单个林将适用于您的设计。 如果需要托管用户的数量多于你确定的最大数目，您需要以增加最小的链接速度，适用于 AD DS 分配更大的带宽百分比或部署其他林。  

如果您确定单个林将容纳需要托管的用户数，则下一步是确定的最大可以支持每个区域的用户数基于最慢的链接位于该区域中。 将你的林划分为对您有意义的区域。 请确保您作出决定的某些内容不太可能发生更改。 例如，而不是销售区域中使用大洲。 这些区域时已确定的最大用户数将您的域结构的基础。  

确定基于速度最慢的链接速度和带宽分配给该区域中的 AD DS 的需要托管在每个区域中，然后验证它们不能超过允许的最大用户数。 下表列出了建议的地区域可以包含的用户数的最大值。 它基于最慢的链接的速度和想要保留复制的带宽的百分比。 此信息适用于林包含最多 100000 个用户和已连接的 28.8 Kbps 或更高版本。 下表中的值基于以下假设：
