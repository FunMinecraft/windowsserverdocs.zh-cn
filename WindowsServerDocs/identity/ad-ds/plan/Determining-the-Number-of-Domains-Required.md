# <a name="determining-the-number-of-domains-required"></a>如何确定所需的域的号码

>适用于：Windows Server 2016，Windows Server 2012 R2、Windows Server 2012

每个树林开始单个域。 最大的单个域森林可能包含的用户数基于必须容纳域控制器和你想要分配给 Active Directory 域服务 (广告 DS) 可用带宽之间复制慢链接。 下表列出了建议域可能包含基于单个域森林的速度慢链接，以及带宽复制你希望保留的百分比的用户数的最大。 此信息适用于森林 100000 用户最多的包含还有 28.8 千 / 秒 (Kbps) 或更高版本的连接。 森林包含 10 万多个用户或小于 28.8 kbps 连接到应用的建议，请查阅经验丰富的 Active Directory 设计。 下表中的值基于具有以下特点环境中生成复制交通：  
  
-   新的用户加入家庭组的每 20 年 %速率森林。  
  
-   用户都离开的每 15 年 %速率森林。  
  
-   每个用户都的五个全球组和五通用组成员。  
  
-   访问计算机的用户的比 1:1。  
  
-   使用 active Directory 集成域名系统 (DNS)。  
  
-   使用 DNS 清理。  
  
> [!NOTE]  
> 下表中列出的数字都近似值。 复制通信数量主要取决于数在给定时间内到的目录所做的更改。 确认你的网络，可以按测试的估计的数量和更改在你的设计的速率实验部署你域之前容纳复制通信。  
  
|慢连接域控制器 (Kbps) 的链接|最大如果 1%带宽可用的用户数|最多 5%带宽是否可用的用户数|最多 10%带宽是否可用的用户数|  
|--------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|-----------------------------------------------------------------|  
|28.8|10,000|25,000|40,000|  
|32|10,000|25,000|50,000|  
|56|10,000|50,000|100,000|  
|64|10,000|50,000|100,000|  
|128|25,000|100,000|100,000|  
|256|50,000|100,000|100,000|  
|512|80,000|100,000|100,000|  
|1,500|100,000|100,000|100,000|  
  
若要使用此表：  
  
1.  在**连接域控制器慢链接**列中，找到你域跨的广告 DS 将复制的慢链接相匹配的值。  
  
2.  在对应于你的最低链接速度行中，找到代表你想要分配给广告 DS 的百分比带宽的列。 在该位置的值为最多可以包含一个域森林中的域的用户数。  
  
如果你确定你森林中的用户总数小于最多可以包含你的域的用户数，你可以使用单个域。 请务必容纳计划将来成长，当您进行此确定。 如果你确定你森林中的用户的总数量大于最多可以包含你的域的用户数，你需要保留带宽复制，较高百分比提高速度链接，或将你的组织分成若干个区域的域。  
  
## <a name="dividing-the-organization-into-regional-domains"></a>在除以分成若干个区域的域组织  
如果你不能满足您的所有用户单个域中，你必须选择区域域型号。 将你的组织分成若干个区域的方式有意义的是你的组织和你现有的网络。 例如，你可能会创建地区根据大陆限制。  
  
请注意，你将需要创建建立每个区域的域的 Active Directory，因为我们建议你使您的广告 DS 定义的地区大量降至最低。 尽管树林中包含的域无数，但的可管理性建议森林包括不超过 10 域。 你必须建立之间优化复制带宽，并将你管理复杂程度降至最低当你的组织分成若干个区域的域的分隔相应的余额。  
  
首先，确定最大的可以容纳你森林的用户数。 跨的域控制器将复制森林中的慢链接为基础，并且想要分配给 Active Directory 复制平均带宽量。 下表列出了建议森林可能包含的用户数的最大。 这取决于的速度慢链接以及你希望保留复制的百分比带宽。 此信息适用于森林 100000 用户最多的包含并且连接到的 28.8 Kbps 或更高版本。 下表中的值基于下面假设：  
  
-   所有域控制器将都是全球目录服务器。  
  
-   新的用户加入家庭组的每 20 年 %速率森林。  
  
-   用户都离开的每 15 年 %速率森林。  
  
-   用户都五个全球组通用的五个组中的成员。  
  
-   访问计算机的用户的比 1:1。  
  
-   使用 active Directory 集成 DNS。  
  
-   使用 DNS 清理。

> 下表中列出的数字都近似值。 复制通信数量主要取决于数在给定时间内到的目录所做的更改。 确认你的网络，可以按测试的估计的数量和更改在你的设计的速率实验部署你域之前容纳复制通信。

|--------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|-----------------------------------------------------------------|

若要使用此表：  
  
1.  在**连接域控制器慢链接**列中，找到慢你森林跨的广告 DS 将复制的链接相匹配的值。  
  
2.  在对应于你的最低链接速度行中，找到代表你想要分配给广告 DS 的百分比带宽的列。 在该位置的值为最大的可以容纳你森林的用户数。  
  
如果最大的可以容纳你森林的用户数大于需要托管的用户的数量，单个森林适用于你的设计。 如果你需要在主机超过你标识最大数目更多的用户，你需要以加快最小的链接，分配带宽一个更高百分比的广告 DS 或部署其他森林。  
  
如果你确定单个森林将容纳你需要在主机的用户的数量下, 一步是确定最多可以支持的每个区域的用户数基于慢位于该区域中的链接。 将你的森林划分为使感知到你的地区。 请确保你基本某些你很难更改的决策。 例如，而不是销售地区中使用大陆。 当你已确定最大的用户数将的基础结构，域这些区域。  
  
确定基于最低链接速度和分配给该区域中的广告 DS 带宽需要承载在每个区域中，然后确保他们不能超过允许的最大的用户的数量。 下表列出了建议可能包含区域的域的用户数的最大。 它所采用的速度慢链接和带宽复制你希望保留的百分比。 此信息适用于森林 100000 用户最多的包含并且连接到的 28.8 Kbps 或更高版本。 下表中的值基于下面假设：

尽管你可以将其放入某个域多域森林中的用户数总小于单个域森林中的域中的用户的数量，总多域森林中的用户数可以更高版本。 较小的每个域多域森林中的用户数容纳通过维护中的全局目录该环境中创建的其他复制开销。 森林包含 10 万多个用户或小于 28.8 kbps 连接到应用的建议，请查阅经验丰富的 Active Directory 设计。
