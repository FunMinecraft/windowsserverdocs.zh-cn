# <a name="executive-summary"></a>摘要

>适用于：Windows Server 2012

>[!IMPORTANT] 
>以下文档 2013 年写入了，并作为历史记录仅提供。  当前我们正在查看此文档和很可能会发生更改。  它可能不会反映当前的最佳实践。

没有组织提供的信息 (IT) 技术基础结构是免受攻击，但如果相应策略、进程，并控制实现保护主要段组织的计算基础结构，可能会以防止泄露事件的计算环境大规模危及变。  
  
本摘要旨在很有用作为独立文档汇总文档，其中包含建议，以有助于中增强的安全其 Active Directory 安装组织的内容。 通过实现这些建议，组织将能够确定优先安全活动、保护他们的组织的计算基础结构的关键段和创建大幅降低成功攻击 IT 环境中的关键组件的可能性的控件。  
  
尽管本文讨论抵御 Active Directory 和对策减少攻击 surface 的最常见的攻击，它还包含用于恢复如果完全受损的建议。 如果发生的 Active Directory 完成危害恢复仅确保方法是准备危害之前发生这种情况。  
  
本文档的主要部分如下：  
  
-   危害的途径  
  
-   减少 Active Directory 攻击 Surface  
  
-   危害的迹象监视 Active Directory  
  
-   对于危害计划  
  
## <a name="avenues-to-compromise"></a>危害的途径  
此部分中提供了对某些最常利用的漏洞攻击者用于危害客户的基础结构的信息。 它包含常规漏洞以及如何在使用它们最初穿透客户的基础结构，在更多系统传播危害最终目标 Active Directory 和域控制器，以便获得完全控制组织的森林类别。 它不提供有关解决每种类型的漏洞，尤其是在的区域中的漏洞不能用来直接目标 Active Directory 详细的建议。 但是，每种漏洞，我们已提供用于开发对策减少组织的攻击 surface 的其他信息的链接。  
  
包括以下主题：  
  
-   **初始违反目标**-大多数信息的安全漏洞的组织的基础结构通常一个或两个系统小段危害与启动一次。 这些初始事件或入口点到网络中，通常利用的漏洞可能已修复，但并不。 常见的漏洞是：  
  
    -   在防病毒和恶意软件的部署间距  
  
    -   不完整的修补程序  
  
    -   过期的应用和操作系统  
  
    -   错误配置  
  
    -   缺少的安全的应用程序开发做法  
  
-   **吸引帐户凭据盗用**-凭据盗窃攻击是指在其中攻击最初获得到网络上计算机的访问权限，然后使用免费提供的工具来自其他登录帐户的会话解压缩的凭据。   
    在此部分中包含如下所示：  
  
    -   **增加了可能性危害的活动**高权限的域帐户，"非常重要的人"的凭据被盗目标通常是因为 (VIP) 帐户重要的是管理员意识增加的凭据被盗攻击成功的可能性的活动。 这些活动是：  
  
        -   在登录到特权帐户的计算机不安全  
  
        -   浏览 Internet 高权限帐户  
  
        -   系统在具有相同的凭据配置权限的本地帐户  
  
        -   Overpopulation 和特权的域组滥用  
  
        -   管理不足域控制器的安全性。  
  
    -   **特权提升和传播**-特定帐户、服务器和基础结构组件通常是对的 Active Directory 攻击主要目标。 将这些帐户：  
  
        -   永久特权的帐户  
  
        -   VIP 帐户  
  
        -   "特权附加"Active Directory 帐户  
  
        -   域控制器  
  
        -   影响如公共基础结构 (PKI) 服务器和系统管理服务器的身份、访问和配置管理其他基础结构服务  
  
## <a name="reducing-the-active-directory-attack-surface"></a>减少 Active Directory 攻击 Surface  
本部分着重于 technical 控件，以减少攻击的 Active Directory 安装。 在此部分中包括以下主题：  
  
-   **特权帐户和 Active Directory 中的组**部分讨论的最高权限的帐户和 Active Directory 和特权的帐户受保护的机制中的组。 Active Directory 内, 三个内置组是（企业管理员、域管理和管理员）的目录中的最高权限组尽管应该还保护数量的其他组和帐户。  
  
-   **实现最低权限管理模型**部分致力于标识使用高度特权风险帐户的日常管理礼物此外为您提供建议，为了减少该风险。  
  
过多特权仅未受到威胁的环境中找到 Active Directory 中。 当单位开发授予比需要更多特权的习惯时，它通常整个基础结构找到：  
  
-   Active Directory 中  
  
-   成员服务器上  
  
-   工作站上  
  
-   在应用程序  
  
-   中的数据存储库维护  
  
-   **实现安全管理主机**部分介绍安全管理主机，是计算机配置为支持 Active Directory 和连接的系统的管理。 这些主机专用于管理功能，可能无法运行如电子邮件应用程序、web 浏览器或生产力软件的软件（如 Microsoft Office)。  
  
在此部分中包含如下所示：  
  
-   **创建安全管理主机的原则**-常规的原则，需要注意的是：  
  
    -   永远不会从受信任的不太主机管理受信任的系统。  
  
    -   不要依赖于单个身份验证因素时执行特权的活动。  
  
    -   不要忘记物理时设计和实现安全管理主机的安全。  
  
-   **保护域控制器针对攻击**-如果恶意用户获取了域控制器特权访问权限，该用户可以修改、损坏，并破坏 Active Directory 数据库中，并通过扩展的所有系统和由 Active Directory 的帐户。  
  
在此部分中包括以下主题：  
  
-   **对于域控制器物理安全**-包含推荐提供了域控制器在数据中心、分支机构和遥远物理的安全性。  
  
-   **域控制器操作系统**-包含用于保护域控制器操作系统的建议。  
  
-   **安全的域控制器配置**-本机免费提供的配置工具和设置可用于创建安全配置基线，随后由组策略对象 (Gpo) 强制执行的域控制器。  
  
## <a name="monitoring-active-directory-for-signs-of-compromise"></a>危害的迹象监视 Active Directory  
此部分中提供了有关旧版审核类别和审核策略子类别中（其中引入了在 Windows Vista 和 Windows Server 2008），以及高级审核策略（在 Windows Server 2008 R2 引入了）的信息。 此外提供了是事件和监视器指示尝试危害环境，可用于构建全面审核策略 Active Directory 的一些其他参考对象的信息。  
  
在此部分中包括以下主题：  
  
-   **Windows 审核策略**-Windows 安全事件日志具有类别和子类别中，以确定的安全事件跟踪并记录。  
  
-   **审核策略建议**-本部分介绍 Windows 默认审核策略设置，建议使用通过 Microsoft，并的组织的更多危害程度大建议使用审核关键服务器和工作站的审核策略设置。  
  
## <a name="planning-for-compromise"></a>对于危害计划  
此部分中包含了有助于组织发生之前准备危害的建议、可检测到的危害事件完整违反发生之前, 和目录完全受损通过攻击者的情况下提供响应和恢复指南实现控件。 在此部分中包括以下主题：  
  
-   **构思方法**-包含原则和指南创建到其中组织可以使其最重要的资产安全环境。 这些原则如下所示：  
  
    -   确定分离和保护措施关键资产原则  
  
    -   定义风险基于受限、迁移套餐  
  
    -   利用"nonmigratory"迁移，在必要的地方  
  
    -   实现"创意销毁"  
  
    -   隔离传统系统和应用程序  
  
    -   为最终用户简化安全  
  
-   **维护更安全的环境**-包含高级建议打算使用作为指南用于开发不仅有效的安全，而且有效的生命周期管理。 在此部分中包括以下主题：  
  
    -   **创建适用于 Active Directory 业务中心的安全措施**-要实际上管理用户、数据、应用程序和管理通过 Active Directory 的系统的生命周期，请按照这些原则。  
  
        -   **Active Directory 数据分配业务所有权**-分配的基础结构组件到所有权 IT;数据，添加到 Active Directory 域服务（广告 DS）支持企业，例如，新的员工，新的应用程序和新信息存储库，指定的业务设备或用户应该关联的数据。  
  
        -   **实现 Business-Driven 生命周期管理**-应 Active Directory 中的数据实现生命周期管理。  
  
        -   **分类 Active Directory 的所有数据**-商家应提供分类 Active Directory 中的数据。 数据分类模型，在以下 Active Directory 数据分类应包含以下内容：  
  
            -   **系统**-分类服务器人口、其操作系统他们的职位中，它们和 IT 上运行的应用和业务所有者的记录。  
  
            -   **应用程序**-分类的功能，用户基础其操作系统的应用程序。  
  
            -   **用户**-应标记和监视帐户最有可能攻击 Active Directory 安装。  
  
## <a name="summary-of-best-practices-for-securing-active-directory-domain-services"></a>安全保障 Active Directory 域服务最佳实践摘要

大致优先级，该是顺序列出做法，较低的数字表示较高的优先级。 在适用的最佳做法被标识为预防或侦探自然景观中。 这些建议的所有应该会彻底测试和修改根据需要为你的组织的特征，并要求。

||**最佳做法**|**策略或战略**|**预防或侦探**|

| 2 |操作系统的修补程序。|策略 |预防 |  
| 3 |部署和及时跨所有系统和监视器上尝试删除或禁用它的更新防病毒和恶意软件。|策略 |同时 |

| 5 |保护和监视器帐户有权访问敏感数据的用户 |策略 |同时 |
