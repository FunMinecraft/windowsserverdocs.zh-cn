# <a name="administer-software-restriction-policies"></a>管理软件限制策略

>适用于：Windows Server 2016，Windows Server 2012 R2、Windows Server 2012

此主题以供 IT 专业人员包含过程如何管理应用程序控件策略与 Windows Server 2008 和 Windows Vista 使用软件限制策略 (SRP) 的开头。

## <a name="introduction"></a>简介
软件限制策略 (SRP) 是基于组策略的功能，以标识某个域中，在计算机上运行的软件程序，并控制这些程序能够运行。 你可以使用软件限制策略若要创建受限高度对于你在其中允许仅明确识别运行的应用程序的计算机，配置。 这些组策略与 Microsoft Active Directory 域服务集成，但也可单独的计算机上配置。 有关 SRP 的详细信息，请参阅[软件限制策略](software-restriction-policies.md)。

开始与 Windows Server 2008 R2 和 Windows 7、 Windows AppLocker 可用于或配合 SRP 控制策略应用程序的一部分。

本主题包含：

-   [若要打开软件限制策略](#BKMK_Open_SRP)

-   [若要创建新的软件限制策略](#BKMK_Create_SRP)

-   [若要防止应用于管理员的本地软件限制策略](#BKMK_Prevent_Admin)

-   [若要更改的软件限制策略的默认安全级别](#BKMK_Sec_Lvl)

-   [若要将软件的限制策略应用到 Dll](#BKMK_Apply_SRP_DLLs)

有关如何完成使用 SRP 特定的任务，请参阅以下信息：

-   [确定软件限制策略允许拒绝列表和应用程序目录获利](determine-allow-deny-list-and-application-inventory-for-software-restriction-policies.md)

## <a name="BKMK_Open_SRP"></a>若要打开软件限制策略

-   [有关本地计算机](#BKMK_1)

-   [对于域，站点或单位，以及你是成员服务器或已加入域的工作站上](#BKMK_2)

-   [对于域或单位，并且你是该域控制器上或在安装远程服务器管理工具的工作站上](#BKMK_3)

-   [为某个站点，并且你是该域控制器上或在安装远程服务器管理工具的工作站上](#BKMK_4)

### <a name="BKMK_1"></a>有关本地计算机

1.  打开本地安全设置。

2.  在控制台树中，单击**软件限制策略**。

    **位置？**

    -   安全设置/软件限制策略

> [!NOTE]
> 若要执行此过程，你必须是管理员组在本地计算机上的成员，或者你必须已授予相应的权限。

### <a name="BKMK_2"></a>对于域，站点或单位，以及你是成员服务器或已加入域的工作站上

1.  打开 Microsoft 管理控制台 (MMC)。

2.  在**文件**菜单上，单击**添加/删除管理单元在**，然后单击**添加**。

3.  单击**本地组策略编辑器**，然后单击**添加**。

4.  在**选择组策略对象**，单击**浏览**。

5.  在**浏览组策略对象**，选择相应的域、 网站或单位中的组策略对象 (GPO)-或创建一个新相册，然后单击**完成**。

6.  单击**关闭**，然后单击**确定**。

7.  在控制台树中，单击**软件限制策略**。

    **位置？**

    -   *组策略对象*[*ComputerName*] 策略中的计算机配置或

        用户配置中的 Windows 设置中的安全/软件设置限制策略

> [!NOTE]
> 若要执行此步骤，必须是管理员域组成员。

### <a name="BKMK_3"></a>对于域或单位，并且你是该域控制器上或在安装远程服务器管理工具的工作站上

1.  打开组策略管理控制台。

2.  在控制台树中，右键单击你想要打开软件限制策略组策略对象 (GPO)。

3.  单击**编辑**以打开你想要编辑的 GPO。 你还可以单击**新建**以创建一个新 GPO，然后单击**编辑**。

4.  在控制台树中，单击**软件限制策略**。

    **位置？**

    -   *组策略对象*[*ComputerName*] 策略中的计算机配置或

        用户配置中的 Windows 设置中的安全/软件设置限制策略

> [!NOTE]
> 若要执行此步骤，必须是管理员域组成员。

### <a name="BKMK_4"></a>为某个站点，并且你是该域控制器上或在安装远程服务器管理工具的工作站上

1.  打开组策略管理控制台。

2.  在控制台树中，右键单击你想要设置为组策略网站。

    **位置？**

    -   Active Directory 的站点和服务 [*Domain_Controller_Name.Domain_Name*] / 站点/网站

3.  单击中的某个条目**组策略对象链接**以选择现有组策略对象 (GPO)，然后单击**编辑**。 你还可以单击**新建**以创建一个新 GPO，然后单击**编辑**。

4.  在控制台树中，单击**软件限制策略**。

    **在哪里**

    -   *组策略对象*[*ComputerName*] 策略中的计算机配置或

        用户配置中的 Windows 设置中的安全/软件设置限制策略

> [!NOTE]
> -   若要执行此过程，你必须是管理员组在本地计算机上的成员，或者你必须已授予相应的权限。 如果计算机已加入域的域管理员组成员可以要执行此过程。
> -   若要设置将应用到计算机，无论哪用户登录到它们的策略设置单击**计算机配置**。
> -   若要设置将应用到他们登录到，无论哪台计算机的用户的策略设置单击**用户配置**。

## <a name="BKMK_Create_SRP"></a>若要创建新的软件限制策略

1.  打开限制策略的软件。

2.  在**操作**菜单上，单击**新软件限制策略**。

> [!WARNING]
> -   执行此过程中，具体取决于您的环境所需不同管理凭据：
> 
>     -   对于本地计算机创建新的软件限制策略： 在当地的会员**管理员**组中或等效是最低要求完成此过程。
>     -   如果你创建新软件限制策略加入域的计算机的域管理员组成员可以执行此过程。
> -   如果已创建软件限制策略为组策略对象 (GPO)，**新软件限制策略**未显示在命令**操作**菜单。 若要删除的应用于 GPO，控制台树中的软件限制策略右键单击**软件限制策略**，然后单击**删除软件限制策略**。 当您为 GPO 删除软件限制策略时，还该 gpo 删除所有软件限制策略规则。 删除软件限制策略后，你可以为 GPO 创建新的软件限制策略。

## <a name="BKMK_Add_Del"></a>若要添加或删除指定的文件类型

1.  打开限制策略的软件。

2.  在详细信息窗格中，双击**指定的文件类型**。

3.  请执行以下任一操作：

    -   若要添加的文件类型，在**文件扩展名**、 键入文件扩展名，，然后单击**添加**。

    -   若要删除的文件类型，请在**指定的文件类型**，单击该文件类型，然后单击**删除**。

> [!NOTE]
> -   执行此过程中，具体取决于你添加或删除指定的文件类型的环境所需不同管理凭据：
> 
>     -   如果你添加或删除你的本地计算机指定的文件类型： 在当地的会员**管理员**组中或等效是最低要求完成此过程。
>     -   如果你创建新软件限制策略加入域的计算机的域管理员组成员可以执行此过程。
> -   可能需要创建新的软件限制策略设置的组策略对象 (GPO)，如果你没有。
> -   列表中指定的文件类型由所有规则共享计算机配置和用户配置为 GPO。

## <a name="BKMK_Prevent_Admin"></a>若要防止应用于管理员的本地软件限制策略

1.  打开限制策略的软件。

2.  在详细信息窗格中，双击**强制**。

3.  下**适用于以下的用户的软件限制策略**，单击**以外的本地管理员的所有用户**。

> [!WARNING]
> -   在当地的会员**管理员**组中或等效是最低要求完成此过程。
> -   可能需要创建新的软件限制策略设置的组策略对象 (GPO)，如果你没有。
> -   如果很常见的用户将他们在你的组织的计算机上本地中的成员，你可能不希望启用该选项。
> -   如果你正在为您的本地计算机定义软件限制策略设置，使用此过程，以防止本地管理员软件应用到他们的限制策略。 如果你正在定义软件限制策略设置为你的网络，筛选根据安全通过组策略的组成员的用户策略设置。

## <a name="BKMK_Sec_Lvl"></a>若要更改的软件限制策略的默认安全级别

1.  打开限制策略的软件。

2.  在详细信息窗格中，双击**安全级别**。

3.  右键单击你想要设为默认值，然后单击安全级别**设为默认值**。

> [!CAUTION]
> 在某些的目录中设置的默认安全级别到**不允许**可能会影响你的操作系统。

> [!NOTE]
> -   执行此过程中，具体取决于环境软件限制策略的默认安全级别更改为其所需不同管理的凭据。
> -   可能需要创建新的软件限制策略设置此组策略对象 (GPO)，如果你拥有不已完成操作。
> -   详细信息窗格中，有一个复选标记在其中的黑色圆圈表示当前的默认安全级别。 如果你通过右键单击当前的默认安全级别**设为默认值**命令未显示在菜单中。
> -   若要指定的默认安全级别例外创建软件限制策略规则。 当设置为默认安全级别**无限制**，规则可以指定不得运行的软件。 当设置为默认安全级别**不允许**，规则可以指定允许运行的软件。
> -   在安装软件限制策略，在你的系统上的所有文件的默认安全级别设置为**无限制**。

## <a name="BKMK_Apply_SRP_DLLs"></a>若要将软件的限制策略应用到 Dll

1.  打开限制策略的软件。

2.  在详细信息窗格中，双击**强制**。

3.  下**适用于以下软件限制策略**，单击**软件的所有文件**。

> [!NOTE]
> -   若要执行此过程，你必须是管理员组在本地计算机上的成员，或者你必须已授予相应的权限。 如果计算机已加入域的域管理员组成员可以要执行此过程。
> -   默认情况下，不软件限制策略检查动态链接库 (Dll)。 检查 Dll 可以减少系统性能，因为必须评估软件限制策略，每次 DLL 加载。 但是，你可以决定检查 Dll，如果你担心接收该目标 Dll 的病毒。 如果的默认安全级别设置为**不允许**，并启用检查，你必须创建软件限制策略规则允许所有 DLL 运行时 DLL。
